ABSTRACT
======================================================================
GEME (Genetic Economy Memory Entity) is a minimal cognitive unit —
a "Geuron" — implemented in approximately 200 lines of Python.
It forms concepts through competitive memory, observes its own
activity through co-occurrence tracking, and consolidates through
inductive pruning. Without pretrained weights, gradients, or
external objectives, GEME reproduces three mathematical boundaries
at the computational level: Godel's Second Theorem (the system
cannot verify its own evaluation function), and Tarski's geometric
primitive (triangle is axiom-level, not derivable from connection
primitives). The Tarski wall is subsequently broken by a second
Geuron that generates its own vocabulary from the first layer's
internal statistics — an instantiation of Wittgenstein's language
game (meaning as use) running in code.

The full stack (GEME1→GEME2→GEME3→GEME4) produces recursive
cognitive layers where each layer's concepts become the next
layer's perceptual elements. Chain frames (══) between concepts
serve as proto-theorems, enabling the simplest form of endogenous
inference without predefined rules.

The entire system was built in three days through a human-AI
dialogue. The human contributed philosophical vision (Generative
Information Realism, a framework predicting these boundaries);
the AI contributed exhaustive implementation and analysis.

1. INTRODUCTION
======================================================================

On the third day, the system generated its own vocabulary.
The human stared at the output and said: "Wait. This is the
Wittgenstein thing." [S1] The AI noted that the newly formed
symbols — "conn_2" and "conn_3" — had no external referents.
Their meaning was entirely determined by their position in the
system's frame dynamics: which frames they merged with, what
weight they accumulated, which other symbols they co-occurred with.
This is Wittgenstein's language game (Philosophical Investigations,
sec. 43), running in code.

Four days earlier, the human had downloaded a programming interface
during a holiday and uploaded three rough philosophical papers
arguing that Godel's incompleteness theorems should not be read
as static limits but as generative boundaries — as engines of
cognitive growth rather than barriers to it. This framework,
Generative Information Realism (GIR), had been developed over
three years without computational support. It predicted the
existence of "boundary encounters" (碰数) — moments when a
self-referential system reaches its own logical limit and undergoes
a phase transition into a higher cognitive layer.

GEME was built to test this prediction. The system is a minimal
cognitive unit — a "Geuron" — with three mechanisms.

First, competitive memory: each input formula is encoded as a
frequency vector over a 27-symbol alphabet. Vectors compete for
limited frame capacity; frames merge when similar, are created
when novel, and are evicted when low-weight.

Second, co-occurrence self-reference: a sliding window tracks
which structural signatures appear within proximity 3. Pairs
that co-occur frequently form association frames (──);
associations sharing elements form chain frames (══).
The system observes its own activity.

Third, inductive consolidation: periodic pruning evicts the
lowest-weight half of frames, with unmerged frames decaying
faster than merged ones. This is the competitive selection that
drives compression.

These three mechanisms — and nothing else — produced everything
reported in this paper.

The experiments that follow are structured around four questions:
Can a system form conceptual domains without domain labels (Exp 0)?
Can it form abstract relational categories from concrete instances
(Exp 1)? Can it encounter mathematical boundaries naturally (Exp 2,
Exp 3)? And can it break those boundaries by generating its own
vocabulary, building higher cognitive layers from its own output
(Exp 3, closed loop)?

2. RESULTS
======================================================================

2.1 Experiment 0: Autonomous Concept Classification

Two formula types — algebra (swap) and set theory (succ) — were fed
into a single GEME instance in random 50/50 order (n=400 each).
No domain labels were provided.

Frame cluster purity: 100%. All algebra frames occupied one
cluster; all set frames occupied the other. Within-domain
associations: 67% (2/3 associations). Cross-domain associations:
33% (expected from window co-occurrence noise). Baselines
(frequency counting, online K-means) required pre-assigned
cluster counts and produced no concept hierarchies.

Conclusion: conceptual domain boundaries emerge from information
competition alone. The system divided its own world.

2.2 Experiment 1: Commutativity as Emergent Category

Concrete commutative instances — swap(1,2)=swap(2,1),
swap(3,5)=swap(5,3) — were fed without universal quantifiers
or variables (n=600, 70% commutative, 30% identity). Control:
directional instances with different function name (dir).

The system spontaneously separated into two layers: instance
frames (weight ~7) holding specific number pairs, and an
association frame (weight ~47) encoding the shared pattern.
New commutative instances with unseen numbers: S2 (recognized).
Directional controls: S3 (rejected). Frequency inversion test
(70%→30%): concept persisted across all 5 seeds (42-46).
Compression ratio: 320:1 versus baseline vector storage.

Conclusion: Kant's category of Relation (Community/Reciprocity)
is not an a priori given but an emergent structure from
informational difference. Instance (Layer 0) and concept (Layer 1)
separate automatically.

2.3 Experiment 2: The Godel Boundary

All inputs received the same structural signature (eq_succ).
Two types: ordered chains (succ(s0)=s1, succ(s1)=s2...) and
scrambled pairs (succ(s5)=s2, succ(s8)=s3...), each 50% (n=800).

50 runs (seeds 42-91): ordered recognized as concept at 100%;
scrambled recognized at 100%. The wall was absolute. Control:
functions with different signatures (next, other) correctly
rejected (100%). Threshold sweep (overlap 0.70-0.99): wall
persists at all levels. Internal dynamics: no stress spike —
the wall is structural, not dynamic.

The system's evaluate_sig() function checks structural signature
overlap. It cannot check semantic validity. This presents a
computational analog of the limitation described in Godel's
Second Theorem: the system cannot verify the consistency of its
own evaluation function. The boundary encounter (碰数) is
structurally necessary, not a data or threshold artifact.

2.4 Experiment 3: The Tarski Wall

Pure connection inputs — conn(A,B), conn(B,C), conn(C,A) — all
with the same signature (eq_conn). Two types: closed triangles
(3 assertions) and open chains (2 assertions), each 50% (n=600).

100 runs (seeds 42-141): ALL conn inputs collapsed into ONE
concept (mean association weight 1864.5, sd=6.0). Closed vs open:
completely indistinguishable. This is Tarski's (1959) result —
"triangle is an axiom-level primitive, not derivable from
connection primitives" — confirmed computationally. The human,
who had never heard of Tarski, discovered him by searching for
the wall. [S1]

Control: with distinguishing function names (conn_closed vs
conn_open), the system separated immediately (2 frames). The
wall was signature identity, not system failure.

2.5 Experiment 3b: Breaking the Wall — The Wittgenstein Table

Layer 1 detected burst sizes in its co-occurrence window
(3 connections together for closed; 2 for open) but could not
express this distinction — all same signature (eq_conn).

The vocabulary generation procedure (detailed in Methods 4.3):
(1) extract Layer 1's dominant signature content word ("conn");
(2) detect burst sizes [2,3] from window gap analysis;
(3) auto-generate function names: "conn_2" and "conn_3";
(4) instantiate Layer 2 with these names;
(5) Layer 2 forms separate concepts.

100 runs: conn_3 recognized at 100%, conn_2 at 85%. Association
weight mean 237.4, sd 222.4 (variance reflects seed-dependent
burst distribution; qualitative result — wall broken — stable
across all runs). The vocabulary was based on Layer 1's own
statistics. No human labels at any step.

The AI noted: "The meaning of these symbols is their use in the
system." [S1] This is Wittgenstein's language game (Philosophical
Investigations, sec. 43), running in 200 lines.

This separation provides a computational model of Kant's
intuition/concept distinction (Critique of Pure Reason, A51/B75):
Layer 1 felt the difference but had no symbol (intuition without
concept — blind); Layer 2 had symbols but no access to content
(concept without intuition — empty). Together, they produced
knowledge.

2.6 Closed Loop: Parallel GEMEs → Integration → Theorem Chains

The full architecture (Fig. 4) places multiple GEMEs in parallel,
each receiving different data modalities:

  GEME1 (open chains):     conn(A,B), conn(B,C)       → concept
  GEME2 (closed triangles): conn(A,B), conn(B,C), conn(C,A) → concept
  GEME3 (integration): receives BOTH GEME1+2 frame signatures
                        as function names → forms 3 axioms
  GEME4 (meta): receives GEME3 output → 2 theorem chains

GEME3 produced 3 concept frames (integration axioms) and 2 chain
frames (══) from their shared elements (ratio 0.7). These chains
encode proto-theorems: "if concept A relates to concept B, and
concept B relates to concept C, then A and C are connected through
B" — the formal structure of transitivity and, at higher layers,
geometric congruence (SSS, SAS).

GEME4 forms triadic chains (A══B══C══A), the architectural
signature of closure. This is the first running instantiation
of endogenous theorem generation — inference without predefined
rules, emerging from frame dynamics alone.

3. DISCUSSION
======================================================================

GEME's architecture differs fundamentally from deep learning.
It does not minimize a loss function; it competes for memory.
It does not backpropagate; it observes its own co-occurrence
statistics. It does not optimize; it prunes by weight.

The walls GEME encountered — Godel's boundary and Tarski's
geometric primitive — are not bugs. They are structural
consequences of operating with finite alphabetic capacity.
Every cognitive system with sufficient power to form stable
concepts will encounter concepts it cannot evaluate. GEME makes
this observable in running code (Experiment 2: 50 runs, 100%).

The Generative Information Realism (GIR) framework predicted
these boundaries three years before GEME existed. The framework's
two phase transition types — intrinsic (rule restructuring within
a cognitive layer) and architectural (new layer emergence) — both
appear in the experimental results: Experiment 1→2 corresponds to
intrinsic phase transition; Experiment 3→closed loop corresponds
to architectural phase transition.

Limitations. All experiments use 2-4 formula types and ~30 symbols.
The vocabulary generation (Experiment 3b) is a script-level bridge,
not an engine-level mechanism. A fully endogenous implementation
(dynamic alphabet extension) remains future work. GEME forms
concepts but does not understand them; the gap between concept
formation and semantic comprehension is unresolved.

Future directions. Phase Ouroboros: (i) dynamic alphabet extension
within the engine, (ii) multi-Geuron matrix architecture (parallel
GEMEs with automatic cross-observation), (iii) application to
real-world structured data, (iv) scaling the theorem-chain layer
to formal proof discovery.

The mode of production — a human with philosophical vision and an
AI with exhaustive implementation — points toward a new scientific
paradigm. Two hundred lines of code by an outsider with AI
assistance produced behaviors that trillion-parameter models
cannot demonstrate. Scale is not the only path.

4. METHODS
======================================================================

4.1 System Architecture

Default parameters: memory_cap=16, cooccur_window=80,
cooccur_thresh=0.15, max_chains=5 (0 for Exp 3), induction
threshold 0.6, induction minimum interval 15 steps.

Adaptive merge threshold: bootstrap phase (first 10 steps)
collects all nearest-neighbor distances. Threshold = 25th
percentile of bootstrap distances. After bootstrap: median
of last 50 successful merge distances, floored to last_ok×0.5
to prevent zero-deadlock.

Vector encoding: 27-symbol alphabet. Known symbols have dedicated
dimensions; unknown symbols fall into generic "fn"/"const" slots.
Vectors are frequency-normalized. Distance: Euclidean (L2).

Structural signature: tree-traversal collects function names and
keywords (eq, forall, exists); variable and constant values are
discarded. Signatures enable generalization but cause the walls
reported in Experiments 2 and 3 (signature identity masks
semantic difference).

4.2 Evaluation

evaluate_sig(s): signature overlap score =
|test_parts ∩ frame_parts| / min(|test_parts|, |frame_parts|).
S2 if overlap ≥ 0.75 with any frame exceeding median weight.
S3 otherwise.

4.3 Vocabulary Generation Pipeline (Experiment 3b)

The following procedure runs automatically in the experiment
script after Layer 1 completes processing:

1. id_max_frame = argmax(frame.weight in Layer 1)
2. dominant_sig = id_max_frame.sig_full
3. content_word = first non-generic part (not eq/and/forall/empty)
4. Scan window for consecutive occurrences of content_word's
   signature type. Record run lengths ≥ 2. burst_sizes = unique runs.
5. For each bs in burst_sizes: fn_name = content_word + "_" + str(bs)
6. Layer 2 receives these function names as input formulas.
7. Run standard GEME on Layer 2.

This pipeline is automated at the script level but not built
into the GEME engine. A fully endogenous implementation is
identified as future work.

4.4 Reproducibility

Full source code (Supplementary Code S1): ~200 lines Python 3.
Six experiment scripts (Supplementary Code S2-S7): ~50-150 lines each.
Dependencies: standard library only (math, statistics).
Reproduction: <30 seconds per experiment on consumer hardware.
Seeds: 42-91 (Exp 2), 42-141 (Exp 3).
All random seeds produced in advance; no exclusions.
