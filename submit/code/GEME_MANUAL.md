## 第一章 形式化框架（先骨骼，后血肉）

本章使用纯数学语言定义系统的状态、常数、操作和规则。所有符号在此定义，后续章节沿用同一定义，不再重新解释。

### 1.1 状态定义

**定义 1.1（系统状态）**。令系统在离散时间步 $t \in \mathbb{N}$ 的状态为 $S_t = (F_t, \mathcal{H}_t, t)$，其中：

- $F_t = \{f_1, f_2, \ldots, f_n\}$ 为 $t$ 时刻的帧集合，$|F_t| \leq C$（$C$ 为帧容量上限）
- $\mathcal{H}_t$ 为 $t$ 时刻的共现历史记录
- $t$ 为时间步索引

**定义 1.2（帧）**。一个帧 $f \in F_t$ 是一个七元组：

$$f = (\mathbf{v}, w, a, m, \sigma, \rho, \ell)$$

其中：
| 符号 | 名称 | 数学域 | 含义 |
|------|------|--------|------|
| $\mathbf{v}$ | 向量 | $\mathbb{R}^D$，$D = 27$ | 帧的内容表示 |
| $w$ | 权重 | $\mathbb{R}^+$ | 帧的"市场认可度"，越大越不易淘汰 |
| $a$ | 年龄 | $\mathbb{N}$ | 自创建以来的时间步数 |
| $m$ | 合并计数 | $\mathbb{N}$ | 成功合并的次数（影响衰减率） |
| $\sigma$ | 签名 | 字符串（$\leq 30$字符） | 帧的语义标签，用于跨层追踪 |
| $\rho$ | 源标签 | 字符串 | 记录帧的来源（可选） |
| $\ell$ | 层标签 | $\{L_0, L_1, \ldots, L_6\}$ | 帧所属的处理层级 |

**定义 1.3（帧容量）**。$C \in \mathbb{N}^+$ 是系统能同时维护的最大帧数。当 $|F_t| = C$ 且需要创建新帧时，权重最低的帧被淘汰。$C$ 是**用户预设的认知预算参数**，非核心结构常数。默认值 $C = 32$。

**定义 1.4（帧容量 vs. 窗口）**。$C$（帧容量）是系统在**任意时刻**能存储的最大帧数量——它是一个**静态容量约束**。滑动窗口大小 $W$ 是系统在**时间轴上回顾**的步数——它是一个**动态关联检测参数**。二者关系：$C$ 决定"存多少"，$W$ 决定"看多远"。

**定义 1.5（共现历史）**。$\mathcal{H}_t$ 是一个多重集：

$$\mathcal{H}_t = \{ (\sigma_i, \sigma_j, t') \mid t - W < t' \leq t \}$$

其中 $\sigma_i, \sigma_j$ 为在滑动窗口 $W$ 内时间相邻出现的帧签名，$t'$ 为记录时间戳。$W \in \mathbb{N}^+$ 为滑动窗口大小（默认 $W = 50$）。

**定义 1.6（初始状态）**。$t = 0$ 时，$F_0 = \emptyset$，$\mathcal{H}_0 = \emptyset$。

### 1.2 核心结构常数

GEME 仅有的三个核心结构常数（它们是模型中唯三可被调节的"旋钮"）：

| 符号 | 默认值 | 名称 | 功能含义 |
|------|--------|------|---------|
| $\delta$ | 0.19 | 合并自适应缩放系数 | 控制新输入与现有帧合并的严格程度 |
| $\gamma$ | 0.05 | 遗忘衰减率 | 控制帧权重随时间衰减的速度 |
| $\tau$ | 0.60 | 剪枝阈值 | 权重低于此值的帧被淘汰 |

**原则 1**：所有后续定义的派生阈值和系统行为边界必须是 $(\delta, \gamma, \tau)$ 的函数。任何不能表示为三者函数的标量必须明确标注为"固定设计选择"或"用户预设参数"。

### 1.3 基本操作

**定义 1.7（距离度量）**。帧 $f_i$ 与向量 $\mathbf{x}$ 之间的欧氏距离：

$$d(\mathbf{x}, \mathbf{v}_i) = \sqrt{\sum_{j=1}^{D} (x_j - v_{i,j})^2}$$

**定义 1.8（有效合并阈值）**。合并决策取决于一个自适应阈值：

$$\delta_{\text{eff}}(F_t) = \delta \cdot \mu(F_t)$$

其中 $\mu(F_t)$ 是 $F_t$ 中所有帧的成对欧氏距离的均值：

$$\mu(F_t) = \frac{2}{|F_t|(|F_t|-1)} \sum_{i < j} d(\mathbf{v}_i, \mathbf{v}_j)$$

若 $|F_t| < 2$，则 $\mu(F_t) = 1$（初始备用值）。

**定义 1.9（最近帧选择）**。给定输入向量 $\mathbf{x}_t$，最近帧为：

$$f^* = \arg\min_{f_i \in F_t} d(\mathbf{x}_t, \mathbf{v}_i)$$

**定义 1.10（合并操作）**。若 $d(\mathbf{x}_t, \mathbf{v}^*) < \delta_{\text{eff}}(F_t)$，则执行合并：

$$
\mathbf{v}^* \leftarrow \frac{m^* \cdot \mathbf{v}^* + \mathbf{x}_t}{m^* + 1}, \quad
m^* \leftarrow m^* + 1
$$

**定义 1.11（创建操作）**。若 $d(\mathbf{x}_t, \mathbf{v}^*) \geq \delta_{\text{eff}}(F_t)$，则创建新帧：

$$
F_t \leftarrow F_t \cup \{ (\mathbf{x}_t, 1.0, 0, 0, \sigma_{\text{new}}, \rho_{\text{new}}, L_1) \}
$$

若 $|F_t| = C$，则创建前先移除权重最低的帧。

**定义 1.12（遗忘更新）**。每时间步对所有 $f_i \in F_t$：

$$
w_i \leftarrow w_i \cdot \exp\left(-\frac{\gamma}{m_i + \varepsilon}\right), \quad
a_i \leftarrow a_i + 1
$$

其中 $\varepsilon = 10^{-6}$ 为防止除零。合并计数 $m_i$ 越大，衰减越慢。

**定义 1.13（剪枝操作）**。每时间步后：

$$F_t \leftarrow F_t \setminus \{ f_i \mid w_i < \tau \}$$

**定义 1.14（自指向量生成）**。每 $k$ 步（默认 $k = 10$），系统从当前帧集合生成一个自指向量 $\mathbf{o}_t$：

$$
\mathbf{o}_t = \frac{\sum_{f_i \in F_t} w_i \cdot \mathbf{v}_i}{\sum_{f_i \in F_t} w_i}
$$

其中分母为 0 时，$\mathbf{o}_t = \mathbf{0}$（零向量，空经济备用值）。

**定义 1.15（自适应置信度阈值）**。L4 预测层使用一个内生阈值决定是否接受预测：

$$\theta_{\text{conf}}(t) = Q_{0.25}\left( \{\text{conf}_{t-100}, \ldots, \text{conf}_{t-1}\} \right)$$

其中 $Q_{0.25}$ 为最近 100 次预测置信度样本的下四分位数。当样本数 < 20 时，使用初始值 $\theta_{\text{conf}} = 0.3$。

**定义 1.16（L6 怀疑门限）**。L6 激活阈值：

$$ \theta_{\text{doubt}} = \tau $$

L6 解除阈值（带回滞）：

$$ \theta_{\text{clear}} = 1 - 3\gamma $$

### 1.4 核心转移函数

系统状态转移 $T: S_t \rightarrow S_{t+1}$ 在单个时间步内的执行顺序如下：

```
Algorithm 1: 单步状态转移 T(S_t)
─────────────────────────────────────────────
 1:  输入: 外部向量 x_t ∈ ℝ^D, 标签 σ
 2:  ▷ 步骤A: 共现历史记录（基于 W）
 3:  for 上一个输入的签名 σ_prev do
 4:      ℋ_t ← ℋ_t ∪ {(σ_prev, σ, t)}
 5:  
 6:  ▷ 步骤B: L4 预测（基于 ℋ_t 和 σ）
 7:  调用 predict_next(σ, ℋ_t) → (predicted_sig, confidence)
 8:  比较 predicted_sig 与 σ → pred_err 或 correct
 9:  
10:  ▷ 步骤C: 帧竞争合并（基于 δ, C）
11:  f* ← argmin d(x_t, v_i)            ▷ 定义1.9
12:  if d(x_t, v*) < δ_eff(F_t) then    ▷ 定义1.10: 合并
13:      更新 f*.v, f*.m
14:  else                                ▷ 定义1.11: 创建
15:      if |F_t| = C then 移除最低权重帧
16:      F_t ← F_t ∪ {新帧}
17:  
18:  ▷ 步骤D: 遗忘与剪枝（基于 γ, τ）
19:  for f_i ∈ F_t do
20:      w_i ← w_i · exp(-γ / m_i)       ▷ 定义1.12
21:  F_t ← F_t \ {f_i | w_i < τ}        ▷ 定义1.13
22:  
23:  ▷ 步骤E: 自指观察（每 k 步，基于 k, γ）
24:  if t mod k = 0 then
25:      o_t ← 加权质心(F_t)             ▷ 定义1.14
26:      对 o_t 执行步骤A-D（标记为 L4 层）
27:      ▷ 步骤F: L5 元观测更新
28:      更新滚动准确率统计
29:      ▷ 步骤G: L6 统摄判断
30:      if 准确率 < θ_doubt then 生成 sys_doubt 帧
31:      else if 准确率 > θ_clear then 清除怀疑状态
32:  
33:  t ← t + 1
```

> **关键设计原则**：唯三的核心结构常数 $\delta, \gamma, \tau$ 在转移函数中的使用位置已用注释标注。任何一步操作都可以追溯到这三个常数中的至少一个。$C, W, k$ 是用户预设参数（认知预算），非模型参数。

### 1.5 派生阈值总表（可追溯至 $\delta/\gamma/\tau$）

| 派生阈值 | 定义式 | 数值 | 依赖常数 | 功能域 |
|---------|--------|------|---------|--------|
| $\theta_{\text{dw}}$ | $\gamma \times 0.4$ | 0.02 | $\gamma$ | L4 d(w)/dt 显著信号检测 |
| $\theta_{\text{stable}}$ | $\gamma \times 0.2$ | 0.01 | $\gamma$ | 帧元稳定判定 |
| $\theta_{\text{doubt}}$ | $\tau$ | 0.60 | $\tau$ | L6 怀疑激活 |
| $\theta_{\text{clear}}$ | $1 - 3\gamma$ | 0.85 | $\gamma$ | L6 怀疑解除（回滞） |
| $\theta_{\text{healthy}}$ | $1 - 4\gamma$ | 0.80 | $\gamma$ | 系统健康基线 |
| $\theta_{\text{anom\_med}}$ | $\tau - 2\gamma$ | 0.50 | $\gamma, \tau$ | 中等异常线 |
| $\theta_{\text{anom\_high}}$ | $4\gamma$ | 0.20 | $\gamma$ | 高异常线 |
| $\lambda_{\text{decay\_unmerged}}$ | $\exp(-\gamma / 0.25)$ | 0.819 | $\gamma$ | 从未合并帧的单步生存率 |
| $\lambda_{\text{decay\_merged}}$ | $\exp(-\gamma)$ | 0.951 | $\gamma$ | 至少合并一次帧的单步生存率 |

> **注**：乘子 0.4、0.2、0.25 等是体现特定认知假设（如"未合并帧更不稳定"）的固定标量。消融实验已确认系统对这些乘子的变化不敏感（见鲁棒性实验）。

## 第二章 帧竞争动力学

> 本章使用第一章定义的符号，用自然语言解释机制的运作原理。所用术语与形式化章节严格一致。

### 2.1 稀缺性驱动的帧竞争

帧容量 $C$ 是经济分析的起点。$C$ 是有限的——这与 $\mathbb{R}^D$ 的连续空间形成了核心矛盾：输入空间是无限的，但表征空间是有限的。这个矛盾驱动了整个系统的动力学。

当输入向量 $\mathbf{x}_t$ 到达时，所有现有帧 $f_i \in F_t$ 通过距离函数 $d(\mathbf{x}_t, \mathbf{v}_i)$ 竞争"更新权"（定义 1.9）。获胜者 $f^*$ 获得与输入合并的资格——但仅当 $d(\mathbf{x}_t, \mathbf{v}^*) < \delta_{\text{eff}}(F_t)$ 时。这个条件确保新输入不能劫持一个完全不相关的帧。

合并后的帧质心向新输入偏移（定义 1.10），权重间接增加（因为更高的 $m$ 意味着更慢的衰减）。这是帧的"学习"方式：不是梯度下降，而是**内容更新 + 经济奖赏**。

**示例（L1 实体帧竞争）**：

```
输入 "猫" → 与 f1(狗.w=3.2), f2(猫.w=2.1) 计算距离
         → f2 获胜（d = 0.05 < δ·μ）
         → f2.vec 向"猫"更新, f2.merged++
         → f2 的衰减率下降（exp(-γ/3) > exp(-γ/2)）
```

### 2.2 遗忘的"维护成本"效应

遗忘（定义 1.12-1.13）不是被动信息丢失——它是主动维护成本。在容量 $C$ 的限制下，保留一个帧的代价是它占用的槽位无法分配给其他帧。遗忘机制确保低效帧被自动淘汰。

关键因素：**合并计数 $m$**。$m$ 越大，衰减越慢。一个被验证过 10 次的帧的衰减率是 $\exp(-\gamma/10)$，远低于一个从未被验证的帧的衰减率 $\exp(-\gamma/\varepsilon)$。这实现了**渐进知识巩固**——无需外部反馈信号。

### 2.3 共现历史与关联发现

滑动窗口 $W$ 记录了帧签名在时间上的共现（定义 1.5）。当 $(\sigma_i, \sigma_j)$ 在 $\mathcal{H}_t$ 中的出现频率超过阈值 $\theta_{\text{cooccur}}$（默认 0.25）时，系统创建 L2 关联帧。

**形式化**：关联帧 $f^{\text{assoc}}$ 的向量是两个帧向量的拼接均值，签名设置为 ASSOC_SEP 分隔的格式。

关联帧的生存动力与 L1 帧相同：它们参与竞争，如果后续输入证实了关联则权重增加，否则被遗忘剪枝。

### 2.4 自指循环的闭合

自指（定义 1.14）是系统获得"自我感知"的唯一机制。系统每 $k$ 步生成 $\mathbf{o}_t$（帧经济的加权质心），并将其作为新输入反馈。$\mathbf{o}_t$ 不是外部信号——它是系统对自己内部状态的**摘要表示**。

这个循环的闭合效果是渐进的：初始时 $\mathbf{o}_t$ 近似于主要帧向量的加权平均，但随着 $t$ 增加，$\mathbf{o}_t$ 逐渐包含关于 $F_t$ 整体结构的更多信息（因为 $\mathbf{o}_t$ 本身也被合并到 $F_t$ 中，影响后续的 $\mathbf{o}_{t+k}$）。

## 第三章 参数网状结构

### 3.1 参数层次

```
┌─ 用户预设参数 ──────────────────────────────────────────┐
│   C = 32  (帧容量上限 — "认知预算")                       │
│   W = 50  (滑动窗口大小 — "时间回顾范围")                 │
│   k = 10  (自指间隔 — "自我审计频率")                     │
│   D = 27  (向量维度 — "符号空间大小")                     │
└─────────────────────────────────────────────────────────┘
         │
         ▼ 作用于
┌─ 核心结构常数 ──────────────────────────────────────────┐
│   δ = 0.19  → 控制合并严格程度 → 帧分化             │
│   γ = 0.05  → 控制遗忘速率     → 帧更新             │
│   τ = 0.60  → 控制剪枝门槛     → 帧淘汰             │
└─────────────────────────────────────────────────────────┘
         │
         ▼ 推导出
┌─ 派生行为阈值 ──────────────────────────────────────────┐
│   第一章§1.5 总表中的全部 9 个派生阈值                    │
│   全部可表示为 δ/γ/τ 的函数                               │
└─────────────────────────────────────────────────────────┘
         │
         ▼ 自适应校准
┌─ 内生生成本地阈值 ──────────────────────────────────────┐
│   θ_conf(t) = Q0.25({conf_{t-100..t-1}})               │
│   非外部参数——系统从自身置信度分布中动态提取             │
└─────────────────────────────────────────────────────────┘
```

### 3.2 参数之间的独立性

三个核心结构常数 $\delta, \gamma, \tau$ 控制系统的三个正交维度：

| 常数 | 控制 | 影响 | 正交性 |
|------|------|------|--------|
| $\delta$ | 合并严格度 | 帧的分化程度（类别边界宽容度） | 决定 L1 分类的粗/细 |
| $\gamma$ | 遗忘速率 | 帧的更新速度（历史记忆长度） | 决定 L2-L3 关联的持续时间 |
| $\tau$ | 剪枝门槛 | 帧的淘汰底线（保留标准严格度） | 决定 L4-L6 的激活条件 |

消融实验确认：改变 $\delta$ 主要影响 L1 帧数而不影响 L4 行为；改变 $\gamma$ 影响 L2 关联持续性和 L4 预测稳定性；改变 $\tau$ 影响 L6 怀疑激活门槛。三者之间无耦合效应。

### 3.3 工程常数的鲁棒性

用户预设参数 $C, W, k, D$ 在合理范围内变化时，系统的定性行为不变：

- $C \in [8, 48]$：帧数收敛至 $C \pm 2$，L4 预测始终存在
- $W \in [20, 100]$：关联帧数量变化不超过 30%
- $k \in [5, 20]$：自指频率不影响 L4-L6 的存在性，仅影响检测延迟
- $D \in [20, 64]$：扩大编码空间不影响收敛行为

（见鲁棒性实验数据。）

## 第四章 分层信息处理机制

### 4.1 六层架构总览

帧经济的信息处理在六个并发层级上运行，每层对应于不同的抽象级别和处理功能：

```
时间步 t 的完整处理流水线（对应 Algorithm 1）：

  输入 x_t ∈ ℝ^D
     │
     ▼
  ┌─ L0: 输入层 ──────────────────────────────────┐
  │  功能: 接收原始输入                            │
  │  操作: 无处理——仅编码                         │
  │  输出: x_t 及其签名 σ                         │
  └───────────────────────────────────────────────┘
     │ x_t, σ
     ▼
  ┌─ L1: 实体层 ──────────────────────────────────┐
  │  功能: 将输入流离散化为可区分的单元            │
  │  操作: 竞争性合并（定义1.9-1.11）              │
  │  依赖常数: δ                                   │
  │  签名: 输入信号的原始名称                      │
  │  输出: 实体帧集 F_t^(L1) ⊆ F_t                │
  └───────────────────────────────────────────────┘
     │ F_t^(L1)
     ▼
  ┌─ L2: 关联层 ──────────────────────────────────┐
  │  功能: 发现帧之间的时间共现                    │
  │  操作: ℋ_t 中高频率共现 → 创建关联帧          │
  │  依赖常数: γ（关联衰减速度）                   │
  │  签名: σ_i ── σ_j（ASSOC_SEP 分隔）           │
  │  输出: L1 帧之间的关系网络                     │
  └───────────────────────────────────────────────┘
     │ F_t^(L2)
     ▼
  ┌─ L3: 桥接层 ──────────────────────────────────┐
  │  功能: 时间积分——从瞬态关联中提取稳定结构      │
  │  操作: 检测持续共现模式 → 创建桥接帧          │
  │  依赖常数: γ（稳定性衰减判定）                 │
  │  签名: fa ══ fb（CHAIN_SEP 分隔）             │
  │  输出: 跨时间稳定的知识结构                    │
  └───────────────────────────────────────────────┘
     │ ═══════════ 自指边界 ═══════════
     │ F_t^(L3) + 自指向量 o_t
     ▼
  ┌─ L4: 预测层 ──────────────────────────────────┐
  │  功能: 从共现表统计推断下一个最可能出现的帧    │
  │  操作: predict_next(σ, ℋ) → (σ_pred, conf)   │
  │  触发条件: d(w)/dt > θ_dw（= γ × 0.4）       │
  │  比较 σ_pred 与实际 σ → pred_err 帧            │
  │  依赖常数: γ（触发边界）                       │
  │  签名: pred_err_σ / correct_σ                  │
  │  输出: 预测准确/误差信号                       │
  └───────────────────────────────────────────────┘
     │ acc(t)
     ▼
  ┌─ L5: 元观测层 ────────────────────────────────┐
  │  功能: 追踪系统自身的预测准确率                │
  │  操作: 维护滚动窗口的预测准确率统计            │
  │  输出: L4 预测质量的持续评估                   │
  └───────────────────────────────────────────────┘
     │ acc_rolling(t)
     ▼
  ┌─ L6: 统摄层 ──────────────────────────────────┐
  │  功能: 基于 L5 的评估做出系统级的"判断"        │
  │  操作: if acc < θ_doubt → sys_doubt          │
  │         elif acc > θ_clear → 清除怀疑         │
  │  依赖常数: τ（= θ_doubt）, γ（= θ_clear 推导基）│
  │  签名: sys_doubt (怀疑) / 无 (确信)            │
  │  输出: 系统对外部世界的唯一信号——标量语气判断  │
  └───────────────────────────────────────────────┘
```

### 4.2 跨层级自指机制（核心创新）

自指通过两个机制实现跨层连接：

**机制一：签名系统（Sig）**

每一帧的签名 $\sigma$ 是其"身份标签"和"源信息记录"。不同层级的帧签名遵循不同格式：

| 层级 | 签名格式示例 | 含义 |
|------|-------------|------|
| L1 | `cat`, `ext_42` | 输入源名称 |
| L2 | `cat──mat` | 两帧在时间上共现 |
| L3 | `f3══f1` | 两个 L2 关联帧形成稳定桥接 |
| L4 | `dwdw_0.39`, `pred_err_cat` | 权重变化超过 $\theta_{\text{dw}}$，或预测与事实不符 |
| L6 | `sys_doubt` | 系统对当前模型的怀疑信号 |

L4 预测通过查询 $\mathcal{H}_t$ 中与当前输入签名 $\sigma$ 相关的共现记录来工作：

```pseudocode
predict_next(σ_input, H_t):
    从 H_t 中找出所有与 σ_input 共现过的签名 σ_j
    按共现频率排序
    返回最高频者(σ_j, 置信度)
```

这是统计的、非语义的——是频率而非理解。但这个简单的统计机制在帧经济收敛后，产生的预测准确率可达持续高于随机基线。

**机制二：自指反馈循环**

自指向量 $\mathbf{o}_t$（定义 1.14）被当作常规输入处理（标记为 L4 层），经历完整的帧竞争流程。这意味着 $\mathbf{o}_t$ 参与合并、更新共现表、影响后续的 $\mathbf{o}_{t+k}$。这形成了一个**经济自指循环**：系统的自我感知不仅是被观察的状态——它反馈回来影响被观察的状态本身。

### 4.3 为什么是六层

六层结构对应的是一系列不完备性的递归解决：

| 层 | 功能问题 | 不完备性 | 被下一层解决 | 对应的认知类比 |
|----|---------|---------|------------|--------------|
| L1 | "这是什么" | 帧孤立，无法表达关系 | L2 在时间上连接帧 | 命名与分类 |
| L2 | "什么和什么一起出现" | 关联随窗口滑动消失 | L3 在时间上积分 | 因果与关联 |
| L3 | "什么模式是稳定的" | 稳定≠真——无法判断真假 | L4 通过预测检验模型 | 知识结构 |
| L4 | "下一个应该是什么" | 能检测单次预测错误，但不知总体准确率 | L5 追踪历史准确率 | 贝叶斯更新 |
| L5 | "我预测得有多准" | 能观测趋势但不行动 | L6 基于趋势做出判断 | 元认知监控 |
| L6 | "我的模型还有效吗" | 输出最终判断——闭环完成 | —（网络层） | 自我怀疑/确信 |

六层是最小完整路径：去掉任何一层，从"原始输入"到"自我判断"的路径断裂。六层之上的扩展不属于单个认知单元——属于多个单元（geuron 网络）的交互层。
