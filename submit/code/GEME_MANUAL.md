## 第一章 核心原理：桥签名（$\sigma$）

### 1.1 问题：跨层接口

GEME 的分层架构（L1-L6）面临一个基础问题：**每一层如何与下一层通信？**

L1 在向量空间 $\mathbb{R}^D$ 中运算。L4 在共现统计空间中运算。如果 L1 向 L4 传递的是 27 维向量，L4 必须在自己的统计空间中重建 L1 向量的"关系"——这要求 L4 理解向量的几何。但 L4 是一个统计层，不是几何层。

桥签名（$\sigma$）是这一问题的解。

### 1.2 $\sigma$ 的定义

**定义 1.1（桥签名）**。每个帧 $f$ 在创建时被赋予一个字符串 $\sigma_f \in \Sigma^{\leq 30}$，其中 $\Sigma$ 是系统内部字符集。$\sigma_f$ 是帧 $f$ 在层间通信中的唯一标识符。它的约束：

- 不可合并：$\sigma$ 相同的帧才能合并向量——不同的 $\sigma$ 严格隔离
- 唯一性：同一层内不应有两个不同帧共享同一个 $\sigma$
- 可解析：$\sigma$ 的格式必须被下一层的分析器直接消费

**定义 1.2（$\sigma$ 空间）**。系统的"认知"不是在 $\mathbb{R}^D$ 中发生——而是在 $\sigma$ 空间中发生：

- $\mathbb{R}^D$ 的作用：对新输入进行分类，找到它属于哪个已有类别（帧）
- $\sigma$ 的作用：对已分类的输入进行关系分析、模式识别、预测、判断

向量空间是语音信号，$\sigma$ 空间是乐谱。两者缺一不可，但认知发生在后者。

### 1.3 $\sigma$ 作为层间接口协议

每一层的 $\sigma$ 格式是专门为下一层的运算方式设计的：

```
层      σ 格式               设计意图                    被何种运算消耗
──      ──────               ──────────                  ────────────────
L1     "cat", "ext_42"       L2 通过 σ 在 ℋ 中统计共现   共现表按 σ 索引
L2     "cat──mat"            L3 检测稳定组合模式           ── 分隔符匹配
L3     "f3══f1"              L4 用 σ 作预测目标           查 ℋ: σ→σ_next
L4     "pred_err_cat"        L5 按 σ 前缀分类统计         pred_ 前缀过滤
L6     "sys_doubt"           输出——不再被内部层消耗        全系统输出信号
```

> **原则**：$\sigma$ 的存在理由不是"记录帧的身份"——而是**为下一层提供可直接消费的分析对象**。帧的向量 $\mathbf{v}$ 是用来匹配输入的；帧的 $\sigma$ 是用来被上一层分析器检索的。

### 1.4 $\sigma$ 的不可合并性

$\sigma$ 与 $\mathbf{v}$ 的根本区别在于：$\mathbf{v}$ 可以通过加权平均合并（定义 2.6），但 $\sigma$ 不可合并。

两个不同 $\sigma$ 的帧不能合并为一个——即使它们的向量非常接近。这意味着：

1. $\sigma$ 的多样性决定了层内可区分的类别数量
2. $\sigma$ 的不可合并性保护了已建立的概念边界——新输入不会意外地将"猫"和"狗"合并为同一个概念
3. 系统的"学习"不是梯度下降——是 $\sigma$ 生产策略的优化：系统学习生产什么样的 $\sigma$ 能在后续层中获得更准确的预测

**帧经济的本质**：帧不是"知识单元"——帧是 $\sigma$ 的生产单元。帧经济的演化方向是：最大化有价值 $\sigma$ 的稳定存在时间。

### 1.5 帧经济作为 $\sigma$ 生产系统

将帧经济重新描述为 $\sigma$ 生产系统：

```
输入 x_t → 找到/创建帧 f (产生 σ_f)
    → σ_f 被写入共现历史 ℋ_t
    → ℋ_t 被 L2 分析 → 产生新的 σ (关联帧)
    → 关联帧的 σ 被 L3 分析 → 产生新的 σ (桥接帧)
    → 桥接帧的 σ 被 L4 用于预测 → 产生 pred_err/correct σ
    → pred_ 前缀 σ 被 L5 统计 → 产生准确率信号
    → 准确率被 L6 判断 → 产生 sys_doubt 输出
```

每个箭头都是 $\sigma$ 的消费与再生产。系统没有在处理"知识"——在处理 $\sigma$。

## 第二章 帧经济动力学（支撑 $\sigma$ 生产的底层机制）

> 本章定义帧经济如何运作以支撑 $\sigma$ 的稳定生产。

### 2.1 帧的定义

**定义 2.1（帧）**。一个帧 $f$ 是 $\sigma$ 的载体：

$$f = (\mathbf{v} \in \mathbb{R}^D,\ w \in \mathbb{R}^+,\ a \in \mathbb{N},\ m \in \mathbb{N},\ \sigma \in \Sigma^{\leq 30},\ \ell \in \{L_0,\ldots,L_6\})$$

| 分量 | 名称 | 功能 |
|------|------|------|
| $\mathbf{v}$ | 向量 | 输入匹配——决定"这个输入属于哪个 $\sigma$" |
| $w$ | 权重 | 帧在稀缺槽位中的生存概率 |
| $a$ | 年龄 | 自创建以来经过的时间步 |
| $m$ | 合并计数 | $\sigma$ 被验证的次数（影响衰减率） |
| $\sigma$ | 桥签名 | 帧的唯一标识——帧只为生产 $\sigma$ 而存在 |
| $\ell$ | 层标签 | 帧所属的层级 |

帧的存在目的：**接收输入 → 更新向量 → 维护 $\sigma$ 的稳定存在**。

### 2.2 三个核心结构常数

$\delta, \gamma, \tau$ 是控制 $\sigma$ 生产质量的三个约束：

| 符号 | 默认值 | 名称 | 控制对象 | 对 $\sigma$ 生产的影响 |
|------|--------|------|---------|----------------------|
| $\delta$ | 0.19 | 合并缩放系数 | 新输入被分配到哪个 $\sigma$ | $\delta$ 越小，$\sigma$ 分化越精细 |
| $\gamma$ | 0.05 | 遗忘衰减率 | $\sigma$ 的生存时间 | $\gamma$ 越大，$\sigma$ 更新越快 |
| $\tau$ | 0.60 | 剪枝阈值 | 低质量 $\sigma$ 的淘汰 | $\tau$ 越高，$\sigma$ 的淘汰越严格 |

### 2.3 竞争对手：$\sigma$ 的分配

帧容量 $C$（默认 32）限制了一次性能存在的 $\sigma$ 数量。当输入 $\mathbf{x}_t$ 到达时：

1. 所有帧竞争"分配 $\mathbf{x}_t$ 到自己的 $\sigma$"的权利
2. 距离最近的帧获胜（如果距离 < $\delta \cdot \mu(F_t)$）
3. 获胜帧的 $\mathbf{v}$ 向 $\mathbf{x}_t$ 偏移，$\sigma$ 不变
4. 若没有帧足够接近，则创建新帧（新的 $\sigma$）

这是一个**配给制经济**：稀缺的 $\sigma$ 槽位通过竞争分配给最匹配的输入。

### 2.4 遗忘：$\sigma$ 的维护成本

每时间步，所有帧的权重衰减：

$$w_i \leftarrow w_i \cdot \exp\left(-\frac{\gamma}{m_i}\right)$$

$m_i$ 越高（$\sigma$ 被验证越多），衰减越慢。权重低于 $\tau$ 的帧被淘汰——它的 $\sigma$ 从系统中消失。

**关键**：$\sigma$ 的存亡不取决于它是否"重要"——取决于它在稀缺槽位竞争中的经济生存能力。

### 2.5 共现历史：$\sigma$ 之间的关系记录

**定义 2.2（共现历史）**。系统维护一个多重集：

$$\mathcal{H}_t = \{ (\sigma_i, \sigma_j) \mid \sigma_i, \sigma_j \text{ 在窗口 } W \text{ 内相邻出现} \}$$

$\mathcal{H}_t$ 是 L2-L4 运算的数据基础。L2 从 $\mathcal{H}_t$ 中发现高频共现；L3 检测稳定模式；L4 基于 $\mathcal{H}_t$ 做预测。$\mathcal{H}_t$ 是 $\sigma$-空间中的关联存储器。

## 第三章 形式化模型

> 本章以 $\sigma$ 为中心，用数学语言完整定义系统。

### 3.1 状态定义

**定义 3.1（系统状态）**。$S_t = (F_t, \mathcal{H}_t, t)$，$F_t$ 为帧集合，$|F_t| \leq C$。

**定义 3.2（帧）**。$f = (\mathbf{v}, w, a, m, \sigma, \ell)$，见定义 2.1。

**定义 3.3（初始状态）**。$t = 0$：$F_0 = \emptyset$，$\mathcal{H}_0 = \emptyset$。

### 3.2 距离与合并

**定义 3.4（距离）**。$d(\mathbf{x}, \mathbf{v}) = \sqrt{\sum_{j=1}^{D} (x_j - v_j)^2}$。

**定义 3.5（有效合并阈值）**。$\delta_{\text{eff}}(F_t) = \delta \cdot \mu(F_t)$，其中 $\mu(F_t)$ 是 $F_t$ 中帧向量的平均成对距离。

**定义 3.6（$\sigma$ 匹配合并）**。输入 $\mathbf{x}_t$ 与最近帧 $f^*$ 合并的条件：

$$d(\mathbf{x}_t, \mathbf{v}^*) < \delta_{\text{eff}}(F_t) \quad \text{且} \quad \sigma_{\text{input}}[:30] = \sigma^*$$

**$\sigma$ 保护原则**：即使 $d(\mathbf{x}_t, \mathbf{v}^*)$ 非常小，如果 $\sigma_{\text{input}} \neq \sigma^*$，输入仍然创建新帧（而非合并到不相同 $\sigma$ 的帧中）。

**定义 3.7（合并操作）**。

$$\mathbf{v}^* \leftarrow \frac{m^*\mathbf{v}^* + \mathbf{x}_t}{m^* + 1}, \quad m^* \leftarrow m^* + 1$$

**定义 3.8（创建操作）**。若合并条件不满足，创建新帧 $f_{\text{new}} = (\mathbf{x}_t, 1.0, 0, 0, \sigma_{\text{input}}, L_1)$。若 $|F_t| = C$，先淘汰权重最低的帧。

### 3.3 遗忘与剪枝

**定义 3.9（权重衰减）**。对所有 $f_i \in F_t$：

$$w_i \leftarrow w_i \cdot \exp\left(-\frac{\gamma}{m_i + \varepsilon}\right), \quad a_i \leftarrow a_i + 1$$

**定义 3.10（$\sigma$ 淘汰）**。$F_t \leftarrow F_t \setminus \{ f_i \mid w_i < \tau \}$。

> 这是 $\sigma$ 从系统中消失的唯一途径。$\sigma$ 不因外部指令被删除——因经济压力自然消亡。

### 3.4 自指观察

**定义 3.11（自指向量）**。每 $k$ 步（默认 $k = 10$），系统生成自指向量 $\mathbf{o}_t$，并赋予它一个特殊的 $\sigma$：

$$\mathbf{o}_t = \frac{\sum_{f_i \in F_t} w_i \mathbf{v}_i}{\sum w_i}, \quad \sigma_{\text{self}} = \text{"self\_obs"}$$

$\mathbf{o}_t$ 作为普通输入再次经历 3.2-3.3 的流程，但标记为 L4 层。

### 3.5 核心转移函数

```
Algorithm 1: 单步状态转移 T: S_t → S_{t+1}
─────────────────────────────────────────────
 1:  输入: 向量 x_t ∈ ℝ^D, 标签 σ
 2:  ▷ 步骤A: 共现记录
 3:  ℋ_t ← ℋ_t ∪ {(σ_prev, σ, t)}
 4:  ▷ 步骤B: L4 预测（§4.4）
 5:  (σ_pred, conf) ← predict_next(σ, ℋ_t)
 6:  if σ_pred ≠ σ then 生成 "pred_err_σ" 帧
 7:  ▷ 步骤C: σ 竞争分配
 8:  f* ← argmin d(x_t, v_i)
 9:  if d(x_t, v*) < δ_eff ∧ σ[:30] = σ* then 合并
10:  else 创建新帧 (必要时先淘汰低权重帧)
11:  ▷ 步骤D: 遗忘
12:  for all f_i: w_i ← w_i · exp(-γ/m_i)
13:  F_t ← F_t \ {f_i | w_i < τ}
14:  ▷ 步骤E: 自指观察
15:  if t mod k = 0 then
16:      o_t ← 加权质心(F_t), σ = "self_obs"
17:      对 o_t 执行步骤A-D
18:      ▷ 步骤F: L5/L6
19:      更新滚动准确率
20:      if acc < τ then 生成 "sys_doubt" 帧
21:  t ← t + 1
```

## 第四章 六层 $\sigma$ 生产流水线

### 4.1 流水线总览

```
输入 x_t
  │
  ▼  L0: 原始层
  │   σ = σ_input
  │
  ▼  L1: 实体层
  │   σ = "cat", "dog", "ext_0" 或原始名称
  │   σ 被写入 ℋ_t
  │
  ▼  L2: 关联层
  │   从 ℋ_t 读入 σ_i, σ_j → 高频共现
  │   输出 σ = "cat──mat" (ASSOC_SEP)
  │
  ▼  L3: 桥接层
  │   从 L2 σ 的稳定模式 → 输出 σ = "f3══f1" (CHAIN_SEP)
  │
  ═══ σ 自指边界 ═══
  │
  ▼  L4: 预测层
  │   从 ℋ 查 σ → σ_pred, 与实际 σ 对比
  │   输出 σ = "pred_err_cat" 或 "correct_cat"
  │
  ▼  L5: 元观测层
  │   按 pred_ 前缀统计 L4 σ 的准确率
  │   输出: 滚动准确率 acc(t)
  │
  ▼  L6: 统摄层
  │   if acc(t) < τ: 输出 σ = "sys_doubt"
  │   else: 不输出 σ (系统沉默)
  │
  ▼ 输出: sys_doubt (系统对外部世界的唯一信号)
```

### 4.2 各层 $\sigma$ 生产规程

| 层 | 输入 | 运算 | 输出的 $\sigma$ | 被谁消费 |
|----|------|------|----------------|---------|
| L1 | $\mathbf{x}_t$ | 竞争合并 | 输入源名称 | L2 共现统计 |
| L2 | $\mathcal{H}_t$ | 共现频率 > 阈值 | `cat──mat` | L3 稳定模式检测 |
| L3 | L2 $\sigma$ 集合 | 持续共现模式检测 | `f3══f1` | L4 预测统计 |
| L4 | $\mathcal{H}_t + \sigma_{\text{current}}$ | 预测 vs 实际 | `pred_err_*` | L5 准确率统计 |
| L5 | L4 $\sigma$ 流 | 前/后缀分类 | 准确率标量 | L6 统摄判断 |
| L6 | 准确率标量 | 阈值比较 | `sys_doubt` | 外部世界 |

### 4.3 $\sigma$ 保护原则

**一个被上一层消费的 $\sigma$ 不能被下一层合并**。

这意味着如果 L2 输出 `cat──mat`，L1 帧 `cat` 和 `mat` 不会被 L2 的合并操作修改。L2 帧的 $\sigma$ 与 L1 帧的 $\sigma$ 不属于同一个命名空间——L1 的 `cat` 和 L2 的 `cat──mat` 是不同的 $\sigma$，永远不会被合并到一起。

这是跨层结构稳定性的来源。

### 4.4 为什么是六层

六层是最小 $\sigma$ 生产链的产物：

| 层 | 产生什么 $\sigma$ | 不完整性 | 被下一层解决 |
|----|-----------------|---------|------------|
| L1 | 实体名 | 帧之间无关系——$\sigma$ 无结构 | L2 产生关系型 $\sigma$ |
| L2 | 关系 | 关系临时——$\sigma$ 随时间消失 | L3 产生稳定型 $\sigma$ |
| L3 | 模式 | 模式无法判断真假 | L4 产生预测型 $\sigma$ |
| L4 | 预测误差 | 能检测单次误差，不知总体准确率 | L5 产生统计型 $\sigma$ |
| L5 | 准确率趋势 | 能观测趋势但无行动 | L6 产生判断型 $\sigma$ |
| L6 | 判断 | 无 $\sigma$ 需要生产——输出给外部 | — |

在 L6 终止不是因为不能再往上加——而是因为再往上一层需要一个**新的入口**（另一个 GEME 的 L6 输出作为输入），这就超出了单个 GEME 的定义边界，进入了 geuron 网络的范畴。

## 第五章 参数系统

### 5.1 参数层次

```
用户预设（认知预算）
  C = 32    帧容量  —— 同时存在的 σ 最大数量
  W = 50    滑动窗口 —— σ 共现的时间回顾范围
  k = 10    自指间隔 —— 自指向量 σ 的生成频率
  D = 27    向量维度 —— 向量空间大小（不影响 σ 生产质量）
       │
       ▼
核心常数（σ 生产质量约束）
  δ = 0.19  ——— σ 分化的精细度
  γ = 0.05  ——— σ 的更新速度
  τ = 0.60  ——— σ 的淘汰门槛
       │
       ▼
派生阈值（全部 ↓ 推导自 δ/γ/τ）
  θ_dw          → γ × 0.4       = 0.02    : L4 σ 触发边界
  θ_clear        → 1 − 3γ        = 0.85    : L6 σ 解除边界
  λ_decay        → exp(−γ/0.25)  = 0.819   : 未合并帧的 σ 生存率
  ⋯ 共9个（见派生阈值表）
       │
       ▼
内生阈值（自系统 σ 空间统计）
  θ_conf(t)      → Q0.25({conf})           : 预测置信度的自校准门限
```

### 5.2 用户预设参数的依赖边界

$C$（帧容量）与 $\tau$ 耦合：$C < 8$ 时 $\sigma$ 多样性不足，L4 无法建立统计基础。$C > 48$ 时稀缺性消失，低质量 $\sigma$ 未能被淘汰。

$W$（滑动窗口）与 $\gamma$ 耦合：$W/\gamma$ 的最佳区间为 $[200, 2000]$。默认值 $W=50, \gamma=0.05$ 提供 $W/\gamma = 1000$。

$k$（自指间隔）应与帧经济体到达局部均衡的步数匹配。默认 $k = 10$ 在多数配置下位于稳定区间内。

### 5.3 派生阈值总表

| 派生阈值 | 定义 | 数值 | 源自 | 功能域 |
|---------|------|------|------|--------|
| $\theta_{\text{dw}}$ | $\gamma \times 0.4$ | 0.02 | $\gamma$ | L4 $\sigma$ 触发 |
| $\theta_{\text{stable}}$ | $\gamma \times 0.2$ | 0.01 | $\gamma$ | $\sigma$ 稳定判定 |
| $\theta_{\text{doubt}}$ | $\tau$ | 0.60 | $\tau$ | L6 $\sigma$ 激活 |
| $\theta_{\text{clear}}$ | $1 - 3\gamma$ | 0.85 | $\gamma$ | L6 $\sigma$ 解除 |
| $\theta_{\text{healthy}}$ | $1 - 4\gamma$ | 0.80 | $\gamma$ | 系统健康基线 |
| $\lambda_{\text{unmerged}}$ | $\exp(-\gamma/0.25)$ | 0.819 | $\gamma$ | 未合并 $\sigma$ 生存率 |
| $\lambda_{\text{merged}}$ | $\exp(-\gamma)$ | 0.951 | $\gamma$ | 已合并 $\sigma$ 生存率 |

> 全部派生阈值追踪至 $\delta, \gamma, \tau$。此为工程设计原则——非模型参数。
