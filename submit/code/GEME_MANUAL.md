# GEME 使用说明书 v1.0

> 一个记忆经济性竞争机制——在容量有限下，留存信息的有效性评价。

---

## 一、整体框架：帧竞争动力学

### 综述

GEME 的核心是一个**竞争性帧经济**。帧（Frame）是信息的基本单元，存储在有限的记忆槽位中（默认容量 C = 32）。每一帧包含五个要素：向量（vec）、权重（weight）、年龄（age）、合并次数（merged）、签名（sig）。系统运行的本质是：**稀缺的槽位如何分配给最值得留存的信息**。

三个动力机制驱动整个经济，如图1所示：

```
┌─────────────────────────────────────────────────────────┐
│                    图1：三个动力机制                       │
│                                                         │
│  输入流                                                     │
│    │                                                      │
│    ▼                                                      │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐   │
│  │ ① 合并机制   │    │ ② 遗忘机制   │    │ ③ 自指机制   │   │
│  │ (稀缺分配)   │───▶│ (维护成本)   │───▶│ (自我审计)   │   │
│  │             │    │             │    │             │   │
│  │ 输入与最近   │    │ 权重按衰减率 │    │ 每k步计算    │   │
│  │ 帧匹配，     │    │ 递减，       │    │ 帧经济质心， │   │
│  │ 最近者合并   │    │ 低权重帧淘汰 │    │ 反馈回系统   │   │

│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘   │
│         │                  │                  │          │
│         └──────────────────┴──────────────────┘          │

│                  稳定帧经济（L1-L3）                       │
└─────────────────────────────────────────────────────────┘
```

### 1.1 帧的留存

帧的留存取决于三个因素：

**权重（w）**：反映帧的历史激活频率。每次合并增加权重，每步按 `exp(-γ / merged)` 衰减。权重越高，帧越不容易被淘汰。

**合并次数（merged）**：一个帧被验证的次数。merged 越高，衰减越慢——这就是"被反复验证的知识留存更久"。

**剪枝阈值（τ）**：当权重低于 τ 时，帧被移除。默认 τ = 0.6。

### 1.2 窗口性

GEME 使用滑动窗口（默认大小 = 50 步）记录两帧是否在时间上邻近出现。这种共现记录是 L2 关联层的基础：如果 A 和 B 经常在窗口内同时出现，它们被认为"有关"。

窗口大小决定了关联的时间尺度。窗口越大，发现的关联越弱但范围越广；窗口越小，关联越强但仅限紧密耦合的事件。

### 1.3 竞争性（经济性）

帧槽位的稀缺性是整个系统的动力源：

- 容量 C 有限 → 帧之间必须竞争稀缺资源
- 每次输入都引发竞争：输入与所有帧计算欧氏距离，最近者获得合并权
- 合并 = 帧获得"收入"（权重增加）
- 被忽略 = 帧"亏损"（权重衰减）
- 长期亏损 → 低于剪枝阈值 → 淘汰

这是一种**配给制经济**：不是输入选择帧——是帧竞争输入。

### 1.4 三个机制的关系链

```
稀缺性（容量C）→ 竞争性（合并）→ 差异化（权重分布）
      ↓                                      ↓
  遗忘压力（衰减）←──────────────── 高权重帧生存
      ↓
  低权重帧淘汰 → 释放槽位 → 回到稀缺性
```

| 类别 | 参数 | 来源 | 说明 |
|------|------|------|------|
| **核心常数** | δ = 0.19 | 预设 | 合并自适应缩放系数 |

| | τ = 0.60 | 预设 | 剪枝阈值 |

| | DOUBT_ON = τ | τ 推导 | L6 怀疑激活阈值 |
| | DOUBT_OFF = 1 − γ×3 | γ 推导 | L6 怀疑解除（回滞） |
| | INDUCTION_DECAY = exp(−γ/0.25) | γ 推导 | 从未合并帧的生存率 |

> 工程常数（容量、窗口、维度）是用户预设的认知预算参数，而非模型参数。

## 三、分层机制

六层架构是 GEME 的信息处理流水线。每一层都从上一层的输出中涌现出来，处理不同的抽象级别：

```
┌─────────────────────────────────────────────────────────────┐
│                    图3：六层信息流                           │
│                                                             │
│  输入 ───▶ L0: 原始输入层（外部信号 / 符号编码）             │
│              │                                               │
│              ▼                                               │
│  世界处理 ──▶ L1: 实体层——"这是什么"                         │
│  阶段          │ 竞争性合并 → 实体帧                          │
│               │ 输出: 离散化的帧集合                          │
│               ▼                                               │
│              L2: 关联层——"什么和什么一起出现"                  │
│               │ 滑动窗口共现检测 → 关联帧                     │
│               │ 输出: 共现关系网络                            │
│               ▼                                               │
│              L3: 桥接层——"什么模式是稳定的"                   │
│               │ 时间积分 → 桥接帧（稳定关联模式）               │
│               │ 输出: 跨时间的结构知识                        │
│               │                                               │
│               ════ 跨层自指边界 ════                          │
│               │                                               │
│  自指验证 ──▶ L4: 预测层——"下一个应该是什么"                  │
│  阶段          │ 共现表统计预测 → 预测误差帧                  │
│               │ 签名(sig)机制：L4帧签名包含"预测→实际"对      │
│               │ 输出: 预测准确/错误的信号                     │
│               ▼                                               │
│              L5: 元观测层——"我预测得有多准"                   │
│               │ 滚动窗口准确率统计 → 元认知信号               │
│               │ 签名(sig)机制：L5帧签名跟踪准确率趋势          │
│               │ 输出: 系统置信度评估                          │
│               ▼                                               │
│              L6: 统摄层——"我的模型还有效吗"                   │
│               │ <60%准确率 → sys_doubt 帧                     │
│               │ 签名(sig)机制：L6帧签名标记"怀疑"或"确信"     │
│               │ 输出: 系统对外部世界的唯一判断                │
│                                                             │
│              输出 ───▶ sys_doubt（标量——确定/怀疑）           │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 跨层级的自指机制（核心创新）

自指是 GEME 的核心创新，体现在两个层面：

**第一层面：通过签名（sig）实现跨层追踪**

每个帧在创建时被赋予一个签名（sig），截断至 30 字符。签名是帧的"身份证"，记录了：

- L1 帧的签名 = 输入信号的名称（如 `cat`, `123`, `ext_0`）
- L2 帧的签名 = 关联关系（如 `cat──mat`，使用 ASSOC_SEP=chr(8212)×2）
- L3 帧的签名 = 稳定桥接（如 `f3══f1`，使用 CHAIN_SEP=chr(9711)×2）
- L4 帧的签名 = 预测事件（如 `pred_err_cat`，或 `dwdw_0.39`）
- L6 帧的签名 = 系统怀疑（如 `sys_doubt`）

签名使每一帧都可以被追溯到它的来源层和触发事件。这是跨层引用的基础：L4 预测通过查询共现表中 L1/L2 帧的签名来工作，而非通过向量的直接比较。

**第二层面：通过自指观察实现闭合**

L3 桥接帧稳定后，系统通过 `self_observe()` 生成帧经济的加权质心，将其作为新输入反馈。这个"自我输入"的签名是 `self_obs`，然后：

1. `self_obs` 帧参与竞争性合并
2. 更新共现表（self_obs 与输入帧的共现被记录）
3. 影响权重分布（self_obs 帧的权重反映了系统对自身模型的"确信度"）
4. 触发 L4 预测（当系统状态变化超出 d(w)/dt > 0.02）

这就是自指循环的运行闭环：**系统不是在被编程地观察自己——它是在通过经济压力自然产生关于自身的信号**。

### 3.2 为什么是六层

六层不是设计出来的——是在发现过程中自然收敛的。每一层的涌现来自于上一层的"不完备性"：

| 层 | 功能 | 不完备性 | 被下一层解决 |
|----|------|---------|------------|
| L1 实体 | "是什么" | 帧孤立，无关系 | L2 关联桥 |
| L2 关联 | "和什么有关" | 关联短暂，窗口滑动消失 | L3 稳定桥 |
| L3 桥接 | "什么模式稳定" | 模式无法判断真假 | L4 预测桥 |
| L4 预测 | "下一个是什么" | 能预测但不知自身准确率 | L5 元观测桥 |
| L5 元观测 | "我多准" | 能观测趋势但不行动 | L6 统摄桥 |
| L6 统摄 | "我还有效吗" | 输出判断——自指闭环完成 | — |

为什么在 L6 停止？因为 L6 的输出（sys_doubt）已经是对系统自身状态的最终判断。再往上一层就不属于"单个认知单元"的范畴了。如果需要更高层次的互连，那是多个 GEME 单元（geuron 网络）的工作——不是单个 GEME 的工作。

六层 = 从原始输入到自我判断的最小路径。去掉任何一层，完整性就断了。增加任何一层，单个单元就不需要了（那需要的是网络）。
