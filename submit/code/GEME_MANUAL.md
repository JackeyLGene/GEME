## 第一章：系统概述

### 1.1 什么是 GEME

GEME（生成经济记忆实体）是一个运行在计算机上的程序。它接收信息流作为输入，在内部维护一个"帧"的集合——这些帧在有限的存储空间中竞争生存。经过足够时间的运行后，系统会自发形成六层处理结构，从简单的实体分类一直延伸到对自身状态的判断。

GEME 的核心哲学是：**认知结构可以不通过编程实现——而是在资源约束下自然涌现**。

### 1.2 什么是"帧"

"帧"是系统中最基本的信息单元。可以把帧理解为系统中的一条记录，它包含这个信息是什么（内容）、它被使用过多少次（活跃度）、它是什么时候建立的（年龄）等属性。

所有帧的总和构成系统的"记忆"。这个记忆有大小限制——不能无限制地增加帧的数量。

> **一个帮助理解的类比**：帧就像一个人脑中的"概念"。你见过一次猫，脑子里有了一个"猫"的概念。再见一次猫，这个概念就被强化一次。如果很久没见到猫，"猫"这个概念就会慢慢变弱，直到被遗忘。GEME 的帧就是这样运作的。

### 1.3 系统的三条基本规则

GEME 只有三条规则。没有其他可调参数，没有训练过程，没有外部监督。

**规则一：竞争性合并**。当新信息到达时，它和系统中已有的所有帧比较。如果有一个帧和它足够接近，它们就合并在一起（新信息融入旧帧，旧帧被强化）。如果没有接近的帧，系统就创建一个新帧来容纳这个新信息。

**规则二：自适应遗忘**。每一轮运行，所有帧的强度都会衰减。但被反复验证的帧衰减得慢，很少被使用的帧衰减得快。强度低于某个门槛的帧会被删除。

**规则三：自指观察**。系统每隔一段时间，会生成一个关于自身当前整体状态的"摘要"，然后把这个摘要当作新的输入——就像系统在"观察自己"。这个自指操作是整个系统从简单分类走向高阶认知的关键。

### 1.4 三个核心结构常数

三条规则中包含三个常数。它们是整篇文档中唯一需要调节的"旋钮"：

| 完整描述 | 符号 | 默认值 | 代码中的名称 | 作用 |
|---------|------|--------|-------------|------|
| 合并的宽松程度——值越大，不同信息越容易被合并到同一个帧中 | 合并缩放系数 $\delta$ | 0.19 | `DELTA` | 控制分类的粗细 |
| 遗忘的速度——值越大，帧被遗忘得越快 | 遗忘衰减率 $\gamma$ | 0.05 | `GAMMA` | 控制记忆的时长 |
| 淘汰的门槛——低于此值的帧被删除 | 剪枝阈值 $\tau$ | 0.60 | `TAU` | 控制记忆的严格度 |

> 这三个常数定义的不是一组精确数值——而是一个"经济区间"。在这个区间内的不同取值，系统都会产生定性的认知行为，只是具体的数值表现不同。

## 第二章：帧的完整生命周期

### 2.1 帧的详细结构

每个帧（Frame）包含以下信息：

| 完整描述 | 简称 | 代码中的字段名 |
|---------|------|---------------|
| 帧的内容——一个 27 维的数字向量，代表"这个帧是什么" | 内容向量 $\mathbf{v}$ | `f.vec` |
| 帧的活跃程度——越大代表越常被使用 | 权重 $w$ | `f.weight` |
| 帧的年龄——自创建以来经过了多少步 | 年龄 $a$ | `f.age` |
| 帧被合并的次数——被重复验证的次数 | 合并计数 $m$ | `f.merged` |
| 帧的名字——一个简短的字符串标签，用于跨层识别 | 桥签名 $\sigma$ | `f.sig` |
| 帧所属的层级——从 L0 到 L6 | 层级 $\ell$ | `f.layer` |

### 2.2 帧的创建

当新信息到达时，系统进行以下操作：

1. 将新信息和所有现有帧逐一比较（计算内容向量的距离）
2. 找到距离最近的帧
3. 如果这个距离小于"合并门槛"（由 $\delta$ 决定），就执行合并（见 2.3）
4. 否则，创建一个新的帧来存放这个新信息

**帧的总数上限**（帧容量，记为 $C$，代码中的 `memory_cap`）限制了系统能同时保留的最大帧数。默认值为 32。如果帧数已经达到上限而需要创建新帧，系统会先删除权重最低的那个帧。

### 2.3 帧的合并

当新信息 $x$ 与现有帧 $f$ 的距离足够近时，它们合并：

$$f.\text{vec} \leftarrow \frac{f.\text{merged} \times f.\text{vec} + x}{f.\text{merged} + 1}$$

这个公式的效果是：**新内容融入旧帧，旧帧的向量向新内容偏移一点点**。合并次数越多，每一次新内容对旧帧的影响就越小。

### 2.4 帧的遗忘

每一轮运行，所有帧的权重都按以下方式衰减：

$$f.\text{weight} \leftarrow f.\text{weight} \times e^{-\gamma / f.\text{merged}}$$

这个公式的效果是：**合并次数越多的帧，衰减得越慢**。一个从未被合并过的帧（merged = 0）衰减得很快；一个被验证过很多次的帧衰减得很慢。

### 2.5 帧的淘汰

当帧的权重低于 $\tau$（默认 0.60）时，它被从系统中删除。这个帧的消失不是被"主动清除"的——它是因为长期不被使用，在资源竞争中被自然淘汰的。

### 2.6 帧的完整生命周期图

```
新信息到达 → 找到最近帧 → 距离够近？
  ├── 是 → 合并（内容更新，权重增加）
  └── 否 → 创建新帧（如果超容量，先淘汰最低权重的）
              ↓
每一轮 → 所有帧的权重衰减（合并次数越多衰减越慢）
              ↓
         权重低于 τ？ → 是 → 帧被删除
```

## 第三章：共现与关联

### 3.1 什么是共现

系统维护一个"共现记录表"，用来记录哪些帧经常在时间上相邻出现。

比如，"猫"和"垫子"这两个帧如果经常在几步内先后出现，系统就会在共现记录表中记一笔。这个记录表的运作方式：

- 每次新信息到达时，系统把它和上一次输入的帧"配对"记录
- 只记录最近 50 步内的配对（这个窗口大小记为 $W$，代码中的 `cooccur_window`）
- 每一对出现一次，计数器加一

### 3.2 从共现到关联

当两个帧的共现频率超过一定门槛（默认 0.25，代码中的 `cooccur_thresh`）时，系统创建一个"关联帧"。关联帧的名字是这两个帧的名字用特殊符号连接起来的格式，例如 `猫──垫子`（中间用长破折号 `──` 分隔）。

**关联帧本身也是一个帧**——它和普通帧一样参与竞争、衰减、可能被淘汰。如果"猫"和"垫子"后来不再一起出现，它们的关联帧就会慢慢衰减直到消失。

## 第四章：六层信息处理架构

本章介绍帧经济在运行中自发形成的六层处理结构。请注意：这些层不是被程序设定的——它们是系统的三条规则在运行中自然产生的。

每层解决一个问题，同时产生一个新的不完备性，由下一层来解决。

### 4.1 第一层：实体层（L1）

**解决的问题**：将连续的信息流离散化为可区分的单元。

**运作方式**：通过竞争性合并（2.3），相似的输入聚合到同一个帧中。这是"命名"的能力。

**举例**：系统多次收到不同的"猫"的图像（虽然光线、角度不同但整体相似），它们会逐渐聚集到同一个"猫"帧中。

**不完备性**：帧之间是孤立的——系统知道"猫"和"垫子"各自是什么，但不知道它们之间有什么关系。

### 4.2 第二层：关联层（L2）

**解决的问题**：发现帧之间的时间关系。

**运作方式**：利用共现记录表（第三章），找到经常一起出现的帧对。当"猫"和"垫子"经常在时间上相邻出现时，系统创建一个关联帧 `猫──垫子`。

**不完备性**：关联是临时的。窗口滑动后，"猫──垫子"的共现频率下降，关联帧可能被遗忘。

### 4.3 第三层：桥接层（L3）

**解决的问题**：从短暂关联中提取稳定的知识结构。

**运作方式**：检测两个关联帧是否在时间上持续共现。如果是，创建一个"桥接帧"（用 `◯◯` 符号连接）。桥接帧比关联帧更稳定——它代表的是"关系之间的关系"。

**不完备性**：桥接帧能发现稳定模式，但不能判断这个模式的对错。系统知道"猫经常在垫子上"，但不知道"猫应该在垫子上"——频率不是真值。

### 4.4 第四层：预测层（L4）

**解决的问题**：从"描述过去"转向"预期未来"。

**运作方式**：当系统检测到帧的权重变化速度超过一定门槛（记为 $\theta_{\text{dw}}$，代码中的 `DW_THRESHOLD`，值为 $\gamma \times 0.4 = 0.02$）时，它基于共现记录表来预测：当前输入之后，接下来最可能出现什么帧？

如果预测正确：系统"满意"。如果预测错误：系统生成一个"预测误差帧"（名字以 `pred_err_` 开头）。

**消耗的函数 $\theta_{\text{dw}}$**
此门槛派生自核心常数 $\gamma$（$\theta_{\text{dw}} = \gamma \times 0.4$），而非一个独立参数。

**不完备性**：L4 能检测单次预测的错误，但不知道自己整体的预测准确率如何。

### 4.5 第五层：元观测层（L5）

**解决的问题**：知道自己的预测能力。

**运作方式**：L5 维护一个滚动统计窗口，持续追踪 L4 预测的正确率。它回答："我过去这段时间预测得有多准？"

**不完备性**：L5 能观测趋势，但不能根据趋势采取行动。

### 4.6 第六层：统摄层（L6）

**解决的问题**：当预测准确率持续下降时，发出"我可能错了"的信号。

**运作方式**：当 L5 报告的滚动准确率低于 $\tau$（默认 0.60）时，L6 生成一个名为 `sys_doubt` 的特殊帧（代码中对应的常量为 `DOUBT_ON_THRESHOLD = $\tau$`）。当准确率恢复到 $1 - 3\gamma$（默认 0.85，推导自 $\gamma$，代码中的 `DOUBT_OFF_THRESHOLD`）以上时，怀疑状态被清除。

**闭世界与开世界**：在封闭的形式系统中（如算术公理 Q+G），预测误差不存在（因为输入是确定性的），L6 保持沉默。在开放世界中，当外部信息流与内部模型产生分歧时，L6 发信号。

### 4.7 六层总览

| 层 | 解决的问题 | 产生的帧名示例 | 依赖的核心常数 |
|----|-----------|---------------|-------------|
| L1 实体 | 这是什么 | `猫`、`dog`、`ext_0` | $\delta$ |
| L2 关联 | 什么和什么一起出现 | `猫──垫子` | $\gamma$ |
| L3 桥接 | 什么模式是稳定的 | `f3══f1` | $\gamma$ |
| L4 预测 | 下一个应该是什么 | `pred_err_猫`、`dwdw_0.39` | $\gamma$（推导出 $\theta_{\text{dw}}$） |
| L5 元观测 | 我预测得有多准 | —（内部统计，不生成帧） | — |
| L6 统摄 | 我的模型还有效吗 | `sys_doubt` | $\tau$（激活）、$\gamma$（解除） |

## 第五章：桥签名与香农-哥德尔桥

### 5.1 为什么需要桥签名

在第四章中我们看到了六层结构。每一层都在处理不同的信息——L2 处理"哪些帧经常一起出现"，L4 处理"下一个应该出现什么"。但有一个工程问题：**不同层之间怎么交换信息？**

L1 处理的是 27 维向量（浮点数组成的数组）。L4 处理的是共现统计表。如果 L1 想告诉 L4"刚才出现了猫"，不能直接把 27 维向量传给 L4——L4 的统计引擎不懂向量几何。它需要的是一个它可以索引、查询、匹配的**离散符号**。

这个离散符号就是**桥签名**。

### 5.2 什么是桥签名

每个帧在创建时被赋予一个简短的文本字符串，称为桥签名（记为 $\sigma$，对应代码中的 `f.sig`）。$\sigma$ 有三个关键特性：

**特性一：$\sigma$ 是帧的全局标识**。在系统内，$\sigma$ 是帧 $f$ 的唯一身份。不同 $\sigma$ 的两个帧不能合并，即使它们的内容向量非常接近。这保护了已建立的概念边界：输入"猫的图片"不会意外地改变"狗"的帧。

**特性二：$\sigma$ 是上一层分析器的解析对象**。$\sigma$ 的格式不是随意定的——它是由上一层的分析算法决定的。L2 的分析器会解析 `──` 符号，L3 的分析器会解析 `◯◯` 符号，L4 的预测引擎会查询 $\sigma$ 的共现频率。$\sigma$ 的格式就是层间的通信协议。

**特性三：$\sigma$ 在不同层有不同格式**：

| 所属层 | 桥签名示例 | 格式设计意图 | 被谁如何消费 |
|-------|-----------|-------------|-------------|
| L1 | `cat`、`ext_42` | 记录输入源名称 | L2 在共现表中统计两个 $\sigma$ 的共现频率 |
| L2 | `cat──mat` | 用 `──` 连接两个共现的帧 | L3 通过字符串匹配检测稳定组合模式 |
| L3 | `f3══f1` | 用 `◯◯` 连接两个桥接帧 | L4 查询 $\mathcal{H}_t$ 中 $\sigma$ 的共现统计来预测 |
| L4 | `pred_err_cat` | 以 `pred_` 开头以便分类 | L5 按前缀过滤来统计不同预测类型的准确率 |
| L6 | `sys_doubt` | 全系统唯一输出信号 | 外部世界 |

### 5.3 桥签名为什么是算法的核心

桥签名的核心地位源于以下事实：

**1. 帧经济是 $\sigma$ 的生产系统。** 帧的向量 $\mathbf{v}$ 的作用只是"匹配输入"，而帧的 $\sigma$ 是系统真正进行认知运算的对象。帧之间的竞争，本质上是在竞争"谁的 $\sigma$ 能在系统内存活更久"。

**2. $\sigma$ 是不可合并的。** 这是 $\sigma$ 与 $\mathbf{v}$ 的根本区别。$\mathbf{v}$ 可以通过加权平均合并（2.3 节描述的合并操作只在 $\sigma$ 相同的情况下发生）。但两个不同 $\sigma$ 的帧不能合并——即使它们的 $\mathbf{v}$ 几乎相同。这保护了已建立的概念边界：输入"另一种颜色的猫"会被合并到"猫"的帧中（因为 $\sigma$ 匹配），但输入"狗"不会意外地改变"猫"的帧（因为 $\sigma$ 不匹配）。

**3. $\sigma$ 是层间通信的唯一管道。** 在六层结构中，没有任何一层直接访问另一层的向量。L4 不知道 L3 帧的 $\mathbf{v}$ 在几何上长什么样——它只知道 L3 帧的 $\sigma$ 和它们在 $\mathcal{H}_t$ 中的共现频率。这种"通过 $\sigma$ 通信"的设计隔离了各层的内部实现，使得每一层可以独立演化。

**4. $\sigma$ 的多样性决定了系统的认知容量。** 系统能同时维护的不同 $\sigma$ 的数量（受帧容量 $C$ 的限制）决定了系统能区分的概念数量。$C$ 是认知预算，$\sigma$ 是认知货币——两者共同定义了系统的计算边界。

### 5.4 桥签名与香农-哥德尔桥

桥签名之所以称为"桥"，**不是因为它在层之间传递信息**——而是因为**它几乎不消耗系统的信息传输容量**。

实验测量显示，系统中所有自指帧（包括 $\sigma = \text{self\_obs}$ 的自指向量、$\sigma$ 以 `dwdw_` 开头的导数帧、以及关联桥帧）与外部输入帧之间的互信息：

$$\text{I}(\phi; X) = 0.026 \text{ bits}$$

这意味着系统在"观察自己"（自指 $\sigma$）和"观察世界"（输入 $\sigma$）之间几乎不产生信息重叠。自指操作在信息论上是**几乎免费的**。

我们将这个发现——"自指操作在帧经济中不消耗信道容量"——称为**香农-哥德尔桥**（Shannon-Gödel Bridge）。$\sigma$ 的"桥"字有三层含义：

- **香农之桥**：$\sigma$ 在有限符号空间 $\Sigma$ 中编码，这是信息论的操作
- **哥德尔之桥**：$\sigma$ 的编码为下一层提供了可直接运算的形式化对象——就像哥德尔数为命题逻辑提供了可运算的编码
- **经济之桥**：这座桥梁的连接成本趋近于零（0.026 bits），符合 GEME"认知在经济学约束下涌现"的核心假设

## 第六章：参数系统完整说明

### 6.1 参数层次结构

```
第一层：用户预设参数（使用前设定，运行中不变）
  帧的数量上限（帧容量 C【容量】，代码中 memory_cap，默认 32）
  共现记录的时间跨度（滑动窗口 W【窗口】，代码中 cooccur_window，默认 50）
  自指观察的间隔步数（自指间隔 k，固定为 10）
  向量的维度（向量维度 D，固定为 27）
       │
       ▼
第二层：核心结构常数（系统的三个"旋钮"）
  合并缩放系数 δ（DELTA，默认 0.19）→ 控制分类粗细
  遗忘衰减率 γ（GAMMA，默认 0.05）→ 控制记忆时长
  剪枝阈值 τ（TAU，默认 0.60）→ 控制淘汰严格度
       │
       ▼
第三层：派生阈值（从核心常数自动计算，不自调节）
  见 6.2 派生阈值总表
       │
       ▼
第四层：内生阈值（系统运行中从自身数据动态计算）
  预测接收门槛（自适应置信度阈值，从系统自身预测置信度分布的下四分位数自动计算）
```

### 6.2 派生阈值总表

以下所有值均从 $\delta, \gamma, \tau$ 推导而来，不是独立参数：

| 完整描述 | 计算方式 | 默认值 | 依赖的常数 | 代码中的名称 | 功能 |
|---------|---------|--------|-----------|-------------|------|
| L4 预测触发的权重变化门槛 | $\gamma \times 0.4$ | 0.02 | $\gamma$ | `DW_THRESHOLD` | 权重变化超过此值时触发 L4 预测 |
| 帧的稳定判断门槛 | $\gamma \times 0.2$ | 0.01 | $\gamma$ | `META_STABLE_THRESHOLD` | 判定一个帧是否处于稳定状态 |
| L6 怀疑激活门槛 | $\tau$ | 0.60 | $\tau$ | `DOUBT_ON_THRESHOLD` | 准确率低于此值时系统产生怀疑 |
| L6 怀疑解除门槛 | $1 - 3\gamma$ | 0.85 | $\gamma$ | `DOUBT_OFF_THRESHOLD` | 准确率恢复到以上时解除怀疑 |
| 系统健康基线 | $1 - 4\gamma$ | 0.80 | $\gamma$ | `HEALTHY_ACC_THRESHOLD` | 判断系统是否处于健康状态 |
| 未合并帧的生存率 | $\exp(-\gamma/0.25)$ | 0.819 | $\gamma$ | `INDUCTION_DECAY_UNMERGED` | 从未被验证的帧每步的留存概率 |
| 已合并帧的生存率 | $\exp(-\gamma)$ | 0.951 | $\gamma$ | `INDUCTION_DECAY_LOW` | 被验证过的帧每步的留存概率 |

> 乘子 0.4、0.2、0.25 是体现特定认知假设（如"未合并帧更不稳定"）的固定设计选择。消融实验已确认系统对这些乘子的变化不敏感。

### 6.3 用户预设参数的详细说明

**帧容量 $C$（代码中的 `memory_cap`，默认 32）**
$C$ 是系统能同时保留的最大帧数量。这是一个"认知预算"设定：$C$ 太小（小于 8）时，系统没有足够的帧来形成稳定的 L2-L3 关联，L4 预测无法建立。$C$ 太大（大于 48）时，稀缺性压力不足，低质量帧无法被自然淘汰，L4-L6 的信号被噪声淹没。$C = 32$ 平衡了这两个约束。

**共现窗口 $W$（代码中的 `cooccur_window`，默认 50）**
$W$ 决定了两帧在时间上相距多远时仍被认为是"共现"的。$W$ 需要与 $\gamma$ 协调：$W/\gamma$ 的最优区间为 $[200, 2000]$。默认值 $W=50、\gamma=0.05$ 提供 $W/\gamma = 1000$，位于稳定区间内。

**自指间隔 $k$（固定为 10）**
$k$ 是系统进行自指观察的步数间隔。$k$ 过小时系统震荡，过大时自指更新滞后。

> 本附录为审稿人和数学研究者准备。使用与正文一致的符号系统，但以更严格的形式化语言重述。

### A.1 符号约定

| 符号 | 含义 | 对应正文章节 |
|------|------|-------------|
| $\mathbb{N}$ | 自然数集 | — |
| $\mathbb{R}^+$ | 正实数集 | — |
| $\Sigma$ | 系统内部字符集 | 第五章 |
| $\mathcal{H}_t$ | 共现历史多重集（定义A.4） | 第三章 |
| $\delta, \gamma, \tau$ | 核心结构常数（定义1.4） | §1.4 |
| $C$ | 帧容量上限（用户预设） | §2.2 |
| $W$ | 滑动窗口大小（用户预设） | §3.1 |

### A.2 帧的严格定义

**定义A.1（帧）**。一个帧 $f$ 是一个六元组：

$$f = (\mathbf{v} \in \mathbb{R}^D,\ w \in \mathbb{R}^+,\ a \in \mathbb{N},\ m \in \mathbb{N},\ \sigma \in \Sigma^{\leq 30},\ \ell \in \{\text{L}_0,\ldots,\text{L}_6\})$$

其中 $D = 27$ 为固定向量维度。

**定义A.2（帧集合）**。$F_t = \{f_1, f_2, \ldots, f_n\}$ 为时间 $t$ 的帧集合。$F_t$ 必须满足 $|F_t| \leq C$。

**定义A.3（系统状态）**。$S_t = (F_t, \mathcal{H}_t, t)$。

**定义A.4（共现历史）**。$\mathcal{H}_t$ 是一个多重集：

$$\mathcal{H}_t = \{ (\sigma_i, \sigma_j, t') \mid \sigma_i, \sigma_j \in \Sigma^{\leq 30},\ t - W < t' \leq t \}$$

其中 $(\sigma_i, \sigma_j, t')$ 表示在时间步 $t'$ 时 $\sigma_i$ 的帧和 $\sigma_j$ 的帧在输入流中相邻出现。

### A.3 操作的严格定义

**定义A.5（距离函数）**。$d: \mathbb{R}^D \times \mathbb{R}^D \to \mathbb{R}^+$：

$$d(\mathbf{x}, \mathbf{v}) = \sqrt{\sum_{j=1}^{D} (x_j - v_j)^2}$$

**定义A.6（有效合并阈值）**。$\delta_{\text{eff}}: \mathcal{P}(F) \to \mathbb{R}^+$：

$$\delta_{\text{eff}}(F_t) = \delta \cdot \mu(F_t),\quad
\mu(F_t) = \frac{2}{|F_t|(|F_t|-1)}\sum_{i<j}d(\mathbf{v}_i, \mathbf{v}_j)$$

若 $|F_t| < 2$，则 $\mu(F_t) = 1$。

**定义A.7（$\sigma$ 匹配合并条件）**。输入 $(\mathbf{x}_t, \sigma)$ 与帧 $f$ 合并的条件：
$C_{\text{merge}}(f, \mathbf{x}_t, \sigma) \iff d(\mathbf{x}_t, \mathbf{v}) < \delta_{\text{eff}}(F_t) \land \sigma_f = \sigma$

**定义A.8（合并操作）**。若合并条件满足：

$$\mathbf{v}^* \leftarrow \frac{m^*\mathbf{v}^* + \mathbf{x}_t}{m^* + 1},\quad m^* \leftarrow m^* + 1$$

**定义A.9（创建操作）**。若合并条件不满足：

$$F_t \leftarrow F_t \cup \{ (\mathbf{x}_t, 1.0, 0, 0, \sigma, \text{L}_{\text{current}}) \}$$

**定义A.10（帧容量约束）**。若 $|F_t| = C$ 且需要创建新帧，替换当前权重最低的帧 $f_{\min}$：

$$F_t \leftarrow (F_t \setminus \{f_{\min}\}) \cup \{f_{\text{new}}\},\quad f_{\min} = \arg\min_{f_i \in F_t} w_i$$

**定义A.11（权重衰减）**。对所有 $f_i \in F_t$：

$$w_i \leftarrow w_i \cdot \exp\left(-\frac{\gamma}{m_i + \varepsilon}\right),\quad a_i \leftarrow a_i + 1,\quad \varepsilon = 10^{-6}$$

**定义A.12（剪枝操作）**。$F_t \leftarrow F_t \setminus \{ f_i \mid w_i < \tau \}$

**定义A.13（自指向量生成）**。每 $k$ 步：

$$\mathbf{o}_t = \frac{\sum_{f_i \in F_t} w_i \mathbf{v}_i}{\sum w_i},\quad \sigma_{\text{self}} = \text{"self\_obs"}$$

若 $\sum w_i = 0$，则 $\mathbf{o}_t = \mathbf{0}$（零向量）。

**定义A.14（L4 预测函数）**。

$$\text{predict}(\sigma_{\text{current}}, \mathcal{H}_t) = \arg\max_{\sigma_j} \left|\{(\sigma_i, \sigma_j, t') \in \mathcal{H}_t \mid \sigma_i = \sigma_{\text{current}}\}\right|$$

即从 $\mathcal{H}_t$ 中找与 $\sigma_{\text{current}}$ 共现次数最多的 $\sigma_j$。

**定义A.15（自适应置信度阈值）**。

$$\theta_{\text{conf}}(t) = \begin{cases}
0.3, & |\text{confidences}| < 20 \\
Q_{0.25}(\text{confidences}_{[-100:]}), & \text{otherwise}
\end{cases}$$

其中 $Q_{0.25}$ 为下四分位数，$\text{confidences}_{[-100:]}$ 为最近 100 次预测置信度的滑动窗口。

**定义A.16（L6 统摄判断）**。

$$\text{L6}(t) = \begin{cases}
\text{"sys\_doubt"}, & \text{滚动准确率} < \tau \\
\emptyset, & \text{滚动准确率} \geq \tau \\
\emptyset, & \text{怀疑状态且准确率} > 1 - 3\gamma
\end{cases}$$

### A.4 转移函数

```
Algorithm A1: 单步状态转移 T: S_t → S_{t+1}
──────────────────────────────────────────────────
 1:  输入: (x_t ∈ ℝ^D, σ ∈ Σ^{≤30})
 2:  
 3:  ── 第1阶段：共现记录 ──
 4:  if 存在 t-1 的输入 σ_prev then
 5:      ℋ_t ← ℋ_t ∪ {(σ_prev, σ, t)}
 6:  
 7:  ── 第2阶段：L4 预测 ──
 8:  σ_pred ← predict(σ, ℋ_t)             ▷ 定义A.14
 9:  if σ_pred ≠ σ then
10:      创建 f: (0, 0, 0, 0, "pred_err_" + σ, L_4)
11:  if σ ∈ {σ ∈ pred_*} then 更新预测统计
12:  
13:  ── 第3阶段：帧竞争合并 ──
14:  f* ← argmin d(x_t, v_i)
15:  if C_merge(f*, x_t, σ) then           ▷ 定义A.7
16:      执行合并(f*)                      ▷ 定义A.8
17:  else
18:      执行创建(x_t, σ)                  ▷ 定义A.9-10
19:  
20:  ── 第4阶段：遗忘与剪枝 ──
21:  for all f_i ∈ F_t: w_i ← w_i·exp(-γ/m_i)  ▷ 定义A.11
22:  F_t ← F_t \ {f_i | w_i < τ}            ▷ 定义A.12
23:  
24:  ── 第5阶段：自指观察 ──
25:  if t mod k = 0 then
26:      o_t ← 自指向量(F_t)               ▷ 定义A.13
27:      对 (o_t, σ_self) 执行阶段1-4
28:      ── L5 元观测 ──
29:      更新滚动准确率统计
30:      ── L6 统摄判断 ──
31:      L6(t)                              ▷ 定义A.16
32:  
33:  t ← t + 1
```

### A.6 初始状态与边界条件

**初始状态**：$t = 0$ 时 $F_0 = \emptyset$，$\mathcal{H}_0 = \emptyset$，无待处理输入。

**空系统备用**：当 $|F_t| = 0$ 时，任何输入直接创建新帧（合并条件无法计算）。

**向量维度假定**：所有 $\mathbf{v}, \mathbf{x}$ 维度固定为 $D = 27$。维度不匹配时系统返回错误。
