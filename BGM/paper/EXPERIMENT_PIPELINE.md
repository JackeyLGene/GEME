# BGM 实验管线设计

> 三部曲第二篇：
> 实验 1（巴赫）→ 引入时间 → 引入 G0 → 证明分化 → 留下一具尸体
> 实验 2（细菌）→ 解剖尸体 → 空间替代 G0 → 不自锁 → 自己的核心实验
> 实验 3（吞噬）→ G0 到底用来干嘛？→ 信息效率驱动的演化

---

## 叙事弧

```
§3 巴赫 — 结尾留下"一具尸体"：
  "简单输入 → τ 平线 → 单元麻木。
  这是 G0 架构的代价：
  为了在均质输入下创造分化，
  必须引入一个中央反馈单元——
  而这个单元本身可以被驯服。"

§4 细菌 — 开头直接解剖这具尸体：
  "如果麻木是 G0 的代价，
  我们能不能在不需要 G0 的前提下
  同时实现分化且避免麻木？"
  回答："空间异质性 + 局部通信，
  不需要 G0，也不会麻木。"
  证据：L4 空间方差持续——16 单元无一个接近锁定。
  
  然后展开自己的核心实验：
  对比 1 — α 消融（证明空间异质性是必要条件）
  对比 2 — 冷/热启动（证明先验知识压制多样性）
  扰动响应（证明动态环境中不自锁）

§5 吞噬 — 再问一步：
  "那 G0 到底用来干嘛？
  如果空间自身够用，为何在进化中会出现集中协调？
  答案是：信息效率。冗余通信被淘汰。
  从细菌向巴赫的'前进方向'是通信拓扑的渐进优化。"

论文弧线：
  §3（有 G0）：创造了分化和锁死风险  ───→  留下一具尸体
  §4（无 G0）：避免了锁死风险         ───→  解剖尸体
  §5（吞噬）：G0 重新出场（进化版） ───→  尸体复活
```

## 实验 1：巴赫 —— 静态棱镜 → 动态生命

### 叙事角色

**读者应该感到三件事：**
1. "原来静态和动态差这么多"（τ 从参数变成读数）
2. "原来同一个输入可以分化出不同的耳朵"（ASA 的计算等价）
3. "复杂信息有价值——简单信息让系统麻木"（警觉度）

**§3 的结尾句应指向 §4：**
  "This numbness is the price of G0-mediated differentiation.
   Is there a way to differentiate without paying this price?"

### Pipeline

```
输入层：
  MIDI 编码 → Hz 分箱（27维）→ 时间步序
  
架构层：
  3 个标准 GEME（只改 seed）→ 同输入 → L6 输出
  G0（标准 GEME）→ 接收 L6 聚合 → 自指向量反馈
  g0_weight ∈ [0.1, 0.8]，由 G0 的 doubt 动态调制

实验层（4 步）：

  Step 1 — 静态 τ 基线
    所有 τ = 0.60（固定）
    期望：三单元冗余（MI spread ≈ 0）
    数据：τ 表、MI spread、graph similarity
    → 论文问题："参数相同 → 结构相同 → 没有多样性"

  Step 2 — 动态 τ 耦合
    τ_i = τ₀ × MI₀ / MI_i（τ₀=0.60, MI₀=0.026）
    期望：τ 分化，MI spread 增大（方向正确即可）
    数据：τ 演化轨迹、最终 τ 值表、pass-by-pass MI
    → 论文问题："参数变成读数 → 结构分化 → 多样性涌现"
    ⚠️ 关键：多次种子均值，不用单次运行。放弃不可复现的"6×"数字。

  Step 3 — G0 消融
    w = 0（无反馈）
    期望：无分化（无论 τ 静/动态）
    数据：有/无 G0 的 MI spread 对比
    → 论文问题："同输入下，分化需要 G0"

  Step 4 — 警觉度对比
    简单（BWV846）vs 复杂（BWV847 赋格）
    期望：简单 → τ 平线，复杂 → τ 振荡
    数据：后期 τ 波动幅度
    → 论文问题："信息价值 = τ 波动幅度"
    → **尸体的诞生**："简单输入 → G0 被驯服 → 系统麻木"

处理层（分析）：
  · τ 轨迹 → 分化图（pass vs τ）
  · MI spread → 声部分离度
  · co-occurrence graph → 单元间图相似度（Jaccard）
  · L4/total → τ 替代读数（≈0.033）
```

## 实验 2：细菌 —— 解剖尸体 + 自己的核心实验

### 叙事角色

**§4 开头的直接对话：**
  "我们在 §3 中看到：当输入缺乏结构时，G0 系统会麻木。
  这不是一个失败——这是一个必然。
  现在我们把 G0 移除。把输入从时间序列替换为空间梯度。
  16 个单元没有中央协调器——它们只和邻居说话。
  结果如何？"

**读者应该感到：**
1. "不需要 G0 也能分化——空间本身就是分散化器"
2. "更惊险的是——空间分化不会锁死"
3. "冷启动比热启动分化更强——先天知识压制多样性"

**对应 §3 的尸体：**

| §3 的局限（尸体） | §4 的应答 |
|--------------|-------------|
| 同输入 → 需要 G0 → 锁死风险 | 异质输入 → 不需要 G0 → 无锁死 |
| τ 平线 → 系统麻木 | L4 空间方差持续 → 警觉由空间梯度维持 |
| G0 是分化瓶颈 | 无中心也能分化 |

**结束 §4 时引出 §5 的问题：**
  "如果空间自身够用，为何进化还会产生 G0？
  我们需要一个理由——这个理由在 §5 中。"

### Pipeline

```
输入层：
  4×4 网格 + 高斯营养场 + 冯诺依曼邻居
  环境向量 α=0.7 + 邻居 L6 均值 (1-α)=0.3

架构层：
  16 个 GEME（冷启动，无预训练）
  无 G0

警觉度定义（与 §3 不同）：
  §3（有 G0）：警觉度 = τ 的时域振荡
  §4（无 G0）：警觉度 = L4 密度的空间方差
  麻木的网格：所有单元 L4 趋同
  警觉的网格：不同位置 L4 持续分化

实验层（3 相 + 2 对比）：

  Phase A — 培养（30 轮，营养源固定中心）
    期望：空间分化（L4 密度范围 [0, 0.313]）
    数据：L4 热力图、MI 分布、置信度阈值范围
    → 证据："空间差异足以产生分化"
    → 警觉度基线：Phase A 结束时 L4 空间方差

  Phase B — 扰动（营养源移到角落）
    期望：分化模式重组，不自锁
    数据：L4 分布变化轨迹
    → 证据："动态环境中维持警觉"

  Phase C — 恢复（10 轮，新位置固定）
    期望：新均衡建立
    数据：最终 L4 热力图
    → 证据："空间异质性防止了 τ 平线迁移"

  对比 1 — α 消融（= 警觉度的正面对决）
    α=1.0（纯环境）：L4 空间方差维持 ✓
    α=0.7（默认）：L4 空间方差维持 ✓
    α=0.0（纯通信）：L4 空间方差 ≈ 0——网格"全麻" ✗
    → 证明："环境异质性 + 局部通信"是警觉的条件
    → 对应 §3 Step 4：G0 振荡 vs 空间方差，同一功能不同实现

  对比 2 — 冷 vs 热启动
    冷：MI spread ≈ 0.098
    热：MI spread ≈ 0.045
    → 证明："先天知识压制多样性"
```

## 实验 3：吞噬 —— G0 的重新出场

### 叙事角色

**读者已经知道：**
- G0 在巴赫中是必要的（创造分化）
- G0 有代价（可能麻木）
- 空间异质性在局部通信中可以替代 G0 且不自锁

**但困惑出现了：**
  "如果空间自身够用，为什么进化会创造出巴赫中那种 G0？
  或者说——集中化协调有什么价值？"

**§5 回答：**
  "集中化的价值不在于更好的协调——
  而在于消除冗余通信。内化一个通信通道 = 提高信息效率。
  G0 不是被设计出来的——是进化中对冗余的自然淘汰。"

### Pipeline

```
输入层：
  同实验 2（16 单元网格）
  但第 31 轮宿主 (1,1) 吞并邻居 (2,1)

架构变化：
  吞并后：15 个外部传感器 + 1 个内化处理器
  猎物：只收宿主内部向量 → 输出只发宿主
  猎物从邻居列表中移除

实验层：

  Phase PRE — 培养（30 轮，同实验 2 Phase A）
  Phase PHAGO — 宿主吞并（第 31 轮）
  Phase POST — 运行（20 轮，追踪演化）
  平行对照组（不发生吞并）

关键指标：
  总 MI（15 传感器 vs 16 传感器）
  每传感器 MI 效率
  L4 分化 spread
  置信度阈值稳定性
  宿主 vs 猎物轨迹

⚠️ 当前数据问题：
  · 总 MI 提升：当前 14%，论文声称 39%
    → 运行 10 个种子取均值 + 误差线
  · 置信度 spread 方向：
    当前吞噬组发散（0.472）大于对照组（0.170）  
    与论文声称反向
    → 检查指标定义：应该是 L4 spread，不是 conf spread
  · 需要确认"信息效率 = total_MI / sensor_count"
```
