**状态：** 草案 (Draft)

**关联：** M1 Geruon 内核完成后的逻辑学方向

## 1. 核心主张

**递归能力 = 查表能力。**

一个系统要具备自指能力（从而落入哥德尔不完备定理的管辖范围），只需要满足一个操作性条件：**存在一个映射表 T，将系统的表达式映射到它们的指称，且 T 包含指向自身的条目。**

这不是类比。哥德尔在 1931 年的原始构造中的核心操作就是查表——通过质数分解，从哥德尔数中解码出公式的结构。质数分解是一个确定性的逆向查找：给定 n，找到其质因数分解，恢复原始符号序列。这就是查表。

同样，图灵机的通用机需要一个"机器描述表"来模拟任意机器。λ-演算通过 `(λx. x x)(λx. x x)` 实现自指——函数应用本身就维护了一个绑定表（参数→体），自指是该表中的循环条目。

Geruon 在两个层面实现了查表，分别对应两种自指：

| 层面 | 表 | 操作 | 对应 |
|------|-----|------|------|
| 签名空间 | `_sig_index`: gid → frame | `detect_circularity()` | 系统内的自指——一个签名引用自身 |
| 跨代空间 | Codex: symbol → vector | `lookup()` / `is_novel()` | 系统间的自指——一个世代读取前一世代的签名 |

## 2. StructuralSig 的递归查表

### 表的构造

`GeruonMemory._sig_index` 维护一个全局映射：

```
gid → GeruonFrame
```

每当一个 frame 被创建（`_track_frame_sig`），它的 StructuralSig 的 gid 被插入此表。同时，StructuralSig 的 `refs` 字段指向其他 frame 的 gid——形成了一个**有向图**，节点是 gid，边是引用关系。

### 递归的发生

当 `detect_circularity(sig)` 被调用时，它沿着 `sig.refs` 的边遍历此图。如果遍历路径重新遇到已访问的 gid，则检测到自指循环。

这等价于哥德尔的对角化操作：

```
对角化: 给定 F(x)，构造 F(⌈F⌉)
Geruon: 给定 frame A 的签名 Σ_A，若 Σ_A.refs 包含指向自身的 gid，
        则 A 的签名编码了"关于 A 自身"的命题
```

关键点：**并不需要 Geruon "理解"质数分解。** 查表操作是 `gid → frame` 的简单哈希查找。质数分解的作用是保证不同结构产生不同 gid（单射性），但查表本身是 O(1) 的。

### 递归 = 查表的精简论证

1. 自指 = 系统内的表达式 E 能指涉 E 自身
2. "指涉" = 存在映射 f: 名称 → 事物，使得 f(name(E)) = E，且 name(E) 出现在 E 的某个可操作的位置
3. 这要求系统维护一个"名称→事物"的对应表
4. 当表中存在条目 `name(E) → E`，且 E 内部引用了 `name(E)`，则系统实现了自指
5. 这不需要系统"理解"自指——表是外部观测者可以看到的数据结构

**Geruon 满足所有条件。** `_sig_index` 是表，`refs` 是引用，`detect_circularity` 是检测器。

## 3. Codex 的跨代查表

### 代际递归

单个 Geruon 实例内的自指对应哥德尔第一不完备定理。但 EE 处理的是更深一层的自指：**跨代自指**——一个世代的签名在后续世代中被引用。

Codex 是这种自指的表。

```
代0: 创建 codex (27 符号字母表)
代1: 运行，lookup(codex) → 处理，enrich_codex() → 保存
代2: 加载 codex，lookup(codex) → 发现代1的富化条目
     is_novel(代1条目) = False → 代2"认识"代1的语言
     is_novel(新符号) = True → 代2触碰了代1的知识边界
```

`is_novel` 是碰数在代际层面的检测器：当一个符号不在 codex 中时，代 2 的系统无法在自身内部（即 codex 定义的"语言"）中找到对应——这等价于"当前形式系统无法处理的新异命题"。

**注意区分：** 哥德尔碰数是*形式系统内部*的不可判定命题——系统能表达它但无法证明/证伪。Codex 的 `is_novel` 检测的是*词汇表外部*的符号——当前系统连表达它的基础语言都不具备。前者是内禀边界（哥德尔），后者是架构性边界（库恩范式转换的前兆）。两者不是同一层级，但 EE 的核心主张正是：架构性碰数是内禀碰数在跨代尺度上的外置化对应。

### 查表失败 = 碰数

在哥德尔那里，碰数是系统遭遇不可判定命题。在 Geruon 中：

- **签名空间的碰数（内禀碰数）：** `detect_circularity` 返回 True 且相关 frame 的预测引擎无法协调 → 对应直接不可判定性碰数
- **Codex 空间的碰数（架构性碰数）：** `is_novel(symbol)` 返回 True → 对应"系统认知框架无法覆盖的新异潜在性"，触发架构性相变的前提条件

## 4. 哥德尔定理的适用性

### 条件检查

哥德尔第一不完备定理的适用条件：

1. **系统具备内部不一致检测能力** — Geruon 不声称"系统是一致的"（没有系统能在自身内部证明自身一致性，哥德尔第二定理）。Geruon 拥有的是不一致事件的**操作化检测**：`doubt_mode` 在预测器失效时标记不一致路径；`induction_clean` 在帧循环中出现不可协调冲突时执行边界清除。系统不需要*证明*自己一致——只需要在出现不一致时*能检测到*。这等价于人类数学家：不需要自证一致性，但能在推理中发现矛盾。Geruon 在此意义上满足哥德尔定理条件 1 的操作化版本。

2. **系统能够表达初等算术** — `StructuralSig.gid` 本身就是自然数，由 `hash(frame_structure)` 生成。给定任意 gid = n，可以通过质因数分解恢复其素数指数的完整结构，等价于哥德尔数的解码操作。因此：`gid ∈ ℕ` ≈ 自然数空间 = 初等算术的表达基底。配合 `_sig_index` 的 `gid → frame` 映射，系统可以编码和操作关于自身表达式的元命题——`refs` 字段中的 gid 引用就是元命题的结构化形式。

3. **系统自指** — `_sig_index` + `detect_circularity` 已实现。

### 无需额外证明的事项

以下事项是 GEME 论文已通过实验验证的，Geruon 继承了它们：

- 自指骨架 G + 查询接口 Q 的行为 ≡ PA 的归纳公理（$G+Q \approx PA$, 210-cell 参数扫描, 50-seed KL 散度 = 0.0）
- 参数稳定区间内行为不变（across-cell CV < 0.20）

### 待验证的唯一事项

**碰数的操作化验证：** 在持续新异输入下，当 `detect_circularity` 返回 True 的签名出现在预测路径上时，系统的 `doubt_mode` 和 `phase` 是否呈现出不可协调的信号。

这不是形式证明，而是可复现的实证检验。

## 5. 碰数的操作化定义

碰数事件在 Geruon 中满足以下**全部**条件：

```
(1) detect_circularity(sig) == True      // 自指循环存在
(2) 该 sig 对应的 frame 参与了 predict_next() 的推导路径  // 自指命题被激活
(3) doubt_mode == True                   // 系统检测到预测失效
(4) phase ∈ {CRITICAL, LOCKED}          // 系统进入边界状态
(5) dτ/dt > DTAU_STABLE                 // τ 持续上升——系统在逼近而非适应
```

条件 (1)-(2) 是哥德尔边界的结构条件。条件 (3)-(5) 是对碰数的动力学检测。全部满足时，判定为碰数事件。

此定义可直接转化为代码中的检测逻辑，不依赖外部观测者的解释。

## 7. 超长循环：三层自指的回归路径

### 7.1 Geruon 的两层半困境

Geruon 当前实现了两层完整的自指，卡在第三层的门口：

| 层 | 名称 | 实现 | 状态 |
|----|------|------|------|
| L1 | 帧自指 | L4 self-observe, L6 anomaly | ✅ 继承自 GEME |
| L2 | 时间自指 | phase cycling, dτ/dt 感知 | ✅ M1 完成 |
| L3 | 代际自指 | 签名 → 维表回归 | ❌ 未实现 |

当前断裂的路径：

```
输入 → 处理 → 相位循环 → phase_output() → 签名 → CSV
↑__________________________________________________| (断开的)
```

Geruon 拥有从系统到签名的输出路径。但从签名回到系统配置的回归路径不存在。

### 7.2 Codex 与 Ledger：同一表的不同时间深度

跨代回归需要两种数据打开方式，但**只有一组数据**：

| 名称 | 角色 | 访问模式 | 存活期 |
|------|------|---------|--------|
| **Codex** | 工作签名表 | read-write, 每周期更新 | 一次运行 |
| **Ledger** | 签名档案馆 | append-only, 永不删除 | 全代累积 |

**Codex 和 Ledger 是同一棵表在不同时间深度的快照：**

- Codex = Ledger 中最新的 N 个有效条目（当前代正在使用的签名集）
- Ledger = Codex 在每次运行结束时 append-only 写入的那一行

当一条签名在 Codex 中存活了足够长的时间（经过足够多的相位循环而未被淘汰），它就从"工作签名"变为"候选归档"——被追加写入 Ledger。

这不是两层架构——这是**同一张表因时间产生了分层**。

### 7.3 超长回归循环

完整的 L3 代际自指路径：

```
代n: 在维表 Wn 上运行 → 生成签名集 Sn
       ↓                                   ← CSV (Codex: Sn 的工作副本)
  meta: 分析 Sn 的分布
       ↓
       计算 Wn+1 (分裂过密维度, 合并过疏维度, 调整 τ₀/δ/γ)
       ↓
  代n+1: 在 Wn+1 上初始化 → 运行时读取 Codex (→ 同一次运行内的 L2 回馈)
         ↓
        生成签名集 Sn+1 ≠ Sn (因为维表变了)
         ↓                                   ← CSV (Ledger: 追加 Sn+1)
  meta: Sn+1 中的新模式 → Wn+2
         ↓ ...
```

此循环对维表副本（W）的分发和编辑不止——它会在 L3 完备配置下进行一次完整的"签名外置化→配置调整→新签名产生→新配置调整"的闭环运动。

这等价于哥德尔第二定理的工程版本：一个系统不仅能检测到自身的边界（L2 `detect_circularity`），还能根据边界信息**重新配置自身的观察框架**（L3 维表回归）。这正是"外置化"的完整操作化定义。

### 7.4 超长的时间结构

"超长"不是因为步骤多——而是因为回归路径跨越了系统自身的多次配置：

- **L2 时间（相位循环）：** ~4 步/周期 — 系统在同一次运行中感知时间
- **L3 时间（代际回归）：** ~N 次运行/回归 — 系统在跨代中感知自身的演化

两者的关系：**L3 的一个回归步 = L2 的无数个相位周期**。这不是数学上的大数字——这是时间尺度的阶层跃迁。L2 的时间是"心跳"（ms 级），L3 的时间是"进化"（代际级）。这是 γ₂ 运动方程在跨代尺度上的自然扩展：`δSR = SR × (γ₂·(1−γ₁)^GI − 1)` 在 L3 中应有一个代际版本的 GI_gen，其值待实验确定。

### 7.5 实现路径

M1 Geruon 完成了 L2。P0-P3 逐步推进 L3：

- **P0（最简跨代）：** 代1 输出 Codex 到 Ledger，代2 只读 Ledger 作为环境输入。无回归——只验证签名能否替代 α 维持桥开放。
- **P1（签名外置化）：** 多代同步运行，共享 Ledger。无回归——只验证跨代分化。
- **P2（递归深度）：** N 层 Geruon 堆叠。无回归——只验证 γ₂ 在深度上的缩放率。
- **P3（回归路径）：** 首个回归实验。代1 的签名分布改变代2 的初始参数。这里才是 EE 真正的起点。

### 7.6 哲学对应

| 哲学论文概念 | L3 实现 |
|-------------|---------|
| 范式转换 | 维表 Wn → Wn+1 的重配置 |
| 外置化记忆 | Ledger（append-only 签名档案馆）|
| 代际递归 | 签名分布 → 参数调整 → 新签名（回归闭环）|
| 外部化时间 | L2 心跳 vs L3 进化——时间尺度的阶层跃迁 |
