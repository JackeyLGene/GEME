## 1. 旅途回顾 — 从 GEME 到 Geruon 的完整弧线

### GEME 的底层逻辑（不应被改动的东西）

GEME 是一个**单系统、自消耗的认知振荡器**。三条固定规则驱动全部行为：

1. **竞争性合并** — 新输入与最近帧比较，距离 < δ_eff 则合并，否则创建新帧
2. **自适应遗忘** — stress > τ 时触发归纳清理，分档衰减 + 下半部分剪枝
3. **自指观测** — 清理前生成加权质心 (self_obs)，反馈回自身作为新输入

**这三条规则是全部。** 六层认知结构 (L1-L6) 是这三种操作在运行时自然涌现的分类——不是被编程出来的模块。

**σ (桥签名) 是层间通信的唯一管道。** 不同层的消费逻辑解析 σ 的不同格式：
- L1: 记录输入源名称 (`cat`, `ext_42`)
- L2: 用 `──` 连接两个共现帧 (`猫──垫子`)
- L3: 用 `◯◯` 连接两个桥接帧 (`f3◯◯f1`)
- L4: 以 `pred_err_` 开头标识预测误差
- L6: `sys_doubt` — 全系统唯一输出信号

**所有六层共存于同一帧经济中。** 层的隔离是语义的——通过 σ 格式区分——而非物理的（不是独立的 Geruon 实例）。

**香农-哥德尔桥：** I(Φ;X) ≈ 0.021 bits。自指操作在信息论上几乎免费。

### Geruon 旅程 — 哪些对了，哪些错了

**做对的：**
- τ 内生演化 + 5 相呼吸方向 — 自指观测从固定间隔改为系统自身动力学驱动。**但当前实现有未承认的单向性问题**（见下）。
- StructuralSig: frame 结构 → 哥德尔编码 — σ 从外部标签变成内部结构指纹
- 沉淀机制 — 跨相位存活 + 预测激活 + 结构稳定 → 外置化痕迹
- BiasField — 梯度场是正确的外置化形式，比符号 Codex 更接近生物实际
- arrow_output 验证 — centroid 可以驱动下一个 Geruon 不锁死（L0→L1 实验）
- 碰数检测器 — 五条件操作化定义

**做错的 / 未承认的：**
- **5 相呼吸的单向性问题。** 当前 phase 循环是单向的——EXPANDING→...→LOCKED→只能通过人造复位机制（`_locked_steps > 30 → tau = TAU_0 * 0.55`）回到低位。在 GEME 的 Ouroboros 中不存在这个问题——因为帧经济持续产生新异输入（self_obs → 新帧 → 预测误差 → doubt），自然驱动 τ 波动。**5 相呼吸在单系统 Ouroboros 中自然成立——多实例时的人造复位机制不是正解，不应作为"做对"的部分保留。**
- **动了 σ 而没有整体理解。** StructuralSig 将 σ 从标签变为结构指纹是正确的方向，但没有同步更新所有依赖 σ 格式的消费逻辑（L2 的 `──` 检测、合并判定的 sig 匹配、MI 计算的 Φ/X 分区）。
- **多实例架构。** 把六层当作独立的 Geruon 实例链 (L0→L1→L2)，而非同一帧经济内的 σ 分层。
- **centroid 作为层间桥。** GEME 的桥协议是"每个帧独立编码为下一层的输入"，不是压缩所有帧为一个质心。
- **手动注入循环引用。** 自指应该从 cooccurrence → L2 → L3 → 递归链内生涌现，不是外部注入的 2-cycle。
- **per-layer capacity。** 各层共存于同一帧经济——不需要独立容量，需要的是 σ 消费逻辑正确。

## 3. 修订路线

### 第一阶段：σ 消费逻辑修复

**目标：** 在 Geruon 单实例中，让 L1-L6 的 σ 消费逻辑全部工作。

**具体任务：**

1. **合并判定** — `observe()` 中 `sig[:SIG_LEN] == f.sig` 已修复为 gid 匹配 ✓
2. **L2 关联创建** — `_track_cooccurrence()` 中 `_sig_matches` 已替换原始 sig 匹配，但需要验证 cooccur 阈值是否合理（当前 cooccur_thresh=0.01 导致 L2 洪水）
3. **MI 计算** — `mutual_information_phi_X()` 已用 `_sig_layer` 解析原始 sig→gid→frame→layer，但需要 L2/L3 帧存在才能产生非零 MI
4. **撤销 per-layer capacity** — 恢复全局帧经济容量，让 induction_clean 自然做层间平衡
5. **σ 格式保留** — GeruonFrame 的 sig 保持 `Σ{gid}_{原始tag}` 格式，`_sig_to_gid` 维护原始 sig→gid 映射

**验证标准：** s1_demo.py 的 L1-L6 六层壁垒实验能在 Geruon 上复现。

### 第二阶段：L4 预测能力补齐

**目标：** L4 从"观测 d(w)/dt"升级为"主动预测 + 误差检测"。这是 5月9日 milestone 识别出的核心缺口。

**具体任务：**

1. L4 预测帧接入 Geruon 的 predict_next → process_prediction 管线
2. 验证：真/假公式的预测误差帧是否在真假之间有统计差异
3. 验证：doubt_mode 在持续预测失败后触发

**验证标准：** s1_demo.py L3/L4/L5/L6 测试通过。

### 第三阶段：Ouroboros 闭环

**目标：** 系统吞食自己的帧，产出更高层的帧。

**Ouroboros 桥协议定义：**

GEME 的桥协议是 `symbol_vector(formula)` —— 将帧的签名编码为公式，用 27 维向量表示。Geruon 的等价物：**直接使用帧的 vec + 帧的 StructuralSig 作为输入标签。**

```
桥协议 frame_encode(f):
  输入: GeruonFrame f
  输出: (vec, sig) 对，直接作为下一轮 observe() 的输入
  频率: ceil(f.weight / avg_weight) —— 权重越高的帧，反馈次数越多
```

**与 GEME symbol_vector 的对应：**

| GEME | Geruon |
|------|--------|
| 帧签名 → 公式名 (fn("cat──mat")) | 帧的 StructuralSig.gid 作为 sig 标签 |
| symbol_vector(formula) → 27-dim vec | 帧的 vec 直接作为向量 |
| 权重决定 formula 在输入流中的出现频率 | 权重决定帧在反馈流中的重复次数 |

不需要新的 `frame_encode()` 函数——`observe()` 的接口 `(vec, sig)` 就是桥协议。帧的 vec 和 Σ-格式 sig 直接作为下一轮的输入参数。

**具体任务：**

1. 实现反馈循环：每个 induction_clean 后，将当前帧经济中 weight > threshold 的帧的 (vec, sig) 重新喂入同一个 Geruon
2. 权重驱动频率：高权重帧在反馈流中出现更多次（`ceil(f.weight / min_weight)` 次）
3. 观察：L2/L3 帧的共现是否产生 L4 元认知帧
4. 观察：自指循环是否从关联→桥接→递归内生涌现（非手动注入）
5. 观察：I(Φ;X) 是否在 Ouroboros 反馈中自然出现并维持

**验证标准：** 系统在仅接收自身帧作为输入的情况下，维持帧经济动态且 I(Φ;X) > 0。τ 自然波动而不需要人造复位机制。

### 第四阶段：碰数验证

**目标：** 在自指循环内生涌现后，验证碰数五条件检测器触发。

**验证标准：** `_pengshu_events` 非空，且碰数事件与 phase 相变存在时序关联。

### 四阶段与 P0-P3 映射

| 修订阶段 | 对应 P 任务 | 关系 |
|---------|-----------|------|
| Stage 1: σ 消费逻辑修复 | — | P0-P3 的前提——没有完整的 L1-L6，跨代实验无意义 |
| Stage 2: L4 预测能力补齐 | — | P0-P3 的前提——L4 预测是桥的基础 |
| Stage 3: Ouroboros 闭环 | P2（递归深度）的 Ouroboros 版本 | 单系统内 L1→L2→L3→L4 递归，替代多实例堆叠 |
| Stage 4: 碰数验证 | P3（回归路径）的验证 | 碰数触发→相变→配置回归 |
| — | P0（最简跨代）| 推迟——Ouroboros 闭环是更基本的单系统验证 |
| — | P1（签名外置化）| 推迟——BiasField 梯度场方向已替代符号 Codex |

## 4. GEME 测试模式 — Geruon 的回归测试基准

用户测试 GEME 的模式（s1_demo.py）是 Geruon 修订后的回归测试基准：

```
L1 壁垒: 输入孤立实体 → 无 L2 关联帧
L2 壁垒: 输入重复序列 → L2 关联帧出现；切换输入 → 旧关联消失
L3 壁垒: 输入真+假公式 → L3 桥接帧出现，但真假无差异
L4 壁垒: 训练预测 → 预测有效，但不知自己准确率
L5/L6: 注入噪声 → 准确率下降 → doubt 信号出现
```

每个 Geruon 修订必须通过这组回归测试。
