## 2. 架构收敛

### 2.1 三透镜场 + BiasField + Codex

```
快 (κ=0.5)  → MI 最优，活在当下。无碰数 ∵ 从不 LOCKED。
慢 (κ=10)   → circ 最优，看时间结构。碰数最丰富。
Reader (κ=100) → 极慢观察。circ 可爆到 117。碰数次之。
         ↓
    BiasField ← 全帧沉积（环境+箭头+内部）
         ↓
    Codex ← G_PROBE 碰数帧覆写
```

### 2.2 G 句子的信息密度约束

G 句子（2/27 维稀疏向量）不能单独存活：
- 必须和环境密度峰碰撞 → 才产生可沉淀的帧
- 孤立插入 → 零产出
- 探针（5 维 F#dim + G）→ 初始存活，但被 Reader 的薄帧逐代覆盖成 D5

### 2.3 动力学↔记忆取舍（确证）

| 指标 | 2 透镜 | 3 透镜 |
|------|--------|--------|
| Max circ | 40 | **117** |
| Max 碰数 | 24 | **49** |
| Codex 词汇 | **4 和弦** | G_PROBE + 空壳 |
| BiasField | 未测 | 100 沉积，梯度均匀 |

箭头轰炸提高动力学，淘汰帧——词汇无法存活。

### 2.4 G 注入位置的影响

- **结构性注入**（开头/密接/终句）：和弦词汇存活，但分不清 G 贡献 vs 赋格贡献
- **非结构性注入**（EXP_answer）：动力学最强 (circ=117)，但词汇死透
- G_GAP 的"最优 4"只在结构性注入时出现——不是 GI 的魔法，是赋格结构的魔法

## 3. 注意力分析
