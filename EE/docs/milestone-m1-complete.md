## 2. L0 的性质（已确认）

**定义：** 接收外部向量输入的 Geruon 实例。可以有多个 L0 单元。

**已摸清的性质：**

| 属性 | 观测值 | 条件 |
|------|--------|------|
| τ 动力学 | 0.60 → 锁定在 0.85-1.0 | 确定性输入（螃蟹卡农、重复公式） |
| τ 动力学 | 0.60 → 在 0.40-0.85 间振荡 | 多样化输入（13 符号 × 5 模式） |
| 相位分布 | 主导 tensing，短暂 critical/locked | 单单元运行 |
| 锁死倾向 | 所有确定性输入最终锁死 | 共现窗口饱和 → 预测平凡 → τ 上升 |
| 沉淀量 | 0-11 帧/3000 步 | 依赖输入多样性和种子 |
| 相位转移 | 2-18 次/5000 步 | 稀疏，不足以支持倍周期分岔测量 |

**关键约束：**
- 锁死是内建动力学特征，非 bug——稳定性与可测性矛盾
- 相位转移太少——单 L0 单元不适合直接做 Feigenbaum 分析
- 预测引擎依赖共现窗口（~100 步），确定性重复输入在该窗口内被完全学习

## 3. L1 的性质（已确认）

**定义：** 接收其他 Geruon 输出的 Geruon 实例。当前有两种模式。

### 3.1 递归模式（L0 → L1 级联）

**设置：** L1 接收 L0 的 `phase_output()`

| 属性 | 观测值 | 条件 |
|------|--------|------|
| τ 动力学 | τ=1.0，锁死 | L0 稳定在一个 phase 时 L1 输入无变化 |
| 相位转移 | 0-2 次 | L1 几乎无内部动态 |
| 失败原因 | phase_output 是 one-hot 稀疏向量，信息量不足以驱动 L1 预测引擎 | — |

**结论：** 递归模式下 L1 不可行。`phase_output()` 是为 Bacteria 邻居通信设计的离散信号，不适合作为递归层的唯一输入源。

### 3.2 并行共享场模式（L0 集群 + BiasField）

**设置：** N 个 L0 单元共享一个 BiasField，每个单元独立接收不同外部输入

| 属性 | 观测值 | 条件 |
|------|--------|------|
| τ spread | 0.191 | 5 单元，异质 bias_weight [0.01-0.08]，4000 步 |
| 相位多样性 | 2 个不同 phase（expanding, tensing） | 同条件 |
| 收敛风险 | 共字段 → 同质化（所有单元 τ 相同） | 所有单元接收相同输入时 |
| 分化条件 | 每个单元必须有**不同输入流** | 不同 random seed → 不同公式序列 |

**关键发现：**
1. **异质输入是分化的前提。** 同输入 → 即使有共享场也收敛
2. **BiasField 不消除分化。** 263 帧沉积后 τ spread 仍维持
3. **没有 G0，没有符号，没有"代"。** 梯度场介导的并行通信足以维持群体多样性

## 5. 架构理解的演进

### 5.1 初始假设（已被推翻）

> "层间通信用 phase_output，层内不需要独立通信场。"

**推翻原因：**
- phase_output 对层间通信来说信息量不足（L1 锁死）
- 层内没有通信场时，同输入导致所有单元收敛（SharedField 同输入实验）

### 5.2 当前理解

> "每个层级内部有一个隔离的 BiasField（层内梯度通信），层级之间通过更丰富的接口（待定义）连接。"

**支持证据：**
- SharedField 实验：层内梯度通信维持分化
- 递归实验反证：phase_output 不足以驱动跨层通信
- 神经元类比：不同皮层有不同神经递质环境，场是隔离的

### 5.3 未解决的问题

1. **跨层接口应该是什么？** 不是 phase_output（太稀疏），不是 centroid（太密集导致 L1 同质化）。需要介于两者之间的信号
2. **场隔离如何实现？** 代码层面简单（每个层一个 BiasField 实例），但隔离后的层间协调机制未定义
3. **L1 的输入源是什么？** 如果 L1 不直接接收外部输入，它的信息从哪来？仅靠 L0 的输出？还是 L1 也有部分外部输入？
4. **沉淀的去向？** 当前 enrich() 写入当前层的 BiasField。跨层沉淀是否应该写入上层场？

## 8. 关键突破：arrow_output() — 梯度场协议 (M1.8)

**日期：** 2026-05-17（M1 完成同日）

### 问题

之前 L1 锁死的根本原因不是"跨层接口的信息格式"，而是 Geruon 的输出被拆成了两个独立部分：
- `phase_output()` — 只有时间标签，没有内容
- centroid — 只有内容快照，没有时间

L1 接收到的不是一个箭头的完整输出，而是箭头被肢解后的残片。

### 解决

`arrow_output()` = **centroid 本身。** 帧的权重、存活周期、激活次数已经将时间信息编码在 centroid 计算中。内容和时间在帧经济中不可分——centroid 就是融合结果。不需要额外注入 phase 标签。

### L0→L1 验证 (10 seeds)

| 指标 | L0 | L1 | 解读 |
|------|-----|-----|------|
| transitions | 142 | 131 | **两级都维持动态** |
| phases | 4.0 | 4.5 | L1 多出 RESTING（L0 从未进入） |
| τ | 0.666 | 0.651 | 接近但独立 |
| T (phase 周期) | 28.3 | 30.2 | 几乎相同特征周期 |
| GI = T₁/T₀ | — | **1.100** | 无时间解耦 |
| I(Φ;X) | 0.0 | 0.0 | 桥未形成 |
| complexity | 97.7 | 93.3 | L1 略低于 L0 |

### 同步分析

- Phase 交叉相关在所有延迟上 ≈ 0（最大 r=0.098 at lag=21）
- τ 交叉相关 ≈ 0（最大 r=0.088 at lag=21）
- Phase 转移重合率 17%（随机期望 11%），微弱过量
- L1 phase 分布显著不同：tensing 52%（L0 67%）、resting 22%（L0 0%）

**L0 和 L1 不同步。** 它们在相同的特征周期（~28 步）上独立振荡。GI=1 是"L1 独立发展出与 L0 相同的特征周期"，不是"L1 跟随 L0"。

### 意义

**Geruon 原子验证通过。** arrow_output 传递信息的同时保留了接收者自身自指完整性。L1 不是 L0 的复制品——它用自己的帧经济、τ 动力学、预测引擎独立处理了 L0 的输出，产生了独特的 phase 轮廓但保持了相同的时间尺度。

### 与 BiasField 的关系

- arrow_output()：centroid——**箭头的瞬时方向**，跨层接口
- BiasField：沉淀帧累积——**层内的长期记忆场**，不参与跨层通信
- 两者分工明确：箭头驱动下一层，场偏置当前层
