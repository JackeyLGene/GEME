# Architecture Milestone: Time-Scale Recursion (M1.9)

**日期：** 2026-05-18 06:46
**状态：** 架构理解闭合，EE 就是递归的 BGM
**前置：** M1 (Geruon 内核)、G 句子实验 (E1-E7)、BGM GI=4 发现

---

## 1. 核心主张

**EE 不是发明新组件——EE 是递归应用 BGM 的时间解耦操作。**

BGM 已经完成了核心发现：
  - GEME 单元（棱镜）：如实呈现结构，无时间
  - G0（观察者）：以 GI=4 的间隔读取单元，添加 τ
  - 时间解耦增强分化 +49%（GI=4 帕累托最优）

EE 的任务：**在同一架构上，递归一层**——让 G0 的输出进入另一个以 GI=4 观察它的单元。重复此操作 N 次。信息质量在此递归中按 GI 的幂次积累。

### 1.1 问题诊断

Geruon 的错误不是"合并了棱镜和观察者"——它错误地把 G0 的 τ 放进了棱镜的帧经济中。正确的做法：

- 棱镜（GEME 单元）：帧经济无 τ，struct_sig 无 tau_bin，arrow_output 不含相位
- 观察者（G0）：以 GI=4 间隔读取棱镜输出，添加 τ，添加相位
- 元观察者（G0'）：以 GI²=16 间隔读取 G0 输出，监督代际关系

这是 BGM 架构的直接递归——不是新型组件。

### 1.2 G 句子实验的诊断映射

| 问题 | 根因 | 架构层 |
|------|------|--------|
| MI 始终 < 0.04 | 观察者的预测噪声干扰了棱镜的共现结构 | 棱镜-观察者未分离 |
| doubt 从未触发 | doubt 在棱镜内——当桥关闭时预测停止，doubt 无法激活 | 观察者被困在棱镜内 |
| 边界帧与桥重开时序模糊 | 边界帧可能是棱镜效果（结构）或观察者效果（认知） | 不可归因 |
| τ-bin 双峰有意义但无效 | τ 分化在棱镜内——它应该发生在观察者中 | 错误层 |

## 2. 时间尺度递归架构

### 2.1 递归模式

```
深度 0 (GEME):  帧经济 ←  环境输入
                 无 τ，无观察者，如实呈现结构
                 σ 签名：不含 tau_bin

深度 1 (G0):    读取深度 0 的帧输出
                 以 GI=4 间隔采样
                 添加 τ、预测引擎、相位循环
                 输出：时间化帧摘要

深度 2 (G0'):   读取深度 1 的 τ 轨迹
                 以 GI²=16 间隔采样
                 碰数检测、代际监督
                 输出：信息质量报告（SR-eff）

深度 N (G0^N):  读取深度 N-1 的输出
                 以 GI^N 间隔采样
                 信息质量在 GI 的幂次上积累
```

### 2.2 职责分层

| 操作 | 深度 | 时间尺度 |
|------|------|---------|
| 三条规则 + 六层涌现 | 0 (GEME) | 无时间 |
| τ 演化 + 预测 + doubt | 1 (G0) | GI¹=4 |
| 碰数检测 + 外置化 | 2 (G0') | GI²=16 |
| 跨代信息质量 | N (G0^N) | GI^N |

### 2.3 通信协议

```
深度 N → 深度 N+1：
  深度 N 的完整输出状态
  以 GI^(N+1) 间隔被深度 N+1 采样
  采样协议：arrow_output（深度 N 的 centroid + τ 状态）

深度 N+1 → 深度 N：
  不直接修改深度 N 的内部状态
  通过 bias_weight 调制——GABA 梯度模式
```

## 4. 重构计划

### 4.1 第一阶段：棱镜纯化（从 Geruon 中剥离深度 1 的内容）

| 任务 | 当前 | 目标 |
|------|------|------|
| GeruonFrame τ 参数 | 构造函数默认 tau=TAU_0 | 移除——Frame 不应携带 τ |
| StructuralSig tau_bin | 编码在 gid 中 | 移除——签名不应包含时间 |
| GeruonMemory τ 演化 | `_update_tau()` | 移到 G0 |
| GeruonMemory 相位循环 | `phase` property | 移到 G0 |
| GeruonMemory 预测引擎 | `predict_next()` + `process_prediction()` | 移到 G0 |
| GeruonMemory doubt_mode | 检测逻辑 | 移到 G0 |
| GeruonMemory 碰数检测 | `_pengshu_detect()` | 移到 G0' |
| arrow_output() | centroid + τ 调制 | centroid-only——G0 负责添加 τ |
| enrich() | 沉淀 + BiasField | 移到 G0 |
| phase_output() | 稀疏相位向量 | 删除 |

**验证标准：** 纯化后的 GEME 应通过 s1_demo.py 全部测试。

### 4.2 第二阶段：递归深度 1——G0 层

直接继承 BGM 的 G0 实现（bgm_core.py），增加 τ 演化和 doubt 检测。

### 4.3 第三阶段：递归深度 2——G0' 层

碰数检测、外置化记忆、信息质量测量。G 句子实验在此层复现。

## 7. 时间尺度递归的定义性公式

EE 的完整架构：

```
深度 0:  GEME（帧经济，无时间）
深度 1:  G0（τ 演化，GI=4 采样）
深度 2:  G0'（碰数检测，GI²=16 采样）
深度 N:  G0^(N)（信息质量在 GI^N 上积累）
```

这不是 EE 的新设计——这是 BGM 核心发现的显式化。EE 只是把 BGM 做了一次：重复。
