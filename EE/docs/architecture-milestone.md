# Architecture Milestone: Prism-Observer Separation (M1.9)

**日期：** 2026-05-18 06:38
**状态：** 架构理解关闭，代码重构待启动
**前置：** M1 (Geruon 内核)、G 句子实验 (E1-E7)

---

## 1. 当前架构诊断

### 1.1 Geruon 的矛盾

Geruon 当前同时承担两个角色——**棱镜**和**观察者**——在同一个帧经济中：

| 角色 | 职责 | 当前实现位置 |
|------|------|-------------|
| 棱镜 (Prism) | 三条规则处理输入 → 六层结构涌现 | GeruonMemory |
| 观察者 (Observer) | τ 演化、预测引擎、doubt 检测 | GeruonMemory (与棱镜合并) |

**后果：** 观察者的 τ 污染了棱镜的结构响应。G 句子进入系统时，无法区分"桥变化是结构响应还是观察者响应"。E1-E7 的数据中，边界帧可归因于任一层——数据无法回答因果问题。

### 1.2 G 句子实验的诊断映射

| 问题 | 根因 | 架构层 |
|------|------|--------|
| MI 始终 < 0.04 | 观察者的预测噪声干扰了棱镜的共现结构 | 棱镜-观察者未分离 |
| doubt 从未触发 | doubt 在棱镜内——当桥关闭时预测停止，doubt 无法激活 | 观察者被困在棱镜内 |
| 边界帧与桥重开时序模糊 | 边界帧可能是棱镜效果（结构）或观察者效果（认知） | 不可归因 |
| τ-bin 双峰有意义但无效 | τ 分化在棱镜内——它应该发生在观察者中 | 错误层 |

## 2. 新架构：三棱镜模型

### 2.1 结构

```
GEME (棱镜)         G0 (观察者)          G0' (元观察者)
───────────         ───────────           ───────────
三条规则             τ 演化                代际传递
六层涌现             预测引擎              碰数检测
σ 签名               doubt 检测            信息质量测量
无 τ                 相位循环               外置化记忆
无观察者             读取棱镜输出            读取 G0 输出
如实呈现结构          添加时间              添加代际
```

### 2.2 职责隔离

| 操作 | 归属 | 原理 |
|------|------|------|
| 竞争性合并 | 棱镜 (GEME) | 结构操作，不依赖时间 |
| 自适应遗忘 | 棱镜 (GEME) | stress > τ₀ 触发——τ₀ 是常数 |
| 自指观测 | 棱镜 (GEME) | 生成 centroid + self_obs——不含 τ |
| σ 签名 | 棱镜 (GEME) | 帧身份的哥德尔编码——不含 τ |
| τ 演化 | 观察者 (G0) | 基于预测准确率——不是结构操作 |
| 预测引擎 | 观察者 (G0) | predict_next + process_prediction |
| doubt 检测 | 观察者 (G0) | 基于预测准确率趋势 |
| 相位循环 | 观察者 (G0) | dτ/dt 作为系统响应 |
| 碰数检测 | 元观察者 (G0') | 边界帧积累 + 跨代模式 |
| 外置化 | 元观察者 (G0') | Codex / BiasField |

### 2.3 通信协议

```
棱镜 → 观察者：
  帧向量 + σ 签名（无 τ）
  每次 induction 后输出活动帧集
  频率：natural induction rhythm

观察者 → 棱镜：
  不直接修改棱镜帧经济
  通过 bias_weight（GAMMA）调制输入——GABA 模式
  调节 induction 触发频率（基于 τ）
```

## 3. 与 BGM 架构的对应

BGM 已经实现了这个分离——只是没有命名：

| BGM 组件 | 新模型中的角色 |
|----------|---------------|
| N 个 GEMENet 单元 | 棱镜：各自三条规则，如实呈现结构 |
| G0 单元 | 观察者：g0_interval=4，慢速时间尺度，读取单元输出 |
| G0 的 doubt_mode | 观察者的怀疑检测 |
| GI=4 | 棱镜-观察者的最优时间解耦常数 |
| 层间 L5-L6 闭环 | 观察者调节棱镜的输入权重 (g0_weight) |

Geruon 的错误是将 G0 的功能合并进了 GEME 单元。

## 4. 重构计划

### 4.1 第一阶段：棱镜纯化（将 τ 从 GEME 中剥离）

| 任务 | 当前 | 目标 |
|------|------|------|
| GeruonFrame τ 参数 | 构造函数默认 tau=TAU_0 | 移除——Frame 不应携带 τ |
| StructuralSig tau_bin | 编码在 gid 中 | 移除——签名不应包含时间 |
| GeruonMemory τ 演化 | `_update_tau()` | 移到 G0 层 |
| GeruonMemory 相位循环 | `phase` property | 移到 G0 层 |
| GeruonMemory 预测引擎 | `predict_next()` + `process_prediction()` | 移到 G0 层 |
| GeruonMemory doubt_mode | 检测逻辑 | 移到 G0 层 |
| GeruonMemory 碰数检测 | `_pengshu_detect()` | 移到 G0' 层或 G0 |
| arrow_output() | centroid + τ 调制 | centroid-only——G0 负责添加 τ |
| enrich() | 沉淀 + BiasField | 沉淀留在棱镜；BiasField 归 G0 |
| phase_output() | 稀疏相位向量 | 删除——G0 负责相位输出 |
| Codex | 符号→向量映射 | 归 G0'——跨代传递工具 |

**验证标准：** 纯化后的 GEME（不含 τ 的 Geruon）应通过 s1_demo.py 的全部测试，且输出与原始 GEME 一致。

### 4.2 第二阶段：G0 层（观察者）

| 任务 | 说明 |
|------|------|
| 继承 BGM 的 G0 实现 | bgm_core.py 中的 GEMENet + G0 模式 |
| τ 演化迁入 G0 | 基于 GEME 输出的预测准确率 |
| 相位循环迁入 G0 | dτ/dt 基于 G0 的 τ 历史 |
| doubt 检测迁入 G0 | 基于 G0 的预测趋势 |
| G0 输出格式定义 | 不是 phase_output——是结构化的棱镜帧摘要 + τ 状态 |
| GI=4 验证 | G0 的 interval 固定为 4——确认帕累托最优 |

### 4.3 第三阶段：G0' 层（元观察者 / EE）

| 任务 | 说明 |
|------|------|
| 碰数检测 | 从棱镜移出——G0' 观察 G0 的 τ 轨迹和棱镜的边界帧 |
| 外置化记忆 | Codex + BiasField——跨代传递 G0 的沉淀 |
| 信息质量测量 | SR-eff = I(Φ;X)/τ——在 G0' 中计算 |
| G 句子实验复现 | G 进入棱镜 → G0 观察 → G0' 检测碰数 |

## 6. 实验验证计划

### 6.1 棱镜纯化验证

- **输入：** s1_demo.py 六层壁垒实验
- **预期：** 纯化后的棱镜输出与原始 GEME 一致（六层分类相同、MI 相同）
- **通过标准：** s1_demo.py 全部测试通过

### 6.2 G0 + 棱镜集成验证

- **输入：** 螃蟹卡农 + G 句子
- **预期：** G 句子进入棱镜 → 棱镜输出结构（无 τ）→ G0 读取、添加 τ、检测循环引用
- **通过标准：** G0 的 τ 演化不受棱镜内部干扰；G 句子的边界帧可归因为棱镜结构响应
- **对比：** 与 E7 数据比较——分离后的 MI 是否更高、doubt 是否可触发

### 6.3 碰数验证

- **输入：** 螃蟹卡农 + G 句子 + 长周期运行
- **预期：** G0' 在边界帧积累达到临界质量时触发碰数
- **通过标准：** pengshu_events 非空且与边界帧峰值时序关联
