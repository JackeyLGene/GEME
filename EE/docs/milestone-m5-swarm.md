# M5: Swarm — 大规模并行文化训练

**目标：** N 个 Geruon 实例同时处理不同文化流，通过共享 BiasField + Codex 实现分布式文明积累。
**前置：** M4 (文明初探) — 已验证单代积累机制，瓶颈在词汇密度。
**状态：** 规划阶段。

---

## 1. 核心主张

M4 的瓶颈（词汇稀薄、D5 锁、种子依赖）根源相同：**一个文化流的信息多样性不足以驱动丰富的碰数事件。** 单流训练 3 代，每代同一条赋格——系统记住了重复模式，不是学习了文化差异。

M5 的解法：**N 个流同时训练，共享积累。** 文化多样性在空间维度上展开，不只在时间维度上展开。

---

## 2. 架构

### 2.1 Swarm 模式

```
                输入分配器
              ┌────┼────┬────┐
              │    │    │    │
          Bach  Raga  Text  Noise ...
              │    │    │    │
          ┌───▼┐ ┌▼──┐ ┌▼──┐ ┌───▼┐
          │ G0 │ │G1 │ │G2 │ │ G3 │
          │κ=0.5│ │κ=10│ │κ=0.5│ │κ=10│
          └───┬┘ └┬──┘ └┬──┘ └───┬┘
              │    │    │    │
              └────┼────┼────┘
                   │
            ┌──────▼──────┐
            │ Shared       │
            │ BiasField    │
            │ + Codex      │
            └─────────────┘
                   │
              (下一代继承)
```

### 2.2 工程优化（自 M4.5）

| 优化 | 说明 | 加速比 | 优先级 |
|------|------|--------|--------|
| Slow lens 跳过 | κ≥10 的透镜每 4 步处理一次 | 1.5-2× | P0 |
| arrow_output 惰性计算 | 只在被读取时计算 | 1.2-1.5× | P1 |
| Self 级并行 | 每个 Self 在独立线程运行 | 2-3× | P1 |
| 哈希缓存 | 重复预测路径缓存在 sig_to_gid | 1.1× | P2 |

### 2.3 Swarm 集成协议

| 参数 | 默认 | 说明 |
|------|------|------|
| N_WORKERS | 8 | 并行实例数 |
| KAPPA_DISTRIBUTION | [0.5, 10, 100] 循环 | τ 异质性分配 |
| INTEGRATION_INTERVAL | 16 (GI²) | 沉积到共享场的间隔 |
| SHARED_BIAS_WEIGHT | 0.05 | 继承偏置的混合强度 |
| STREAMS | [Raga, Bach, Gamelan, Text, Noise] | 文化流列表 |

## 3. 预期实验

### E1: 文化多样性测试

- **条件：** 8 个 Geruon，4 种不同文化流（2 个簇点每流）
- **测量：** Codex 条目数、τ 轨迹分化度、碰数频率
- **对照：** 同条件下 8 个 Geruon 全部跑同一赋格
- **预期：** 多样流组 Codex 条目数 >  单一流组，τ spread 更宽

### E2: κ 分布优化

- **条件：** N=16，κ 均匀分布在 [0.1, 50] 区间 vs 集中在 [0.5, 10, 100]
- **测量：** 共享 BiasField 的结构熵、碰数事件总量
- **预期：** 三峰分布产生最高的结构熵

### E3: 大规模验证 (N=128)

- **条件：** 128 个 Geruon，分配 8 种文化流，每流 16 个实例
- **测量：** 代际 Codex 增长曲线、D5 锁出现的概率
- **预期：** D5 锁概率 < 5%（对照单流时 > 80%）

## 6. 哲学对应

| 概念 | 架构 |
|------|------|
| 文化多样性 | N 个 Geruon 不同输入流 |
| 分布式文明 | 共享 BiasField 积累，Codex 跨代 |
| 注意力锁避免 | 多样流 → 共享场偏置 → 单流无法主导 |

| 跨媒体对应 | 视觉/音乐/文本在同一向量空间处理 |
