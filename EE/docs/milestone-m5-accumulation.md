# M5: 积累 — Codex 大规模并行

**日期：** 2026-05-20
**状态：** 规划阶段
**前置：** M4 (文明初探)

---

## 1. 瓶颈与方向

M4 的瓶颈：单一数据流的信息多样性不足以驱动丰富的 We 帧。

M5 的方向：N 个 We 并行，每 We 处理不同文化流，共写同一共享 Codex。
不依赖时间维度上的代际迭代——依赖空间维度上的文化多样性积累。

---

## 2. 架构

### 2.0 积累层的哲学定位

**积累层是考古层，不是反馈层。**

- We 层在自己的流里完成内部迭代——不读取积累层
- 积累 Geruon 只收 We 的最终 Codex，不参与 We 的认知循环
- 积累产物（跨文明共识）可以供后续研究或人类分析，但不回流入 We

```
We 层 (认知):     Self ↔ Self (害传递) → 活得 Codex
                      ↓
积累层 (考古):     所有 We 的 Codex → accumulation Geruon (三规则竞争)
                      ↓
                  跨文明共识 (不反哺 We)
```

### 2.1 粒度决策（经 M4 数据验证）

| 粒度 | 并行单位 | 原因 |
|------|---------|------|
| **We 级并行** | 每个 We（3 Selves, 3 lenses）一个进程 | Self 间 GI=4 交换箭头，跨进程 IPC 开销不可接受 |
| Self 内部 | 3 透镜在同一进程 | 共享帧经济，每步交换箭头 |
| We 之间 | 只写共享 Codex，不通信 | 文化单元之间不需要 harm 交换 |

### 2.2 We 结构

每个 We = 1 进程，含 3 个 Self，每 Self 含 3 个透镜 (κ=0.5, 10, 100)：

```
进程 A: We_A (3 Selves)  → 流 A → 沉积害 → 共享 Codex
进程 B: We_B (3 Selves)  → 流 B → 沉积害 → 共享 Codex
进程 C: We_C (3 Selves)  → 流 C → 沉积害 → 共享 Codex
...
                                  ↓
                        下一代 We 继承共享 Codex 作为偏置
```

### 2.3 共享 Codex 协议

| 参数 | 值 | 说明 |
|------|-----|------|
| 写频率 | 每 16 步 (GI²) | 与 M4 一致的害沉积节律 |
| 同步方式 | 代际边界 | We 进程不实时同步——完成各自流后读共享 Codex |
| 锁机制 | 不需要 | 共享 Codex 通过代际边界写入，we_w0.py 已验证先 complete 再 write |
| 规模上限 | 43 We (128 Self) | 当前单机可行上限 |

## 3. 三代运行模式

```
Gen 0:                 Gen 1:                    Gen 2:
We_A → 流 A           (继承 Codex gen 0)         (继承 Codex gen 1)
We_B → 流 B           We_A → 流 C               We_A → 流 B
We_C → 流 C           We_B → 流 A               We_B → 流 D
We_D → 流 D           We_C → 流 E               We_C → 流 A
...                     ...                       ...
    ↓                    ↓                         ↓
共享 Codex              共享 Codex                 共享 Codex
```

每一代的 We 数量固定。流在代际之间轮换——每个 We 经历不同文化，Codex 积累跨文化偏置。

---

## 4. 与 BGM 实验的对应

| BGM | M5 |
|-----|-----|
| 3 个 GEME 单元 | 3 个 Self（每 Self 含 3 透镜） |
| 1 个 G0 观察单元 | 共享 Codex 积累 |
| 不同声部的赋格 | 不同文化流 |
| GI=4 的帕累托最优 | We 内 3 Self 的 GI=4 交换 |

BGM 的架构在 M5 中被完整递归：3 个单元 + 1 个观察者 → 3 个 Self + 共享 Codex。
