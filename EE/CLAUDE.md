# CLAUDE.md — External Engine (EE)

## 项目定位

EE 是"生成性信息实在论"(Generative Information Realism) 三部曲的第三篇。三部曲脊椎：

| 篇 | 隐喻 | 核心贡献 | 关键量 |
|---|------|---------|--------|
| **GEME** (物理学) | 自指基元 — 最小桥 | 静态。I(Φ;X) ≈ 0.026 bits | δ=0.19, γ₁=0.05, τ₀=0.60 |
| **BGM** (化学) | 时间进入 — 桥呼吸 | τ 分化。锁。信息成为实体 | GI=4, γ₂=1.228 |
| **EE** (生物学) | 信息建造自己的时间 | 签名外置化。桥观察桥观察桥。g_info 作为场方程 | 待发现 |

**核心主张：** 外置化(externalization)是在信息本身内建造时间结构——不是把信息"放到外面"，而是创造新的观察层（签名），它们在个体死亡中存活，并成为新桥的输入。

**哲学根基：** 三大公设贯穿全部工作：
1. **生成第一性** — 实在的本质是永续的"生成之流"，静态存在只是瞬时稳态模式
2. **信息-观测构成性** — 信息是可传递的差异，观测将潜在性转化为事实
3. **自指生成模式** — 认知是维系"生生"存续的高阶自指信息处理

### 哲学笔记与计算论文的关系

`Philosophy/` 下的三篇中文笔记是整个三部曲的**思想源头**——核心公设、形式化定义、动力学框架均在其中完成。`paper/` 下的两篇英文计算论文（GEME/BGM）是对哲学内核的**学术翻译**：将生成性信息实在论的思想，用参数扫描、消融实验、统计验证等学术界可接受的保守策略重新表述，目标是期刊发表。

EE 与此不同：仅以预印本形式完成。这意味着可以摆脱为满足审稿人期待而附加的保守修辞，恢复哲学笔记中更直接、更具思想强度的表达方式。代码仍然严谨，但论证不必藏在"边界声明"后面。

## 代码架构（继承自 GEME/BGM）

### geme.py — 核心引擎

无外部依赖的单文件 Python 库。核心组件：

- **`Term` / `Formula`** — 最小公式语言（constant, function, equation, conjunction, implication）
- **`structural_signature()`** — 从公式提取结构签名
- **`symbol_vector()`** — 公式→27 维符号频率向量（`_ALPHABET`: 27 个符号）
- **`Frame`** — 加权向量 + 签名 + 层级标签 (L1/L2/L3/L4/L6)
- **`Memory`** — 竞争合并、共现追踪、链形成 (L1→L3)、自观察 (L4)、预测、怀疑模式、量子合并、多宇宙分支
- **`GEME`** — 顶层 API：`process_sig()`, `process_vec()`, `input()`, `consolidate()`, `evaluate_sig()`, `save()/load()`

层级系统：**L1** (基础帧) → **L2** (关联帧, `ASSOC_SEP`) → **L3** (链/桥帧, `CHAIN_SEP`) → **L4** (自观察/元帧) → **L6** (异常/怀疑帧)

### bgm_core.py — 多单元网络

`GEMENet` 管理 N 个并行 GEME 单元 + 1 个 G0 (观察者) 单元：
- G0 以较慢时间尺度运行 (`g0_interval`)
- L5-L6 闭环：G0 的 doubt_mode 调节 `g0_weight`
- 核心发现：跨尺度涌现需要层级间时间不对称

### bgm_bacteria.py — 空间网格实验

`BacteriaGrid`: 8×8 = 64 个 GEME 单元，冷启动，无 G0：
- 营养梯度 (Gaussian) + 局部邻居通信 (von Neumann)
- 三阶段实验：培养 (30轮) → 扰动 (10轮) → 恢复 (10轮)
- 分析指标：L4 密度、互信息、L6 异常分数、梯度相关性

### midi_encoder.py — 音乐编码

将乐谱转换为 MIDI 事件流，音符频率分箱到 27 维向量。

### geruon.py — EE 内生时间内核 (Milestone 1)

GEME 的最大局限：`_step_counter` 只是单调递增的步数，没有时间结构。BGM 的"时间"（τ 分化、桥呼吸、GI=4）是外部脚本编排的结果，不是 GEME 内部生成的。

Geruon 用一个核心修改解决此问题：**τ 从固定常数变为内生变量。**

| 属性 | GEME | Geruon |
|------|------|--------|
| τ | 固定 TAU = 0.60 | 随预测历史演化，有惯性 |
| 时间感知 | 无 | dτ/dt, phase (5 相) |
| 单元输出 | L6 标量 | (anomaly, phase) → 27-dim 向量 |
| G0 | 必需（手动设 g0_interval） | 不需要——相位差自然产生观察层级 |
| 适用场景 | BGM 期刊发表 | EE 思想实验 |

**Phase 循环：** `EXPANDING` (τ↓, 桥打开, 新异吸收) → `RESTING` (τ 低且稳定, 维持) → `TENSING` (τ↑, 逼近边界) → `CRITICAL` (τ 近顶, 边界临接) → `LOCKED` (τ 锁定, 桥关闭)。LOCKED → EXPANDING 即相变。

**arrow_output() — 内容-时间融合（梯度场协议）：** 加权帧 centroid（系统"所知"）与 phase 签名（系统"所在的时间"）以 τ 为调制比融合。τ 低→内容主导（学习），τ 高→时间签名主导（逼近锁死）。内容和时间不可分——这就是梯度场协议：每个值都经过了此 Geruon 的完整内部状态处理。

**Phase-aware 输出编码：** `vec_dim` 维向量（默认 16，可配置），5 个 phase 各占一个不重叠的波段（近等分）。两个 anomaly 相同但 phase 不同的单元产生可区分的向量。

**27 维脚手架的拆除：** GEME 的 `_ALPHABET`（swap, pair, comm… 27 个公式符号）是为期刊发表的公式语言实验（G+Q≈PA）服务的。对 EE 的签名外置化和跨代通信没有任何意义。Geruon 用 `vec_dim` 参数替代硬编码 27——默认 16，干净的 2 的幂，无语义附加。`_compute_phase_bands(vec_dim)` 自适应分配相位波段。

**与 Bacteria 的关系：** Bacteria 证明 64 单元 + 空间异质性 → 无 G0 分化。Geruon 将 `l6_to_vec(L6)` 替换为 `phase_output()`，使邻居通信携带相位信息，是 Bacteria 的自然延申。

**StructuralSig — 签名的哥德尔编码 (M1.1)：**

GEME 的签名是程序员给 frame 贴的字符串标签（`"cat"`, `"pred_err_..."`）。Geruon 将签名替换为 frame 结构的哥德尔编码——签名 = 结构的必然产物，不是外部赋予的名称。

- `StructuralSig(vec, weight, layer, refs)` — 不可变、可哈希、可判等。相同结构 → 相同 gid
- `gid` (Gödel ID) 通过质数分解编码 `(vec_hash, weight_log2, layer_int, ref_gids)`，遵循哥德尔原始方案保证单射性
- `refs` 支持签名递归引用 → 签名空间在自指下闭合
- `detect_circularity()` — 追踪引用链检测自指循环，碰数的前身
- `GeruonFrame(Frame)` — 扩展 Frame 携带 `struct_sig`槽位，sig 字符串格式 `Σ{6位gid}_{标签}`
- `_circular_refs` — GeruonMemory 追踪所有检测到的自指循环

**关键常数：**
- `TAU_ADAPT_RATE = 0.025` — τ 惯性（慢于 γ=0.05，确保动量）
- Phase 阈值: `PHASE_RESTING_CEIL = τ₀×0.55`, `PHASE_LOCKED_FLOOR = τ₀×1.25`
- `DTAU_STABLE = γ×0.15` — |dτ/dt| 低于此视为相位稳定
- `_GODEL_PRIMES` — 15 个质数用于哥德尔编码

**Codex — 外置化符号表 (M1.2)：**

27 维字母表被拆掉的是其作为**内部硬编码向量空间**的角色。Codex 是同一个符号表作为**外部配置、跨代可传递**的形态——这是 EE 核心操作的入口。

- `Codex.from_alphabet(alphabet, vec_dim)` — 从符号字母表创建（如 GEME 的 27 符号），每个符号 → 确定性稀疏向量
- `Codex.lookup(symbol)` → vector | None — 查询；返回 None 即"novelty"，潜在的碰数
- `Codex.add(symbol, vec)` — 下一代富化 codex
- `Codex.save/load` — JSON 持久化，代际传递的物理形式
- `Geruon(codex=codex)` — 接受 codex 作为外部锚
- `Geruon.input(str)` — 有 codex 时逐字符查表，未知字符登记为 `_codex_novelties`
- `Geruon.enrich_codex(min_weight)` — 将稳定 L1 帧写回 codex，传给下一代
- Codex 大小（如 27）是代 1 的历史产物，不是系统的先天约束

Codex 直接兑现 EE 种子 #3（签名层外置化）和种子 #6（外置化作为锚）——语言作为跨代锚固。

**碰数 (Pengshu) 检测器 (M1.4)：**

五条件操作化定义，全部满足时判定碰数事件：

1. **circularity** — `detect_circularity(sig)` 返回 True，预测路径中存在自指循环
2. **activation** — 自指 frame 参与了 `predict_next()` 推导路径（通过 `_prediction_path_gids` 追踪）
3. **doubt** — `doubt_mode == True`，系统检测到预测失效
4. **phase boundary** — `phase ∈ {CRITICAL, LOCKED}`，系统进入边界状态
5. **escalation** — `dτ/dt > 0`，τ 持续上升，系统在逼近而非适应

条件 (1)-(2) 是哥德尔边界的结构条件，(3)-(5) 是碰数的动力学检测。带 10 步冷却期防止重复触发。`_pengshu_events` 记录所有碰数事件。

**沉淀机制 (M1.5) — L2→L3 的桥梁：**

EE 的核心叙事：漂浮的话 → 绳结和壁画 → 新一代解读。沉淀是将内部呼吸变成外置化痕迹的操作。

三条件全部满足时帧"沉淀"：
1. **跨相位存活** — `survival_cycles >= 1`（至少存活一次 induction_clean）
2. **预测路径激活** — `activations >= 3`（系统真正使用了这个帧）
3. **结构稳定** — `is_meta_stable()`（未被合并的帧：存续即稳定；被合并的帧：|d(w)/dt| < 阈值）

GeruonFrame 携带 `survival_cycles`、`activations`、`precipitated` 三个追踪字段。`enrich_codex()` 调用 `precipitate()` 标记沉淀帧并写入 Codex。已验证：1500 步 24-cap Geruon 产生 10 个沉淀帧、8 个 codex 条目。

## BGM 已证明的 10 项发现

1. **桥呼吸** — SR-eff = I(Φ;X)/τ，学习/巩固动态切换
2. **两种开桥机制** — 密度(复调重叠)和闭合(结构完成)
3. **闭合检测** — 桥检测结构完成，不区分完成类型
4. **一级相变** — conf_threshold 在步 23 不连续跳到 1.0
5. **锚边界** — α > 0 必须成立，纯自指坍缩
6. **GI=4 帕累托最优** — 时间解耦增强层级分化 49%
7. **G0 是新颖性放大器** — 首次暴露活跃，随后消退
8. **空间反锁** — 64 单元网格无锁死；异质性主动抵制闭锁
9. **跨域同构** — 巴赫-细菌桥呼吸 r=+0.964
10. **离散/连续阈值** — 间隙密度 > 1/16 步

## EE 的六个种子方向

1. **γ₂ 运动方程** — 桥呼吸动力学
2. **自指周期量纲** — 时间新单位。1 自指周期 = 4 步。1 外置周期 = N 代
3. **签名层外置化** — 观察者身份作为跨代可转移对象
4. **G0 递归** — 桥观察桥观察桥，深度→任意时稳定性？
5. **信息重力** — g_info 作为张量：信息密度场 × 桥状态张量 = 外置化曲率
6. **锚边界** — 文明作为外部锚固（文字=跨时间锚，货币=跨交易锚，法律=跨判断锚）

## 三篇哲学论文概要

1. **本体论 (哥德尔定理的生成性转化):** 将哥德尔不完备定理从静态形式边界转化为自指系统永续演化的本体动力学——"稳态优化→量极临界→碰数死锁→范式相变"是万物认知演进的统一引擎。

2. **认识论 (认知的生成性复杂化):** 以大肠杆菌 lac 操纵子为起点，证明在环境不可约新异性下，贝叶斯校真必然驱动历史编码复杂度单调无界增长，最终涌现自指能力并触及哥德尔边界。区分内禀相变（框架内升级）与架构性相变（框架本身跃迁）。

3. **价值论 (逻辑即价值):** "应当"是生成之流为避免自我中断而发出的逻辑指令。引入 ⨀ (害) 算子作为价值判断的唯一原子运算。通过电车难题演示：价值判断本质是对具体 ⨀ 关系网络的识别与权衡。从细菌群体感应到人类道德法律，遵循同一生成性动力学。

## 开发约定

### 技术栈
- **纯 Python 3.8+ stdlib** — 零外部依赖，这是刻意的哲学选择
- 代码即研究笔记 — 无测试框架、无 CI/CD、无包管理
- `geme.py` 是唯一不可修改的内核（"GEME unchanged: always receives from environment"）

### 代码风格
- 中文注释和文档，英文代码标识符
- 科学常数用大写 (`DELTA`, `GAMMA`, `TAU`)
- 大量使用 `__slots__` 优化内存
- 随机种子固定以实现可复现性

### 实验设计模式
- 三阶段实验：培养(cultivation) → 扰动(perturbation) → 恢复(recovery)
- 关键指标：L4 密度、I(Φ;X) 互信息、L6 异常分数、pred_accuracy、doubt_mode、τ 自适应
- 对照实验：冷启动 vs 热启动、不同 α 值、不同网络拓扑

### 命名约定
- 文件名：小写下划线 (snake_case)
- 类名：大驼峰 (PascalCase)
- 内部/私有属性：下划线前缀 `_internal_attr`
- GEME 实例化后通过属性访问而非 getter/setter

### 编码纪律

以下四条约束研究过程中的编码行为，与上述项目特有约定叠加适用。核心权衡：严谨优先于速度。简单任务可凭判断灵活处理。

#### 1. 先想后写

**不假设、不掩盖困惑、暴露权衡。**

实现之前：
- 显式陈述假设。不确定就提问。
- 存在多种解读时，全部列出——不要沉默地选一个。
- 存在更简方案时说出来的，合理时回推。
- 遇到不清楚的地方，停下来。指出困惑所在。提问。

#### 2. 极简优先

**解决问题的最少代码。不写臆测性代码。**

- 不超出需求范围添加功能。
- 不为单次使用的代码创建抽象。
- 不做未被要求的"灵活性"或"可配置性"。
- 不为不可能发生的场景写错误处理。
- 200 行能压缩到 50 行就重写。

自问："资深工程师会觉得这是过度设计吗？" 若是，简化。

研究型任务的"需求"可能天然模糊（如 P0-P3），此时以最小可验证实验为目标，不提前铺架构。

#### 3. 精准修改

**只碰必须碰的。只清理自己制造的混乱。**

编辑既有代码时：
- 不顺手"改进"相邻代码、注释或格式。
- 不重构没坏的东西。
- 匹配已有风格，即使你的偏好不同。
- 发现无关的死代码，口头提及——不直接删除。

清理孤儿引用时：
- 移除因**你的改动**而不再使用的 import / 变量 / 函数。
- 不删除预先存在的死代码，除非被要求。

检验标准：每一行改动都应直接追溯到用户需求。

#### 4. 目标驱动执行

**定义成功判据。循环迭代直到验证通过。**

将任务转化为可验证目标——本项目无测试框架，验证方式为：
- "加验证" → "写出能触发该逻辑的输入样例，跑通并确认输出"
- "修 bug" → "构造能复现该 bug 的最小输入，修复后确认复现消失"
- "重构 X" → "重构前后用相同输入跑 `geme.py` 自检块，确认行为不变"

多步任务先列简要计划：
```
1. [步骤] → 验证: [检查项]
2. [步骤] → 验证: [检查项]
3. [步骤] → 验证: [检查项]
```

清晰的判据支持独立迭代。模糊的判据（"把它搞定"）需要反复澄清。
