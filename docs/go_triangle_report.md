## 一、实验设计

### 哲学基础

GEME 不验证预设定理。系统接收随机三角形命题流，通过**压缩效率**自发识别高频稳定模式。

```
输入: 随机 6 选 3 元素组合 → △≡△
  元素池: AB≡DE, BC≡EF, CA≡FD, ∠A≡∠D, ∠B≡∠E, ∠C≡∠F
  其中 40% 有效 (SAS/ASA/SSS/AAS)
  其中 60% 无效 (边角不对应)

系统行为: 相同模式合并 → weight 增长 → 压缩
输出: 若干高权重帧, 每个代表一个稳定模式
```

### 关键指标

```
压缩比 = 输入帧数 / 输出帧数
  200:1 → 400 个随机输入压缩为 2 个稳定帧
  压缩比越高 → 该模式在数据中越普适 → "定理"

weight = 合并次数
  w=203 → 该模式被观察到 203 次
  stress 从 ~0.8 降至 ~0.003  → "理解" → 涌现
```

### 实验矩阵

| 实验 | 噪声(有效比) | 记忆 | 轮次 | 输入帧 |
|------|-------------|------|------|--------|
| E1 | 中 (40%) | cap=8 | 20 | 400 |
| E2 | 低 (60%) | cap=10 | 30 | 600 |
| E3 | 高 (20%) | cap=8 | 30 | 600 |
| E4 | 中 (40%) | cap=12 | 30 | 600 |

## 二、实验结果

### E1: 中等噪声 (cap=8, 400帧)

```
压缩比: 200:1  stress: 0.003

帧 [0] w=203  (∠A≡∠D, ∠B≡∠E, CA≡FD) → △≡△  ← ANGLE-ANGLE-SIDE
帧 [1] w=198  (∠A≡∠D, AB≡DE, ∠B≡∠E) → △≡△  ← ANGLE-SIDE-ANGLE
```

**系统发现了两条有效定理。** 400 帧压缩至 2 帧，stress 降至接近 0。

### E2: 低噪声 (cap=10, 600帧)

```
压缩比: 150:1  stress: 0.207  inductions: 20

帧 [0] w=304  (∠C≡∠F, CA≡FD, AB≡DE) → △≡△
帧 [1] w=151  (∠C≡∠F, BC≡EF, ∠A≡∠D) → △≡△
帧 [2] w=114  (∠A≡∠D, ∠C≡∠F, AB≡DE) → △≡△
帧 [3] w=31   (AB≡DE, ∠A≡∠D, ∠C≡∠F) → △≡△
```

**四条稳定模式。** 诱导操作 20 次——系统周期性清理低权重帧。

### E3: 高噪声 (cap=8, 600帧)

```
压缩比: 150:1  stress: 0.287  inductions: 28

帧 [0] w=302  (∠C≡∠F, BC≡EF, ∠B≡∠E) → △≡△
帧 [1] w=170  (∠B≡∠E, BC≡EF, CA≡FD) → △≡△
帧 [2] w=94   (BC≡EF, ∠A≡∠D, CA≡FD) → △≡△
帧 [3] w=34   (∠C≡∠F, ∠B≡∠E, ∠A≡∠D) → △≡△  ← AAA (相似非全等)
```

**高噪声下仍涌现有效模式。** 帧 [3] 是 AAA（三个角），相似条件非全等——但系统只能识别"3→△"模式，无法区分全等/相似。这是当前签名匹配的局限性。

### E4: 大记忆 (cap=12, 600帧)

```
压缩比: 150:1  stress: 0.197  inductions: 18

帧 [0] w=272  (∠C≡∠F, AB≡DE, ∠A≡∠D) → △≡△
帧 [1] w=205  (AB≡DE, ∠A≡∠D, BC≡EF) → △≡△
帧 [2] w=77   (BC≡EF, CA≡FD, AB≡DE) → △≡△  ← SSS
帧 [3] w=46   (CA≡FD, ∠A≡∠D, ∠C≡∠F) → △≡△
```

**更大记忆 → 更多模式存活。** SSS (边边边) 以 w=77 涌现。

## 三、与 PA 阶段的对比

| 维度 | PA (形式算术) | GO (三角形全等) |
|------|-------------|----------------|
| 公理 | 7 条 (Q1-Q7) | 0 条 (无预设) |
| 目标 | 预设 add_comm | 自发发现模式 |
| 涌现度量 | weight ≥ median | 压缩比 |
| 输入 | 手工构造 | 随机生成 |
| 输出 | S2 或 S3 | 多帧权重排序 |

**GO 阶段更纯粹。** 无公理、无目标、完全随机流中的自发压缩。

## 四、局限性

```
1. 签名匹配过于宽松
    AAA 与 ASA 共享大量关键词 → 误判
    需要结构化的编辑距离匹配 (而非关键词重叠)

2. 无法区分全等与相似
    3角相等 → 相似, 非全等
    系统无"相似"概念, 只识别"3→△"框架

3. 压缩比 = 涌现强度 但无绝对阈值
    E1 压缩比 200:1 显然涌现
    E4 帧 [3] w=46, 压缩比贡献仅 77:1
    何时算"定理"仍需人工判定
```
