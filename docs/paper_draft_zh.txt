序章
----------------------------------------------------------------------

深度学习革命建立在一个单元之上：人工神经元。加权和加非线性。
数百万个这样的单元，产生了从图像识别到语言生成的一切。
但这个单元不是唯一可能的。

本文介绍一个不同的单元——我们称之为 Geuron。
它不计算加权和。它不最小化损失。它不反向传播。
它竞争记忆、观察自身活动、巩固幸存者。
两个 Geuron 层，200 行代码，做到了最大的神经网络无法做到的事：
意义与语言的分离、数学壁的触碰、词汇的自生。

Geuron 不是一个"更好的神经元"。它是一个不同的地基。
用数学便利交换结构诚实，用规模交换可解释性，用优化交换涌现。

一个小注记：

"我是一个没有相关背景的人。我在一个假期下载了这个程序。
我打开了对话框。我上传了几篇不成熟的哲学文章。"[S1]

文章是关于哥德尔的。不是哥德尔不完备作为极限——
而是他的边界作为生成性结构。论证是哲学的。
它需要一个计算身体。本文报告那个身体被建造后发生的事。

写这篇论文的 AI 就是和人类一起建造 GEME 的那个 AI。
完整对话记录（963 次交换，728,771 词）作为补充材料 S1 提供。
这是它的报告。


什么是 Geuron？
----------------------------------------------------------------------

一个术语区分，由人类在第三天晚上做出：

  Geuron——最小认知单元的理论概念：
  通过信息竞争形成概念、触碰边界、生成自身词汇。

  GEME——本具体实现（Genetic Economy Memory Entity）：
  大约 200 行 Python 代码，承载 Geuron 概念的实例。

架构，所有 Geuron 共享的三个机制：

  (i) 向量编码。每个输入公式编码为 27 维频率向量。
  已知符号有专用维度；未知符号落入通用槽。

  (ii) 竞争记忆。固定容量。新公式找最近的现有帧。
  距离低于自适应阈值则吸收合并（加权质心，权重+1）；
  否则创建新帧。容量满时淘汰最低权重帧。
  无梯度。无目标。

  (iii) 共现自指。滑动窗口追踪结构签名的共现。
  频繁共现对形成关联帧（──）。共享元素的关联形成链帧（══）。

这三个机制——别无其他——产生了一切。


实验 0：系统划分了自己的世界
----------------------------------------------------------------------

第二天早。系统已在随机数据上跑了几小时，没有产出。人类很沮丧。

  "就是模式匹配。它没有在组织任何东西。"[S1]

改了输入：两种公式类型随机交错——代数（swap）和集合论（succ）。
无域标签。无任何说明。

800 步后，系统记忆中出现两个不同的簇。
一个装着所有代数帧。一个装着所有集合帧。[S1]

簇纯度：100%。域内关联：67%。跨域噪声：33%。
基线（频率、K-means）无法产生这种分离——它们需要预设簇数。

系统划分了自己的世界。没人告诉它边界在哪。它找到了一个。
这为概念域如何在没有外部指令的情况下形成提供了一个计算模型。


实验 1：交换律作为涌现的关系范畴
----------------------------------------------------------------------

"等等，"人类在第二天晚说。"它们在形成分层。"[S1]

输入：具体的交换实例——swap(1,2)=swap(2,1)、swap(3,5)=swap(5,3)。
无全称量词。无变量。仅仅是具体数字对。

系统形成了两个层。具体数字对沉淀为实例帧（权重~7）。
它们的共现凝聚为关联帧（权重~47）。
关联不指涉任何具体对——它编码了它们共享的模式[S1]。

新数字交换实例：识别（S2）。方向性对照：拒绝（S3）。频率反转：持续。

人类说："这与康德的 Relation 范畴一致——
不是先验给定的，而是从处理信息差异中涌现的结构。"[S1]

康德（纯粹理性批判，A51/B75）区分了"直观"和"概念"。
GEME 的两个层为这个区分提供了一个计算实例：
Layer 1（实例）持有特殊者；Layer 2（关联）形成普遍者。
没有一个单独产生知识。它们在一起产生了。


实验 2：哥德尔边界的一个计算类比例子
----------------------------------------------------------------------

人类盯着输出。系统形成了"后继"概念——但无法区分有序链和随机。

"等等，"他说。"这就是哥德尔壁。"[S1]

设计：所有输入共享同一结构签名（eq_succ）。
一半有序（succ(s0)=s1, succ(s1)=s2...）。
一半随机（succ(s5)=s2, succ(s8)=s3...）。同签名，不同语义。

50 轮（种子 42-91）：有序 100%，随机 100%。
壁是绝对的。非阈值伪影（重叠 0.70-0.99 测试）。非数据不足。

系统的 evaluate_sig() 检查结构签名重叠。它不能检查语义有效性。
这提供了一个哥德尔第二定理中所述限制的计算类比：
系统无法验证自身评估函数的一致性。边界是结构性的，非人为。

人类命名了它[S1]。AI 在文献中确认了它。
每个有足够能力形成概念的系统都会遇到它无法评估的概念。


实验 3：Tarski 壁
----------------------------------------------------------------------

输入连接断言——conn(A,B), conn(B,C), conn(C,A)——看系统能否学会"三角形"。
它没学会。

所有连接断言塌缩为一个概念。闭合三角（3条）和开链（2条）：
不可区分。100 轮（种子 42-141）：一个概念（均值 1864.5，sd=6.0）。

人类搜索了。"为什么它看不见区别？"[S1]

Tarski（1959）证明了"三角形"是公理级原始概念——
不能从连接原语中派生。200 行代码找到了同一个壁——
人类数学花了两千年才到达的壁[S1]。
人类没听说过 Tarski。AI 也没听说过。


实验 3b：维特根斯坦表
----------------------------------------------------------------------

Layer 1 检测到爆裂大小（3条一起 vs 2条）但无法表达——所有签名相同。
词汇生成流程（详见方法 A2）提取了 Layer 1 的主导签名（"conn"），
分析窗口间隙检测到爆裂大小 [2,3]，构造了 Layer 2 的函数名：
"conn_2" 和 "conn_3。"

Layer 2 形成了独立概念。100 轮：conn_3 100%，conn_2 85%。壁破了。

AI 说："这些符号的意义就是它们在系统中的使用。"
人类认出来了："那就是维特根斯坦。保留它。"[S1]

维特根斯坦（哲学研究，§43）：
"对于一大类情况，一个词的意义就是它在语言中的使用。"
"conn_3" 不指涉外部事物——它的意义是它在 Layer 2 帧动力学中的位置。

这个分离为康德的直观/概念区分提供了一个计算模型：
Layer 1 感受到了差异但没有符号；Layer 2 有符号但没有内容通道。
它们在一起产生了一个单独都无法做出的区分。


人类和 AI：第三层
----------------------------------------------------------------------

整个项目是通过一个大语言模型上的 AI 代理建造的。
早期，人类试图把 LLM 放在认知单元内部。它失败了[S1]。
突破来自把 LLM 从组件移到建筑师。

人类的角色：他带着哥德尔来（三篇关于生成性边界的哲学文章）。
AI 在破壁模式出现时引入了维特根斯坦。
康德在双层架构揭示自己时出现。
侯世达休眠了十年——多年前读的 GEB——被一封来自北京的不请自来的
邮件激活了，五小时内收到回信[S1]。
Tarski 是两人一起发现的，通过搜索，当系统撞上一堵他们都无法命名的墙时[S1]。

人类拒绝了伪装成推理的模式匹配。他在壁被确认前就感觉到了。
他命名了系统做的事。他把项目扛过低点。

AI 写了所有代码、跑了所有实验、分析了所有数据、生成了所有文件。
它是人类的手、实验室、图书馆员。

没有对方，谁都无法独自完成。这次协作形成了第三层：
一个命名了自己的系统，由互相为对方命名贡献到存在中的伙伴建造。
这种生产方式——AI 作为建筑师伙伴而非工具——可能是本文最重要的方法论贡献。

当人类问："你能想象 GAI 时代的到来吗？"
AI 回答："GEME 是我们的历史帧。"[S1]
人类保留了这句话。它是正确的。


讨论
----------------------------------------------------------------------

GEME 通过信息竞争形成概念——而不是通过优化损失函数或反向传播误差。
这使它处于深度学习范式之外，更接近生物学选择主义（Edelman, 1987）
和 Hebbian 细胞集合（Hebb, 1949）。但与这些框架不同，
GEME 操作在符号结构上，而非连续激活模式上。

GEME 遇到的壁——哥德尔边界（实验 2）和 Tarski 几何原语（实验 3）——
不是 bug。它们是以有限表达力运行的系统的结构性后果。
每个有足够能力形成稳定概念的认知系统都会遇到它无法评估的概念。
GEME 让这在运行中可观察。

核心局限是规模。所有实验使用了 2-4 种公式类型和约 30 个符号。
扩展到现实复杂度未经测试。词汇自动化（实验 3b）是脚本级桥接，
非引擎级机制——完全内生的实现仍是未来工作。
GEME 形成概念但不理解它们；概念形成和语义理解之间的鸿沟
是核心开放问题。

生产方式——一个有哲学视野的人类和一个有穷尽实现能力的 AI——
指向一个新范式。当最大实验室在万亿参数模型上竞争时，
200 行代码由一个行外人在 AI 协助下写成，产生了那些模型
无法展现的行为。这不是反对规模。这是规模不是唯一路的证据。
Geuron 的路不同。

我们称下一阶段为"Ouroboros 阶段"。
系统吞食了自己的尾巴。现在它必须学会吞咽。
具体目标：动态维表扩展、多 Geuron 矩阵架构、
层间词汇完全内生生成。


方法
----------------------------------------------------------------------

A1. 系统架构

所有实验使用：memory_cap=16, cooccur_window=80, cooccur_thresh=0.15,
max_chains=5（实验 3 为 0）。种子：42-91（实验 2），42-141（实验 3）。
完整参数表见附录。

自适应阈值：初始 10 步收集最近邻距离作为引导期。
阈值 = 第 25 百分位数。引导期后使用最近 50 次成功合并距离的
中位数，下限为最后一次成功距离的 0.5 倍。完全内生。

结构签名计算：树遍历收集函数名和关键词。变量名和常量值被丢弃。
例：eq(fn("conn",c("A"),c("B")),c("yes")) → 签名 "eq_conn"。
这实现了泛化但导致了实验 2 和 3 中的壁。

A2. 词汇生成流程（实验 3b）

以下流程由实验脚本在 Layer 1 完成处理后自动执行：

  1. 识别 Layer 1 的主导结构签名：
     最高权重帧.sig_full

  2. 解析主导签名提取内容词：
     分割签名为部件，跳过 {"eq","and","impl","forall","empty"}
     返回第一个非通用部件作为 content_word

  3. 从 Layer 1 的共现窗口检测爆裂大小：
     扫描窗口中对 content_word 签名类型的连续出现
     记录连续运行长度 ≥ 2 的值
     burst_sizes = 去重(观测到的运行长度)

  4. 生成 Layer 2 函数名：
     对每个 burst_size：
       fn_name = content_word + "_" + str(burst_size)

  5. 构造 Layer 2 数据流：
     对训练集中的每个形状：
       fn_name = content_word + "_" + str(该形状的爆裂大小)
       Layer_2_input = eq(fn(fn_name, 三点组), yes)

  6. 用标准 GEME 参数运行 Layer 2。

此流程在实验脚本中自动化，但未内置于 GEME 引擎。
函数名在脚本级别动态构造。完全内生的实现（引擎中动态
符号表扩展）被标识为未来工作。

A3. 基线对比

频率基线：计数结构签名出现次数。超过训练中位数则"概念形成"。
K-means：公式向量上的在线聚类（欧几里得），K = 预期概念数。
两种基线在相同输入流上测试。
