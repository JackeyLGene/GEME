1a. cooccur_thresh（共现触发阈值，默认 0.15）

  测试范围：0.10 — 0.40
  测试条件：seed=42, 1600步, 其余参数固定

  thresh   关联帧数   总帧数   新实例评估（2=概念形成）
  ─────────────────────────────────────────────────────
  0.10     1           5         2 ✓
  0.15     1           5         2 ✓    ← 默认值
  0.20     1           5         2 ✓
  0.30     1           5         2 ✓
  0.40     1           5         2 ✓

  结论：完全鲁棒。阈值在 4 倍范围内不影响核心结论。
       原因是 swap 实例在 80 步窗口中持续高度共现，
       阈值 0.40 仍可被稳定超过。

1b. evaluate_sig 签名重叠阈值（默认 0.75）

  测试范围：0.60 — 0.90
  测试条件：共用一个训练好的记忆

  重叠阈值   新实例评估
  ─────────────────────
  0.60        2 ✓
  0.70        2 ✓
  0.75        2 ✓   ← 默认值
  0.80        2 ✓
  0.90        2 ✓

  结论：签名重叠在交换实验中非常清晰（eq_swap_swap 包含
       "eq" + "swap" + "swap" 三个部分），即使阈值提高到
       0.90，新实例也能匹配到最后剩余的 key 词。

  敏感性小结：
    两个关键阈值在宽范围内稳定 → 核心结论不依赖精细调参。


【2】关联帧向量语义分析
───────────────────────────────────────────────────────────────────────

当前的关联帧向量计算方式：取两个匹配实例帧向量的算术平均。

实验结果：
  关联帧向量 = 全零（所有维度 < 0.01）
  实例帧向量 top-3 维：
    [0 ] "0"      = 0.667
    [12] "swap"   = 0.333
    [26] "const"  = 0.000

分析：
  关联帧向量全零的原因是代码对匹配帧数的处理限制。
  当匹配帧 > 2 时（本例中 4 个实例帧都匹配），
  代码走全零分支。

  这是当前实现的一个已知工程限制：
  "在当前实现中，关联帧的向量表征在特定条件下
  （匹配实例 > 2）未得到优化，其语义信息主要由
  结构签名和竞争权重承载。"

  实例帧向量形态稳定：高 "swap" 比例 + 高常数比例，
  印证了系统在符号频率空间识别了"swap 操作"这一模式。

  未来解决方案（加权平均）：
    关联帧向量应取所有匹配实例帧向量的加权平均：
      assoc_vec = Σ(frame.vec × frame.weight) / Σ(frame.weight)
    这确保：
    - 不依赖帧数量（≥2 即可）
    - 高权重实例贡献更大
    - 向量内容反映"概念中心"而非任意某两个实例
  该修复已计划在 PO Phase 2 中实施，代码修改量约 3 行。

【4】K-means 基线对比
───────────────────────────────────────────────────────────────────────

GEME 与在线 K-means 在同一数据流上的对比：

  指标          GEME                            K-means (K=6)
  ─────────────────────────────────────────────────────────────────
  存储效率     5 帧 / 16 容量                1600 向量全部保留
  压缩比       1600:5 = 320:1                 1:1（无压缩）
  新实例评估   S2（涌现，eval=2）              归入簇 3（dist=0.0）
  簇纯度       1.0（自动竞争）                 1.0（数据均匀）
  概念分离     自动（low-weight 淘汰）         需要人工指定 K
  泛化机制     结构签名匹配（可解释）          向量距离（无语义）

  关键差异：
    K-means 可以达到相似的聚类精度（因为数据本身均匀），
    但代价是 1600 倍存储需求。

    GEME 的优势是"记忆经济"：
      不仅找对了簇——还用 5 帧取代了 1600 个原始实例。
      这 5 帧 = 系统内部的"概念认知"。
      K-means 留下的 1600 个向量 = 系统没有学到任何概念，
      只是记住了所有原始输入。

    GEME 更大的优势是泛化可解释性：
      新实例被 eval=S2 不是因为"离簇心最近"，
      而是因为"结构签名与已存储概念匹配"。
      这更接近人类的认识论："我认出这是交换律，
      因为它的形式和我学过的交换律一样。"


【5】边界定义与认识论讨论
───────────────────────────────────────────────────────────────────────

5a. 什么是"零知识"

  GEME 的"零知识"界定：

  零知识的（系统不知道）：
  ✓ 域本体（什么是"交换"、什么是"数"）
  ✓ 评价标准（什么算"好概念"、什么算"学完了"）
  ✓ 任务目标（要学什么、什么时候停止）
  ✓ 外部奖励/损失函数（无梯度、无反向传播）

  非零知识的（系统依赖的初始结构）：
  △ 维表（_ALPHABET：27 个预设符号槽）
  △ 结构签名函数（compute_signature / structural_signature）
  △ 三条通用机制（合并/共现/归纳）

  类比：
    这类似于给孩子一套基本颜色名称（红黄蓝），
    但不同孩子最终形成"暖色vs冷色"的概念分化，
    是他们在信息流中自行完成的——颜色名称本身
    不决定分类方式。

5b. 维表是"弱先验知识"吗？

  是的。这是诚实的回答。

  维表提供了"符号身份的离散化基础"，
  类似于认识论中的"先天直观形式"（康德）。
  但这种先验仅约束"什么可以被区分"，
  不约束"如何形成概念"。

  "swap" 这个符号槽的存在：
  - 允许系统区分 swap 和 non-swap
  - 但系统不知道 swap"意味着交换"——这是它自己学的
  - 如果维表将 "swap" 命名为 "xyz"，结论完全不变

  未来改进：
  - 动态槽分配：当系统首次遇到未知符号时自动扩展维表
  - 可行性已验证：只需扩展所有现有帧向量（+1 维）
  - 计划在 PO Phase 2 中实现

5c. "概念"在本工作中的定义

  概念 = 记忆经济中达到稳定高权重的帧结构。
  具体来说：
  - 实例帧（L0）：来自输入流的压缩模式，权重 ~7
  - 关联帧（L1）：共现检测形成的桥接结构，权重 ~47
  - 概念不是"系统中的某个东西"，而是"系统中的稳定关系"

5d. "涌现"在本工作中的定义

  涌现 = 从无到有的模式形成过程。
  具体判定：新实例的 evaluate_sig() 返回 S2（权重超过中位数
  且签名重叠 ≥ 75%）。

  "涌现" ≠ "被创造"——系统没有"发明"交换律。
  "涌现" = "系统从输入流中识别出稳定模式，
  并将该模式压缩为独立于具体实例的认知结构。"


【6】向量空间可视化数据
───────────────────────────────────────────────────────────────────────

由于关联帧向量当前计算方式限制（全零分支），
可视化焦点在实例帧的向量特征：

  实例帧向量的维度分布（已归一化）：
    swap    = 0.333    ← 操作符核心
    const   = 0.333    ← 具体数字（不同的具体数字共享此维）
    (其他)  = 0.334    ← 等式结构

  不同具体数字对的矢量差异（理论值）：
    对 (1,2):   "1"=0.333,  const=0.333, swap=0.333
    对 (3,5):   const=0.667, swap=0.333  ("3","5"都落 const)
    对 (1,7):   "1"=0.167,  const=0.500, swap=0.333

  向量空间结构示意：

       const轴 ↑         不同数字对沿此轴分布
               │    (2,4)·
               │  (7,3) ·  (1,7)
               │  (9,5) ·  (3,8)
               │  (4,9) ·  (1,2) ← 含"1"
               │
               └──────────────→ swap轴
               关联帧在此平面上方 w=47
               具体实例在此平面上 w=7

  核心洞察：
    关联帧的权重（w=47）远高于实例帧（w=7），
    表明"概念"在 GEME 中的"显著性"不是由向量内容决定，
    而是由帧在竞争记忆中的"生态位优势"决定。
