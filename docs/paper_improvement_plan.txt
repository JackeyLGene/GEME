1. 【致命】词汇自生环节的透明度
───────────────────────────────────────────────────────────────────────

问题：稿件声称是"自动的"，但审稿人无法区分是运行时自动还是
      设计者离线分析后手动硬编码。这是"意义内生"主张的生死线。

行动：
  A. 在 Methods 中新增独立小节 "Vocabulary Self-Generation Pipeline"
  B. 逐条回答三个问题：
     - 窗口间隙分析：是 Memory 内置的统计扫描（observe 中已存在）
       运行时持续执行，非离线脚本
     - 函数名动态创建：conn_2/conn_3 是在实验脚本中根据提取结果
       动态构造的字符串，非预定义符号。不是 GEME 引擎自动完成的
       ——是实验脚本在 GEME 的 Memory 数据上做的后处理
     - 需在 Limitation 中诚实说明：
       当前词汇注入通道是实验脚本→Layer 2 的显式传递。
       完全内生的版本需要 GEME 引擎自身支持动态维表扩展。
  C. 建议措辞调整：
     "System automatically extracted..." 
     → "Layer 1's memory state was queried by the experiment script,
        which extracted the dominant signature's content word and
        burst statistics. These were then used to construct Layer 2's
        function names — a manually implemented bridge that
        demonstrates the principle of self-generated vocabulary."


2. 【严重】基线对比深度不足
───────────────────────────────────────────────────────────────────────

问题：当前对比停留在"有/无"定性层面。Nature 需要量化优势。

行动：
  A. 增加基线：在线Hebbian学习 + 简单RNN（或序列预测器）
     Hebbian: 共现矩阵→权重更新，无合并/淘汰
     RNN: 序列预测，看是否形成概念
  B. 三个量化维度（各基线 + GEME 对比表）：
     - 数据效率：达到稳定概念所需的最少实例数
     - 噪声鲁棒性：输入流中加入 10%/30%/50% 噪声时概念稳定性
     - 可解释性指标：帧内容 vs 聚类中心的语义清晰度评分
  C. 预期结论形式：
     "GEME achieves concept stability at 320:1 compression
      (vs 1:1 for all baselines), with noise tolerance 2-3x
      higher than Hebbian learning, and produces interpretable
      frame hierarchies that baselines cannot match."

───────────────────────────────────────────────────────────────────────

问题："This IS Godel's Second Theorem" 等声称过于强硬。

行动：

     "This IS Godel's Second Theorem"

     "Kant's category computed"

     - 明确说明"计算建模"与"哲学理论"的关系

  C. 在 Introduction 或 Discussion 中放入"精确/防御性"声明段落
