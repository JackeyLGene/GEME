# GEME V6 — 深度分析

## 1. S4 权重分布效应

```
WITH S4 (50个随机运行):
  median_w 稳定在 10.0
  S4_w 恒定 6.0 (单帧, 从不被淘汰)
  swap_w = 0.0 (swap 帧在竞争中失败)
  noise_w 持续增长: 22→129 (噪声累积)
  
NO S4:
  swap_w 持续增长: 10→83 (swap 帧主导)
  median_w: 8→83 (被 swap 拉高)
  noise_w: 23→119 (噪声也被积累)

核心发现:
  S4 帧 weight=6.0 固定在记忆中
  但不帮助 swap 模式——swap_w 在有 S4 时为 0
  S4 实际上和 swap 帧竞争记忆槽
  S4 是"干扰物"而非"催化剂"

限制:
  所有运行使用同一随机种子
  样本内差异小
  S4 的效果强烈取决于具体随机序列
```

## 2. 效应量

```
Cohen's d 无法在零方差条件下计算（全同运行）。
需要改进: 每次运行使用不同随机种子, 引入随机差异。

初步结论: S4 的效果不是单调性"帮助"——
  在某些运行中 S4 压过了 swap (S4 占槽位)
  在某些运行中 S4 无影响 (swap 权重已足够高)
  S4 是"干扰项"而非"必要项"
```

## 3. 梯度噪声

```
s0=3   with=0%   no=0%    (数据太少, 无模式)
s0=5   with=93%  no=100%  (S4 轻微拖累)
s0=8   with=0%   no=100%  (S4 拖累严重!)
s0=10  with=0%   no=100%  (S4 拖累严重)
s0=15  with=0%   no=0%    (噪声掩盖一切)
s0=20  with=0%   no=0%    (噪声掩盖一切)

S4 在无噪声时反而妨碍涌现:
  无 S4 时 swap 权重线性增长 (每轮 +10)
  有 S4 时 S4 占了一个槽位, 且 weight=6.0
  → S4 的实际效果是"减少 swap 可用的记忆槽"
  → 当槽位紧张时, S4 伤害 swap, 非帮助
```

## 4. Stress 阈值

```
0.3-0.9 在 10 轮测试中均无涌现
  不是因为阈值不合适
  是因为当前实验参数下 (s0=10, cap=8) 涌现被噪声压制
  
阈值 0.6 的经验来源:
  cap=8, 填充率 ~0.75 时触发
  约每 2-3 轮一次归纳
  更小阈值 (0.3) 导致每轮归纳 → 记忆无稳定期
  更大阈值 (0.9) 导致记忆满负荷不清理
```

## 5. 课程学习

```
P1: 纯噪声 → median=90, 无模式 (S3)
P2: +S1 → median=48, 略有稀释
P3: +S2(交换实例) → median=66, swap 权重增长
P4: +S4 → median=39, S4_w=11, swap 稳定但不足中位数

最终: S3。模式未达到涌现阈值。

失败原因:
  课程前段噪声积累了大量帧 (noise_w=119)
  即使 S2/S4 加入, 噪声权重仍占主导
  中位数被噪声拉高
  模式帧无法跨越
```

## 6. 签名是概念筐

```
compute_signature 生成分类标签:
  "equation_add_swap" = 预先定义的模式类型
  系统不发明"交换律"——它匹配输入到已有类型中

限制:
  系统不产生新模式类型
  只识别已有类型的新实例
  涌现 = 类型实例达到生存阈值, 非创造新抽象
```

## 整体诊断

```
当前 V6 的核心问题:
  1. 签名 = 预编码 → 无新概念生成
  2. S4 效果不稳定 → 在某些条件下帮助, 在某些条件下伤害
  3. 噪声管理差 → 噪声权重线性累积, 压模式
  4. 权重增长无上限 → 旧帧权重无限膨胀, 新帧难进入

急需的改进:
  1. 权重正则化 (上限或衰减)
  2. S4 的语义交互, 非仅权重竞争
  3. 签名空间的动态扩展
  4. 噪声稀释机制 (非 bin 积累)
```
