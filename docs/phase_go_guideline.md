# GEME — 几何阶段 (GO) 初期工作指引

**日期**: 2026-05-03  
**前阶段**: PA (形式算术) — 已完成  
**本阶段**: GO (几何定理层) — 启动指引

---

## 一、PA 阶段的核心遗产

PA 证明了：

```
1. 系统能从随机实例流中发现模式 (S2 涌现)
2. 自指信号 (S4) 可作为结构发现的催化剂
3. 归纳法 = 选择性衰减 + 权重排序, 无需外部引擎
4. 零样本泛化: 系统学到了"交换律"概念, 非记忆数字
```

**但 PA 没做到：**

```
- 未展示链式推理 (只做了单层模式识别)
- 未展示定理证明 (evaluate 返回 S2 ≠ 证明了定理)
- 签名是预编码, 非动态生成
- 实验报告是最终输出, 无过程可视化
```

## 二、GO 阶段三个子实验的有效性评估

### 实验 1：三角公式自推演

```
目标: 从 sin² + cos² = 1 等基础恒等式出发,
     内生涌现 tan² + 1 = sec² 等衍生公式

可行: ✓
  三角恒等式 = 符号公式
  → 可编码为 W-table 向量
  → 可用编辑距离匹配
  → 权重竞争决定哪条公式"活下来"

前提条件:
  1. 需要三角函数的符号编码 (sin, cos, tan, sec)
  2. 需要三角公理预加载 (类似 Q 公理)
  3. evaluate 需能验证三角恒等式 (类似 _semantic_truth)

PA 因果链复刻: ✓
  S4 (复杂恒等式作为自指) → 记忆压力 → 模式发现

风险: 低
  三角恒等式在结构上比交换律更复杂, 
  但本质仍是"签名匹配"问题, 无新机制挑战

与 PA 本质区别: 无。
  三角实验是 PA 场景的几何平移, 非质的飞跃
```

### 实验 2：欧式定理链式证明

```
目标: 从欧式几何 5 公理出发,
     内生涌现平行线定理 → 内角和定理 → 勾股定理

可行: ⚠️ 有前提条件
  PA 的 evaluate 只做"有无模式"检查
  链式证明需要 "A → B → C" 的推理可追溯

前提条件:
  1. 推导链记录 (不是只记录最终 evaluate 结果)
  2. 中间引理的权重管理 (引理和定理在同一记忆空间?)
  3. 证明验证 (定理 X 需要步骤 1→2→3, 非一步到位)

与 PA 质的飞跃: ✓ (这才是 GO 的核心价值)
  PA: 模式发现 (归纳)
  GO: 链式证明 (演绎)
  
  演绎 = 使用已有规则推导新规则
  这是从 "模式分类" 到 "推理论证" 的跨越

风险: 中高
  链式证明需要证明过程的显式追踪
  当前 GEME 架构无证明追踪机制
  需要设计: ProofChain 类 / ProofStep 记录
```

### 实验 3：碰数式几何定理发现

```
目标: 无预设目标定理, 仅输入随机几何实例,
     系统通过"碰数"自发涌现并证明新的几何定理

可行: ⚠️ 需要重大架构突破
  这等价于 PA 阶段中"无 S2 实例仅靠 S4 自指产生涌现"
  当时 PA 未完全成功 (S4 = 催化剂, 不创造新模式)

与 Lean/Coq 的区别:
  Lean/Coq: 用户给出定理, 系统验证证明
  GEME:     系统自己发现定理, 自己构造证明
  
  如果 GO 实验 3 成功:
  → GEME 是"证明发明器", 非"证明验证器"
  → 这是 AI4Math 社区目前做不到的

前提条件:
  1. 需要实验 2 的链式机制成功
  2. 需要证明空间的可搜索性 (不是验证, 是搜索)
  3. 需要"碰数" = 随机实例组合 → 模式涌现 → 定理识别

风险: 高
  当前全球无成功的"自发定理发现"系统
  GEME 如果成功是首次, 但如果失败也是预期内
```

## 三、新交互层——证明过程可视化

PA 阶段用户看到的是：

```
实验报告: 涌现率 35%, Cohen's d=0.8, p=0.017
```

GO 阶段需要看到：

```
证明追踪:
  Step 1: 输入平行线公理               → weight += 1
  Step 2: 输入角度互补关系               → merge → 组合签名
  Step 3: 内角和引理涌现                → evaluate(S) = S2
  Step 4: 证明链: 公理 → 引理 → 定理   → [可视化链]
```

需要的组件：

```
1. ProofChain 类:
   - steps: [(rule, input, output, weight)]
   - merge 时记录证明历史
   - 链可视化: "A → B → C"

2. ProofViewer 模块:
   - 给定 evaluate(formula) = S2
   - 追溯: 哪些记忆帧共同支撑了这个结论
   - 显示权重路径

3. 时序图:
   - x 轴: 时间/帧数
   - y 轴: 记忆权重
   - 标记: 涌现时刻 / 链合点
```

## 五、总体评估

```
GO 三分实验方案: 有效, 建议按顺序执行

优先级:
  实验 1 (三角自推演) → 建立信心, 验证机制
  → 实验 2 (欧式链式证明) → 质的突破, 核心价值
  → 实验 3 (碰数发现) → 长远目标, 高回报高风险

不推荐跳过:
  实验 1 → 实验 2 的跨越太大
  无链式证明机制直接做定理发现 = 撞运气

建议启动:
  从实验 1 开始, 同步设计 ProofChain
  1-2 周完成 3 个实验, 产出可视化证明报告
```
