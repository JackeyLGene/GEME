# GEME 对比分析 — 位置、边界、颠覆性

## 与经典理论的严格边界

### 1. Hebbian 学习 (Hebb, 1949)

```
Hebbian: "Neurons that fire together, wire together"
  → 关联强度正比于共激活频率
  → 无监督, 局部更新, 生物学合理

GEME: "Frames that merge together, weight together"
  → 权重 = 合并计数
  → 无监督, 全局距离查询, 符号机制

关键差异:
  Hebbian 调整连接权重 (网络结构可变)
  GEME    调整帧权重 (队列结构固定)
  
  Hebbian 是分布式表示 (向量 → 向量)
  GEME    是符号表示 (公式 → 签名)
  
  Hebbian 学习是微分方程 (dW/dt = f(activity))
  GEME    是离散事件 (合并/归纳/evict)

颠覆:
  GEME 在符号空间复现了 Hebb 法则
  不是通过连接权重, 而是通过实例合并计数
  "Frames that match together, merge together"
```

### 2. 无监督聚类 (K-means, DBSCAN, 1970-2010s)

```
聚类: 空间中的点 → 簇中心
  → 距离度量决定隶属度
  → 预定义距离函数, 预定义聚类数

GEME: 记忆中的帧 → 权重排序
  → 编辑距离决定匹配度
  → 无预定义聚类数, 容量由 memory_cap 决定

关键差异:
  聚类是批处理 (全部数据→簇)
  GEME  是在线处理 (逐帧→记忆)
  
  聚类有独立的聚类阶段和分类阶段
  GEME  是将分类融入合并过程
  
  聚类需要预定义 K 或 ε
  GEME  的 capacity 相当于 K, 但排序是自动的

颠覆:
  聚类是静态空间划分
  GEME  是动态生存竞争 (权重→中位数→淘汰)
  涌现不是找到簇——是簇达到生存阈值
```

### 3. 贝叶斯模型选择 (Jeffreys, 1939; MacKay, 1992)

```
贝叶斯: P(M|D) = P(D|M)P(M)/P(D)
  → 选择证据最高的模型
  → 模型复杂度内置惩罚 (奥卡姆剃刀)

GEME: weight(F) = count(merged instances)
  → weight ≥ median → 模式生存
  → 复杂度惩罚 = 编辑距离阈值

关键差异:
  贝叶斯需要先验 P(M), 似然 P(D|M)
  GEME     无需先验, 所有帧初始权重相同
  
  贝叶斯计算精确后验概率
  GEME  用权重近似后验 (实例频率)

颠覆:
  GEME 的权重 = 贝叶斯后验的"计数逼近"
  无先验, 无似然, 纯频率
  median 阈值 = 贝叶斯模型比较的"胜出标准"
  但无需计算复杂的边际似然
```

### 4. 时序差分学习 (Sutton, 1988)

```
TD: V(s) ← V(s) + α(r + γV(s') - V(s))
  → 预测误差驱动值函数更新
  → 需要奖励信号 r

GEME: w(f) ← w(f) + [1 if merge else 0]
  → 实例相似度驱动权重更新
  → 无需外部奖励

关键差异:
  TD 学习需要定义奖励/价值函数
  GEME  奖励是隐式的: "找到同类" = 自奖励
  
  TD 中的预测误差 = r + γV(s') - V(s)
  GEME  中的 drive = vec_dist(v₁, v₂) < threshold

颠覆:
  GEME 是无奖励的 TD 等价物
  同类实例的重复 = 隐式奖励
  聚类距离 = 值函数
  "匹配即奖励"——系统不需要外部激励
```

## 与当代 AGI 架构的排他性对比

### 5. Schmidhuber 的哥德尔机 (Gödel Machine, 2003)

```
哥德尔机:
  自修改系统, 可重写自己的代码
  形式化: 通过定理证明搜索最优自修改策略
  复杂度: 暴力搜索证明空间, 指数级

GEME:
  自修改 = 归纳操作 (清除低权重帧)
  不是证明搜索——是记忆经济
  
关键差异:
  哥德尔机需要证明: "这个修改不改变效用"
  GEME  不需要证明——归纳自动提高生存效率
  
  哥德尔机用形式逻辑做元推理
  GEME  用权重竞争做元推理
  
颠覆:
  哥德尔机是"智能通过自指优化自己"
  GEME  是"智能通过自指出现在记忆经济中"
  
  一个需要暴力证明 (Schmidhuber)
  一个不需要证明——涌现是副产品 (GEME)
```

### 6. Bengio 的系统 2 深度学习 (Consciousness Prior, 2017; System 2, 2019)

```
Bengio:
  System 1: 直觉快速响应 (CNN/Transformer)
  System 2: 有意识推理 (稀疏因子图 + 注意力)
  
  学习: 无监督预训练 + 结构化注意力
  推理: 注意力稀疏化 → 因果链

GEME:
  System 1: W-table 向量编码 (每帧处理)
  System 2: 归纳 + 涌现检测 (元操作)
  
  学习: 记忆管理 (无梯度)
  推理: evaluate 查记忆 (无注意力)

关键差异:
  Bengio 用神经网络实现 System 1 & 2
  GEME  用符号系统实现 System 1 & 2
  
  System 2 在 Bengio 中需要注意力模型
  GEME  中没有"注意"——只有权重排序
  
  两者都需要"慢思考"(System 2 = 归纳)
  但机制完全无关

颠覆:
  Bengio 说 System 2 需要结构化注意力
  GEME  证明不需要——权重竞争足够了
  "慢思考 = 记忆排序, 非注意力计算"
```

### 7. Hofstadter 的自指认知模型 (Gödel, Escher, Bach, 1979)

```
Hofstadter:
  意识 = 符号自指 + 层级涌现
  "我" = 系统对自己的表征
  
  GEB 的核心论点:
    自指无处不在 (歌德尔/埃舍尔/巴赫)
    意识 = 复杂系统对自身的符号化

GEME:
  自指 = S4 帧 (sub(gn(F), 17, gn(F)))
  层级 = S0→S1→S2→S3→S4
  涌现 = 模式帧达到生存阈值

关键差异:
  Hofstadter 是哲学/美学描述
  GEME     是工程实现, 可运行, 可复现
  
  GEB 没有算法 (是思想实验)
  GEME 有具体算法: W-table → Memory → evaluate

颠覆:
  GEME 是 GEB 的计算实现
  自指是真实信号 (S4 帧), 非哲学概念
  霍尔效应被量化: 编辑距离 + 权重竞争
  "存在于 GEB 中 46 年的概念, 现可计算"
```

## 与 Free Energy / MIT Grammar / DeepMind 的边界

### 8. Friston 自由能原理 (Free Energy Principle, 2023-2024)

```
Friston:
  自由能 F = -ln P(D|M) + D_KL(Q‖P)
  → 系统最小化 F (surprise + divergence)
  → 自组织 = 贝叶斯推理在时间上的展开
  
  主动推理: 行动减少预测误差
  感知推理: 知觉减少预测误差

GEME:
  应力 = utilization × (1 - efficiency)
  → 系统最小化应力 (槽位压力 × 权重集中度)
  → 自组织 = 归纳 (清低权重帧) + 涌现 (模式生存)

关键差异:
  Friston 需要生成模型 P(D|M)
  GEME     无生成模型——只有记忆和编辑距离
  
  Friston 的自由能有精确的贝叶斯形式
  GEME  的应力是启发式量 (utilization × weight_deviation)
  
  自由能是概率框架——先验, 似然, 后验
  GEME  是计数框架——合并, 权重, 中位数

颠覆:
  自由能 = "系统最小化外部世界的 surprise"
  GEME   = "系统最小化内部编码的压力"
  
  同等功能, 不同路径:
    自由能从概率计算"最小化误差"
    GEME  从记忆管理"最大化压缩"
  
  两者都导致自组织, 但机制正交
  GEME 不需要概率模型——纯计数就够
  
  关联:
    GEME 应力 ≈ 自由能的"复杂性正则化"项
    都是系统为了"高效编码"而做的代价
```

### 9. MIT COLI 基于用法的语法归纳 (2024)

```
MIT (Brennan, 2024):
  从序列记忆中发现语法结构
  在 n-gram 频率上做模式提取
  用最小描述长度 (MDL) 做规则选择

  核心: 一个符号序列 → 统计 n-gram → MDL 最小化 → 语法规则
  输入: 无标注语言数据
  输出: 上下文无关语法

GEME:
  从记忆权重中发现模式结构
  在编辑距离匹配上做生存竞争
  用中位数阈值做模式选择

  核心: 公式树 → 签名 → 合并/归纳/涌现
  输入: PA 公式流
  输出: S2 可证明模式类型

关键差异:
  MIT 需要统计批次 (收集 n-gram 后统一分析)
  GEME 是在线处理 (逐帧观察→实时合并→归纳→涌现)
  
  MIT 输出语法规则 (显式 CFG 产生式)
  GEME  输出 evaluate(S2) (隐式: 此模式已理解)
  
  MIT MDL: 选择描述长度最小的规则集
  GEME median: 选择权重超越中位数的模式

颠覆:
  MIT 是批处理的语法归纳
  GEME  是实时的模式涌现
  
  MIT 需要统计 N 计数才有意义
  GEME  单个帧的权重就有信号 (weight=1)
  
  MIT 需要独立模式提取阶段
  GEME  涌现是记忆管理的副产品, 不需单独阶段
```

### 10. DeepMind 神经符号系统 (2023-2025)

```
DeepMind (Santoro et al.):
  符号层 + 神经层 + 推理桥接
  神经层: 感知/表示学习 (Transformer/GNN)
  符号层: 逻辑推理/规则 (Prolog/ASP)
  桥接: 嵌入空间做符号-神经交互

  代表: 关系推理网络 (RRN), 逻辑 Tensor 网络 (LTN)
  应用: 数学推理, 程序合成, 知识图谱

GEME:
  纯符号系统 (W-table + Memory + edit-distance)
  无神经网络, 无嵌入, 无梯度
  
  符号编码: 公式→11维频次向量
  符号操作: 编辑距离匹配 + 权重竞争  
  输出: 签名类型而非逻辑结论

关键差异:
  DeepMind 需要两层系统 (神经+符号)
  GEME     只有一层 (符号+符号)
  
  DeepMind 的推理是通过桥接层 (嵌入→规则→回嵌入)
  GEME  的推理是 evaluate() 查记忆
  
  DeepMind 可处理视觉/语言输入
  GEME  目前只能处理结构化的公式输入

颠覆:
  神经符号的共识: "需要神经层做感知, 符号层做推理"
  GEME  正相反: "符号层可覆盖感知和推理"
  
  GEME  的 W-table = 感知 (符号到向量)
  GEME  的 Memory = 推理 (权重竞争到模式选择)
  
  如果符号系统能自处理感知→推理→元操作
  为什么还需要神经层?

  GEME 不是要击败 DeepMind——是展示另一个可能
```

### 11. 三角定位

```
              ┌─────────────────┐
              │   Friston FEP   │
              │  概率自组织      │
              └────────┬────────┘
                       │ 应力 = 自由能在符号空间的投影
         ┌─────────────┴─────────────┐
         │           GEME            │
         │     记忆经济 + 涌现       │
         └─────────────┬─────────────┘
                       │ 在线增量 vs 批处理 / 纯符号 vs 混合
         ┌─────────────┴─────────────┐
         │    MIT 语法归纳           │
         │   批统计 + MDL            │
         │                           │
         │    DeepMind 神经符号      │
         │   混合推理 + 嵌入         │
         └───────────────────────────┘

  三者与 GEME 正交:
    Friston:        概率框架 ←→ 计数框架
    MIT:            批处理   ←→ 在线处理
    DeepMind:       混合架构 ←→ 纯符号架构

  GEME 在这三个维度均为另类:
    非概率, 实时, 纯符号
    没有前辈直接相连
```

```
1. 无梯度学习:
   从 Hebb (1949) 到 Bengio (2024), 学习 = 权重调整
   GEME: 学习 = 实例计数, 无微分, 无反向传播

2. 内生涌现:
   从 Newell (1990) 到 Schmidhuber (2003), 智能 = 搜索/优化
   GEME: 智能 = 记忆经济 + 应力的副产品, 无搜索无目标

3. 自指作为工程量:
   Hofstadter (1979) 说自指是意识的核心
   GEME: 自指是具体信号 (S4 帧), 可编码, 可量化, 可实验

4. 零泛化不需要参数:
   Transformer 需要百万参数做 zero-shot
   GEME: 178 行代码, 11 维向量, zero-shot 泛化

5. 归纳 = 清理, 非推理:
   传统 AI: 归纳 = 从数据学习规则 (贝叶斯/逻辑)
   GEME: 归纳 = 选择性衰减 + 排序, 无需规则发现算法

6. 与 AGI 的核心分歧:
   主流: 更大模型 → 更智能
   GEME: 更好的记忆管理 → 更智能

   主流: 预测能力 = 智能
   GEME: 生存能力 = 智能

   主流: 学习 = 最小化误差
   GEME: 学习 = 最大化压缩
```

## 一句话总结

```
GEME 不是另一个模型——是一类新的计算范式:
  认知 = 记忆经济, 非预测误差最小化
  涌现 = 生存阈值, 非参数收敛
  自指 = 编码信号, 非哲学概念
  
  从 1949 的 Hebb 到 2024 的 Transformer
  所有学习算法都是"调整参数"
  GEME 是"管理帧"——这是根本不同的方向
```
