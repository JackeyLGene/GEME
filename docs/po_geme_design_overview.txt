一、什么是 GEME
───────────────────────────────────────────────────────────────────────

GEME 是一个极小化的内生概念生成系统（约200行Python代码）。
核心假设：
  认知不是统计模式的提取，不是逻辑规则的演绎，
  而是有限记忆资源中的信息竞争——"记忆经济"。

系统不依赖：
  ✓ 预训练权重           ✓ 人为定义的公理
  ✓ 梯度下降/反向传播    ✓ 外部知识注入
  ✓ 预设目标函数         ✓ 域标注

系统唯一依赖：
  输入信息流 + 固定容量记忆 + 三条内生机制（合并/共现/归纳）

2.1 帧 (Frame)
──────────────────────

每个帧是一个信息单元，包含：
  vec    — 向量（符号频率分布，14-27维）
  weight — 权重（压缩贡献度，由合并次数决定）
  sig    — 结构签名（公式树拓扑，用于评估匹配）
  src    — 来源（原始公式字符串）
  merged — 累计合并次数（概念稳定性指标）

帧在记忆体中竞争：高频匹配的模式 → 合并 → weight↑
                               → 存活
孤立的输入 → 低weight → induction淘汰
                               → 消散

2.2 向量化编码 (symbol_vector)
──────────────────────

输入公式 → 符号频率向量（归一化）
  例：swap(1,2)=swap(2,1)
       → 向量 = [0,0,...,swap:2/5,const:4/5,...]

_ALPHABET 是预设的"符号槽"（教学脚手架）：
  索引  符号          域
  ────────────────────────────
  0-11  0,1,s,+,×,...  PA（皮亚诺算术）
  12-14 swap,pair,comm 实验一（代数对称）
  15-18 set,succ,...   实验二（归纳法）
  19-24 point,line,... 实验三（空间逻辑）
  25-26 fn,const       回退槽（未知符号）

设计意图：
  维表是教学的梯子——认知元不知道它的存在。
  未来可扩展为动态自命名（新符号→自动分配新维度）。

2.3 帧合并 (Memory.observe)
──────────────────────

核心动力学（每一步）：

  输入公式 → 计算向量 + 结构签名
    ↓
  找最近帧 (Euclidean distance on vec)
    ↓
  距离 < 自适应阈值? →
    YES: 合并到最近帧（质心移动, weight+1）
    NO:  创建新帧（weight=1.0）
    ↓
  容量满? → 淘汰最低 weight 帧

关键设计：
  ✓ 无梯度、无损失函数、无优化器
  ✓ 唯一动力：结构相似性 → 合并 → weight增长
  ✓ 阈值从数据中自适应（成功合并距离的中位数）

2.4 自适应阈值
──────────────────────

关键设计哲学：系统自己决定"什么算相似"。

  bootstrap阶段：收集前10步的邻近距离，取25%分位数 → 初始阈值
  稳态阶段：使用成功合并距离的中位数
  零距离保护：最近合并距离的50%作为阈值下限（防死锁）

无需硬编码的相似度阈值。

2.5 共现自指 (Co-occurrence)
──────────────────────

这是 GEME 最核心的创新——S4 自指的内生实现。

  滑动窗口（默认80步）追踪哪些签名经常在相近位置出现
  窗口内距离 ≤ 3 步的公式对 → 共现计数 +1
  共现比率超过阈值 → 创建"关联帧" (──)
  关联帧之间有共享元素 → 创建"链帧" (══)

层级产出：
  L0（主帧）：从输入流直接合并的模式
  L1（──）：概念关联（"seg 和 angle 经常一起出现"）
  L2（══）：推理路径（"seg→angle→parallel 是一条链"）

共现 = 系统在观察自己的记忆活动 = 内源自指。
不需要外部注入哥德尔句子（S4退役）。

2.6 归纳淘汰 (induction_clean)
──────────────────────

类似"睡眠中的记忆巩固"：

  未合并帧（merged=0）：weight × 0.80
  低合并帧（merged<3）：weight × 0.95
  保留权重最高的 50%

  淘汰 = "这个模式不重要"的判定。


三、评估体系 (S分类的工程实现)
───────────────────────────────────────────────────────────────────────

输入一个测试公式：
  1. 计算结构签名
  2. 遍历内存帧，找到与测试签名重叠度 ≥ 75% 的高权重帧
  3. 找到 → S2（涌现）：这个概念系统已学得
  4. 未找到 → S3（未涌现）：系统尚未形成概念

S分类的工程映射：
  S0 = 输入噪声（未合并帧，weight=1.0）
  S1 = 基础事实（关联网络中的稳定模式）
  S2 = 可证明概念（高weight帧，可被签名匹配）
  S3 = 未识别模式（不符合任何现有概念）
  S4 = 自指（共现机制本身——不再作为独立类别）


四、参数配置 (可调但非必须)
───────────────────────────────────────────────────────────────────────

参数            默认值    含义                物理类比
────────────────────────────────────────────────────────────
memory_cap      10       最大帧数            工作记忆容量
cooccur_window  50-80    滑动窗口步数         注意时间窗口
cooccur_thresh  0.15-0.25共现触发阈值事件注意敏感度

验证结果：
  memory_cap: 4-16 范围内稳定 (2026-05-04 全域扫描)
  window: 30-120 范围内稳定
  阈值：自适应，无需调节

六、与外部工作的关系（非技术对比）
───────────────────────────────────────────────────────────────────────

GEME 不是：
  ✗ 大语言模型（无统计学习/概率建模）
  ✗ 定理证明器（无逻辑演算/搜索）
  ✗ 聚类算法（无显式距离度量/中心点选择）

GEME 是：
  ✓ Hebb 学习（1949）的符号泛化——增加帧合并与竞争淘汰
  ✓ 发生认识论（Piaget, 1936）的计算模型——概念在交互中建构
  ✓ 哥德尔定理的生成性转化（论文一）的工程实现
  ✓ "怪圈"（Hofstadter, 1979）的具体计算形式

GEME 的独特贡献：
  自指不是外部注入的悖论——是系统观察自身记忆活动的自然产物。
  概念层级不是设计者的分类——是信息竞争的内生结果。
  200行代码，零外部知识，实现了"认知范畴从信息流中内生建构"。
